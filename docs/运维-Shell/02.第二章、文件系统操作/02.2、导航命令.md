## 1. 🧭 基础导航命令


### 1.1 pwd - 显示当前位置


**💡 什么是pwd**
`pwd`全称是"Print Working Directory"，就是告诉你现在在文件系统的哪个位置。

```bash
# 显示当前所在目录的完整路径
pwd
# 输出示例：/home/username/documents
```

**🔸 实际用途**
- 确认当前位置，避免在错误目录执行命令
- 脚本中获取当前路径进行路径拼接
- 迷路时快速定位自己的位置

### 1.2 cd - 切换目录


**💡 什么是cd**
`cd`是"Change Directory"的缩写，用来在不同目录间跳转，就像在楼房里换房间。

**🔸 基本用法**
```bash
# 进入指定目录
cd /home/username/documents

# 进入相对路径目录
cd documents/projects
```

**🔸 特殊切换方式**
```
cd ~        回到用户主目录（家目录）
cd          不加参数也是回到主目录  
cd ..       回到上一级目录（父目录）
cd ../..    回到上上级目录
cd -        回到之前所在的目录
cd /        回到根目录
```

**💫 实用示例**
```bash
# 场景：从深层目录快速回家
/var/log/apache2/error$ cd ~
/home/username$

# 场景：在两个目录间快速切换
/home/username$ cd /var/log
/var/log$ cd -
/home/username$
```

## 2. 👀 目录内容查看


### 2.1 ls - 列出目录内容


**💡 什么是ls**
`ls`是"List"的缩写，用来查看目录里有什么文件和文件夹，就像打开抽屉看里面有什么东西。

**🔸 基本用法**
```bash
# 显示当前目录内容
ls

# 显示指定目录内容
ls /home/username/documents
```

### 2.2 ls常用选项详解


**📋 核心选项对比**

| 选项 | **含义** | **作用** | **适用场景** |
|------|---------|---------|-------------|
| `ls -l` | `详细列表` | `显示文件权限、大小、时间等详细信息` | `查看文件属性` |
| `ls -a` | `显示全部` | `包括隐藏文件（以.开头的文件）` | `查看配置文件` |
| `ls -h` | `人性化显示` | `文件大小用K、M、G等单位显示` | `查看文件大小` |
| `ls -t` | `按时间排序` | `最新修改的文件排在前面` | `找最新文件` |
| `ls -r` | `倒序排列` | `反转排序顺序` | `配合-t找最老文件` |

**💫 组合使用示例**
```bash
# 显示详细信息且包含隐藏文件
ls -la

# 按修改时间排序显示详细信息
ls -lt

# 显示详细信息，文件大小用易读单位
ls -lh

# 找最旧的文件（时间倒序）
ls -ltr
```

**🔍 输出解读**
```bash
$ ls -l
drwxr-xr-x 2 user group 4096 Jan 15 10:30 documents
-rw-r--r-- 1 user group 1024 Jan 14 15:45 readme.txt

解读说明：
d          → 目录（文件则显示-）
rwxr-xr-x  → 权限信息  
2          → 链接数
user       → 所有者
group      → 所属组
4096       → 文件大小（字节）
Jan 15 10:30 → 最后修改时间
documents  → 文件/目录名
```

### 2.3 tree - 树形显示目录结构


**💡 什么是tree**
`tree`命令以树状图形式显示目录结构，让你一眼看清整个目录的层级关系。

**🌳 基本使用**
```bash
# 显示当前目录的树形结构
tree

# 显示指定目录的树形结构  
tree /home/username/projects
```

**📊 输出示例**
```
projects/
├── frontend/
│   ├── src/
│   │   ├── components/
│   │   └── pages/
│   └── package.json
├── backend/
│   ├── api/
│   └── database/
└── README.md
```

**⚙️ 常用选项**
```bash
# 只显示目录，不显示文件
tree -d

# 限制显示层级深度
tree -L 2

# 显示文件大小
tree -h
```

## 3. ⚡ 高级导航技巧


### 3.1 Tab键自动补全


**💡 什么是自动补全**
按Tab键可以让系统自动帮你补全命令、文件名或目录名，大大提高输入效率。

**🔸 使用方法**
```bash
# 输入部分命令后按Tab
cd doc[Tab]  → cd documents/

# 按两次Tab显示所有匹配选项
cd d[Tab][Tab]  → 显示所有以d开头的目录
```

**⭐ 补全规则**
- **唯一匹配**：直接补全完整名称
- **多个匹配**：补全到共同前缀，按两次Tab显示选项
- **无匹配**：不补全，可能发出提示音

### 3.2 路径补全技巧


**🎯 通配符使用**
```bash
# *匹配任意字符
ls *.txt        # 显示所有.txt文件
cd project*     # 进入以project开头的目录

# ?匹配单个字符  
ls file?.txt    # 匹配file1.txt、fileA.txt等

# []匹配指定字符
ls file[123].txt # 匹配file1.txt、file2.txt、file3.txt
```

### 3.3 快速目录切换策略


**🔸 使用环境变量**
```bash
# 设置常用目录的快捷方式
export PROJ="/home/username/projects"
export DOCS="/home/username/documents"

# 快速跳转
cd $PROJ
cd $DOCS
```

**🔸 创建软链接**
```bash
# 为常用目录创建软链接
ln -s /home/username/projects/important ~/quick

# 快速访问
cd ~/quick
```

## 4. 📚 目录栈管理


### 4.1 什么是目录栈


**💡 目录栈概念**
目录栈就像一摞盘子，你可以把目录路径"压入"栈中保存，需要时再"弹出"使用。这让你在多个目录间高效切换。

**🔄 目录栈工作原理**
```
栈的特点：后进先出（LIFO）

压栈过程：
/home/user     ← 当前位置
               ← 栈为空

pushd /var/log
/var/log       ← 当前位置  
/home/user     ← 栈底

pushd /etc
/etc           ← 当前位置
/var/log       ← 栈中
/home/user     ← 栈底
```

### 4.2 pushd - 压入目录栈


**🔸 基本用法**
```bash
# 进入新目录并将当前目录压入栈
pushd /var/log

# 压入相对路径目录
pushd documents/projects
```

**⚙️ pushd的作用**
- 切换到新目录
- 将之前的目录保存在栈中
- 可以随时通过popd返回

### 4.3 popd - 弹出目录栈


**🔸 基本用法**
```bash
# 返回栈顶目录（最后压入的目录）
popd

# 弹出但不切换（移除栈顶目录）
popd +0
```

### 4.4 dirs - 查看目录栈


**🔸 查看栈内容**
```bash
# 显示当前目录栈
dirs

# 显示栈中目录的完整路径
dirs -l

# 显示栈中每个目录的索引号
dirs -v
```

### 4.5 目录栈实战示例


**💫 实际使用场景**
```bash
# 当前在家目录
/home/username$ pwd
/home/username

# 需要到日志目录工作，但稍后要回来
/home/username$ pushd /var/log
/var/log /home/username

# 又需要到配置目录
/var/log$ pushd /etc
/etc /var/log /home/username

# 查看当前栈状态
/etc$ dirs -v
 0  /etc
 1  /var/log  
 2  /home/username

# 完成工作，返回日志目录
/etc$ popd
/var/log /home/username

# 再返回家目录
/var/log$ popd
/home/username
```

## 5. 🔍 文件信息查看


### 5.1 stat - 查看文件详细信息


**💡 什么是stat**
`stat`命令显示文件或目录的详细元数据信息，比`ls -l`更加详细全面。

**🔸 基本用法**
```bash
# 查看文件详细信息
stat filename.txt

# 查看目录信息
stat /home/username
```

**📊 输出信息解读**
```bash
$ stat example.txt
  File: example.txt
  Size: 1024         Blocks: 8          IO Block: 4096   regular file
Device: 801h/2049d   Inode: 123456      Links: 1
Access: (0644/-rw-r--r--)  Uid: (1000/username)   Gid: (1000/username)
Access: 2025-01-15 10:30:45.123456789 +0800
Modify: 2025-01-15 10:25:30.987654321 +0800
Change: 2025-01-15 10:25:30.987654321 +0800
Birth: -
```

**🔍 关键信息说明**
- **Size**: 文件大小（字节）
- **Access**: 最后访问时间
- **Modify**: 最后修改时间  
- **Change**: 状态改变时间
- **Links**: 硬链接数量
- **Inode**: 文件系统中的唯一标识

### 5.2 stat的实用场景


**⭐ 常见用途**
```bash
# 检查文件是否被修改
stat -c %Y filename.txt  # 只显示修改时间戳

# 比较两个文件的修改时间
stat -c %Y file1.txt file2.txt

# 获取文件权限
stat -c %a filename.txt  # 以数字形式显示权限
```

## 6. 🚀 导航效率提升


### 6.1 目录书签功能


**💡 什么是目录书签**
虽然shell没有内置书签功能，但我们可以通过别名和函数来实现类似功能。

**🔸 创建目录别名**
```bash
# 在.bashrc中添加常用目录别名
alias proj='cd /home/username/projects'
alias docs='cd /home/username/documents'  
alias logs='cd /var/log'

# 重新加载配置
source ~/.bashrc

# 使用别名快速切换
proj  # 等同于 cd /home/username/projects
```

### 6.2 快速导航函数


**🔧 自定义导航函数**
```bash
# 在.bashrc中添加智能cd函数
function c() {
    if [ -d "$1" ]; then
        cd "$1" && ls
    else
        echo "目录不存在: $1"
    fi
}

# 使用示例
c documents  # 切换到documents并显示内容
```

### 6.3 历史目录快速访问


**⚡ 使用历史记录**
```bash
# 查看最近访问的目录
dirs -v

# 快速切换到历史目录
cd ~1  # 切换到目录栈中索引为1的目录
cd ~2  # 切换到目录栈中索引为2的目录
```

### 6.4 导航最佳实践


**✅ 高效导航策略**

> 💡 **新手提示**：合理使用这些技巧可以将导航效率提升3-5倍

```
基础策略：
• 多用Tab补全，减少打字
• 善用cd -在两个目录间切换  
• 使用cd ..快速返回上级

进阶策略：
```

## 7. 📋 核心要点总结


### 7.1 必须掌握的基础命令


**🔸 核心导航三件套**
```
pwd  → 知道自己在哪里
cd   → 去想去的地方  
ls   → 看看周围有什么
```

### 7.2 关键命令速查表


| 命令 | **作用** | **常用选项** | **使用场景** |
|------|---------|-------------|-------------|
| `pwd` | `显示当前目录` | `无` | `确认位置` |
| `cd` | `切换目录` | `~、..、-` | `目录切换` |
| `ls` | `列出内容` | `-l、-a、-h、-t` | `查看文件` |
| `tree` | `树形显示` | `-d、-L、-h` | `目录结构` |
| `stat` | `详细信息` | `-c` | `文件属性` |

### 7.3 高效导航记忆要点


**🎯 效率提升技巧**
```
Tab补全：输入少量字符后按Tab自动补全
cd -：在最近两个目录间快速切换
pushd/popd：多目录工作时的神器
ls -la：查看详细信息包括隐藏文件
```

**🔹 常见使用模式**
```
日常工作流：
1. pwd 确认当前位置
2. ls 查看当前目录内容  
3. cd 切换到目标目录
4. 重复上述过程

多目录工作：
1. pushd 保存当前目录并切换
2. 在新目录完成工作
3. popd 返回原目录
4. 继续原来的工作
```

### 7.4 新手避坑指南


> ⚠️ **注意**：这些是新手常犯的错误

```
常见错误：
❌ 不确认当前位置就执行危险命令
❌ 路径中有空格时不加引号
❌ 混淆相对路径和绝对路径
❌ 忘记使用Tab补全浪费时间

正确做法：
✅ 执行命令前先pwd确认位置
✅ 路径有空格用引号包围
✅ 理解./（当前目录）和../（上级目录）
✅ 养成使用Tab补全的习惯
```

**核心记忆口诀**：
- pwd知位置，cd来切换
- ls看内容，tree显层级  
- Tab键补全省时间
- pushd popd管栈道
