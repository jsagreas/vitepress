## 1. 🔗 什么是事件绑定


### 1.1 事件绑定的本质

**简单理解**：事件绑定就是告诉Vue"当用户做了某个操作时，执行什么功能"

```
现实中的开关：
按下开关 → 灯亮了

Vue中的事件：
点击按钮 → 执行函数
```

### 1.2 为什么需要事件

**💡 核心作用**：
- **响应用户操作** - 点击、输入、滚动等
- **触发功能执行** - 保存数据、跳转页面、显示弹窗
- **实现交互效果** - 动画、状态切换、内容更新

### 1.3 常见的用户事件

```
鼠标事件：点击(click)、双击(dblclick)、移入(mouseenter)
键盘事件：按键(keydown)、抬起(keyup)、输入(input)
表单事件：提交(submit)、重置(reset)、失焦(blur)
页面事件：加载(load)、滚动(scroll)、窗口变化(resize)
```

## 2. 📝 v-on基础语法


### 2.1 完整写法 vs 简写

```vue
<!-- 完整写法 -->
<button v-on:click="handleClick">点击我</button>

<!-- 简写语法（推荐） -->
<button @click="handleClick">点击我</button>
```

**🔸 为什么推荐简写**：
- 代码更简洁美观
- 写起来更快
- Vue官方推荐的标准写法

### 2.2 基础事件示例

```vue
<template>
  <div>
    <!-- 点击事件 -->
    <button @click="sayHello">问候</button>
    
    <!-- 双击事件 -->
    <div @dblclick="showMessage">双击我</div>
    
    <!-- 鼠标移入移出 -->
    <div @mouseenter="onEnter" @mouseleave="onLeave">
      鼠标悬停区域
    </div>
  </div>
</template>

<script>
export default {
  methods: {
    sayHello() {
      alert('你好！')
    },
    showMessage() {
      console.log('你双击了我')
    },
    onEnter() {
      console.log('鼠标进入了')
    },
    onLeave() {
      console.log('鼠标离开了')
    }
  }
}
</script>
```

### 2.3 内联表达式

```vue
<template>
  <div>
    <!-- 直接写简单代码 -->
    <button @click="count++">计数: {{ count }}</button>
    
    <!-- 调用方法并传参 -->
    <button @click="increase(5)">增加5</button>
    
    <!-- 条件判断 -->
    <button @click="isVisible = !isVisible">
      {{ isVisible ? '隐藏' : '显示' }}
    </button>
  </div>
</template>

<script>
export default {
  data() {
    return {
      count: 0,
      isVisible: true
    }
  },
  methods: {
    increase(num) {
      this.count += num
    }
  }
}
</script>
```

## 3. 🎯 事件对象与参数传递


### 3.1 事件对象的作用

**简单理解**：事件对象就像一个"信息包"，包含了事件发生时的各种信息

```
事件对象包含什么？
- 触发事件的元素是谁
- 鼠标点击的具体位置
- 按了哪个键盘按键
- 是否同时按了Ctrl键等
```

### 3.2 获取事件对象

```vue
<template>
  <div>
    <!-- 方法1：默认传入事件对象 -->
    <button @click="handleClick">获取事件信息</button>
    
    <!-- 方法2：明确使用$event -->
    <button @click="handleClick($event)">明确传入事件</button>
    
    <!-- 方法3：传递自定义参数 + 事件对象 -->
    <button @click="handleClick('用户A', $event)">传递多个参数</button>
  </div>
</template>

<script>
export default {
  methods: {
    // 情况1：直接接收事件对象
    handleClick(event) {
      console.log('点击的元素：', event.target)
      console.log('鼠标位置：', event.clientX, event.clientY)
    },
    
    // 情况2：接收自定义参数和事件对象
    handleClick(userName, event) {
      console.log('用户：', userName)
      console.log('事件：', event)
    }
  }
}
</script>
```

### 3.3 常用事件属性

```javascript
// 事件对象常用属性
methods: {
  handleEvent(event) {
    // 触发事件的元素
    console.log('目标元素：', event.target)
    
    // 鼠标位置（相对于浏览器窗口）
    console.log('鼠标位置：', event.clientX, event.clientY)
    
    // 按键信息
    console.log('按键代码：', event.keyCode)
    console.log('按键名称：', event.key)
    
    // 修饰键状态
    console.log('是否按了Ctrl：', event.ctrlKey)
    console.log('是否按了Alt：', event.altKey)
  }
}
```

## 4. 🛡️ 事件修饰符详解


### 4.1 什么是事件修饰符

**简单理解**：修饰符就像给事件加上"特殊指令"，让事件按照我们想要的方式工作

```
普通事件：@click="doSomething"
加修饰符：@click.stop="doSomething"  ← .stop就是修饰符
```

### 4.2 .stop - 阻止事件冒泡

**🔸 什么是事件冒泡**：
```
HTML结构：
<div class="外层" @click="外层函数">
  <button class="内层" @click="内层函数">按钮</button>
</div>

不加.stop的结果：
点击按钮 → 先执行"内层函数" → 再执行"外层函数"

加.stop的结果：
点击按钮 → 只执行"内层函数" → 不执行外层函数
```

**实际示例**：
```vue
<template>
  <div class="outer" @click="outerClick">
    外层区域
    <!-- 不阻止冒泡 -->
    <button @click="innerClick">普通按钮</button>
    
    <!-- 阻止冒泡 -->
    <button @click.stop="innerClick">阻止冒泡按钮</button>
  </div>
</template>

<script>
export default {
  methods: {
    outerClick() {
      console.log('点击了外层')
    },
    innerClick() {
      console.log('点击了按钮')
    }
  }
}
</script>
```

### 4.3 .prevent - 阻止默认行为

**🔸 什么是默认行为**：
```
<a>标签 → 点击后跳转页面
<form>表单 → 提交后刷新页面
右键菜单 → 显示浏览器菜单
```

```vue
<template>
  <div>
    <!-- 阻止链接跳转 -->
    <a href="https://www.baidu.com" @click.prevent="handleClick">
      不会跳转的链接
    </a>
    
    <!-- 阻止表单提交 -->
    <form @submit.prevent="handleSubmit">
      <input type="text" v-model="username">
      <button type="submit">提交</button>
    </form>
    
    <!-- 阻止右键菜单 -->
    <div @contextmenu.prevent="showCustomMenu">
      右键我试试
    </div>
  </div>
</template>
```

### 4.4 .once - 只触发一次

```vue
<template>
  <div>
    <!-- 只能点击一次 -->
    <button @click.once="importantAction">
      重要操作（仅能执行一次）
    </button>
    
    <!-- 普通按钮可以多次点击 -->
    <button @click="normalAction">
      普通操作（可多次执行）
    </button>
  </div>
</template>

<script>
export default {
  methods: {
    importantAction() {
      console.log('这个函数只会执行一次')
    },
    normalAction() {
      console.log('这个函数可以多次执行')
    }
  }
}
</script>
```

### 4.5 .self - 只在元素自身触发

**🔸 使用场景**：当你只想在点击元素本身时触发，点击子元素不触发

```vue
<template>
  <div class="container" @click.self="containerClick">
    点击这个文字会触发事件
    <button @click="buttonClick">点击按钮不会触发容器事件</button>
    <p>点击这段文字也不会触发容器事件</p>
  </div>
</template>

<script>
export default {
  methods: {
    containerClick() {
      console.log('只有点击容器空白区域才会执行')
    },
    buttonClick() {
      console.log('点击按钮执行')
    }
  }
}
</script>
```

### 4.6 修饰符组合使用

```vue
<template>
  <div>
    <!-- 多个修饰符组合 -->
    <button @click.stop.prevent.once="complexHandler">
      复杂事件处理
    </button>
    
    <!-- 顺序很重要 -->
    <div @click.prevent.self="handler1">方式1</div>
    <div @click.self.prevent="handler2">方式2</div>
  </div>
</template>
```

## 5. ⌨️ 键盘修饰符


### 5.1 常用键盘修饰符

**🔸 基本按键**：
```vue
<template>
  <div>
    <!-- 回车键 -->
    <input @keyup.enter="submitForm" placeholder="按回车提交">
    
    <!-- ESC键 -->
    <input @keyup.esc="cancelInput" placeholder="按ESC取消">
    
    <!-- 空格键 -->
    <input @keyup.space="addSpace" placeholder="按空格添加">
    
    <!-- Tab键 -->
    <input @keydown.tab="handleTab" placeholder="按Tab切换">
    
    <!-- 删除键 -->
    <input @keyup.delete="handleDelete" placeholder="按删除键清空">
  </div>
</template>
```

### 5.2 箭头键修饰符

```vue
<template>
  <div>
    <!-- 方向键控制 -->
    <div @keyup.up="moveUp" 
         @keyup.down="moveDown"
         @keyup.left="moveLeft" 
         @keyup.right="moveRight"
         tabindex="0">
      使用方向键控制移动
    </div>
  </div>
</template>

<script>
export default {
  methods: {
    moveUp() { console.log('向上移动') },
    moveDown() { console.log('向下移动') },
    moveLeft() { console.log('向左移动') },
    moveRight() { console.log('向右移动') }
  }
}
</script>
```

### 5.3 自定义键码

```vue
<template>
  <div>
    <!-- 使用键码 -->
    <input @keyup.13="handleEnter" placeholder="回车键(keyCode:13)">
    
    <!-- 使用键名 -->
    <input @keyup.f1="showHelp" placeholder="按F1显示帮助">
    
    <!-- 字母键 -->
    <input @keyup.a="handleA" placeholder="按字母A">
  </div>
</template>
```

### 5.4 实用键盘交互示例

```vue
<template>
  <div>
    <!-- 搜索框：回车搜索，ESC清空 -->
    <input 
      v-model="searchText"
      @keyup.enter="doSearch"
      @keyup.esc="clearSearch"
      placeholder="输入搜索内容，回车搜索，ESC清空">
    
    <!-- 计数器：上下箭头调整数值 -->
    <input 
      type="number" 
      v-model="counter"
      @keyup.up="counter++"
      @keyup.down="counter--"
      placeholder="用上下箭头调整数值">
  </div>
</template>

<script>
export default {
  data() {
    return {
      searchText: '',
      counter: 0
    }
  },
  methods: {
    doSearch() {
      console.log('搜索：', this.searchText)
    },
    clearSearch() {
      this.searchText = ''
    }
  }
}
</script>
```

## 6. 🖥️ 系统修饰符


### 6.1 组合键修饰符

**🔸 常用组合键**：
```vue
<template>
  <div>
    <!-- Ctrl + 点击 -->
    <button @click.ctrl="ctrlClick">Ctrl+点击</button>
    
    <!-- Alt + 点击 -->
    <button @click.alt="altClick">Alt+点击</button>
    
    <!-- Shift + 点击 -->
    <button @click.shift="shiftClick">Shift+点击</button>
    
    <!-- Meta(Win/Cmd) + 点击 -->
    <button @click.meta="metaClick">Win/Cmd+点击</button>
  </div>
</template>
```

### 6.2 键盘组合快捷键

```vue
<template>
  <div tabindex="0" @keyup.ctrl.s.prevent="saveDocument">
    <!-- Ctrl + S 保存 -->
    按 Ctrl+S 保存文档
  </div>
  
  <div>
    <!-- Ctrl + Enter 快速提交 -->
    <textarea 
      @keyup.ctrl.enter="quickSubmit"
      placeholder="Ctrl+Enter快速提交">
    </textarea>
  </div>
</template>

<script>
export default {
  methods: {
    saveDocument() {
      console.log('保存文档')
    },
    quickSubmit() {
      console.log('快速提交')
    }
  }
}
</script>
```

### 6.3 .exact - 精确匹配

**🔸 问题**：`@click.ctrl`会在按Ctrl+Alt+点击时也触发
**🔸 解决**：使用`.exact`确保只有指定键被按下

```vue
<template>
  <div>
    <!-- 只有Ctrl+点击才触发 -->
    <button @click.ctrl.exact="onlyCtrl">仅Ctrl+点击</button>
    
    <!-- 没有修饰键的点击 -->
    <button @click.exact="onlyClick">仅点击（无修饰键）</button>
    
    <!-- Ctrl+Shift+点击 -->
    <button @click.ctrl.shift.exact="ctrlShiftClick">
      Ctrl+Shift+点击
    </button>
  </div>
</template>
```

## 8. 📋 核心要点总结


### 8.1 必须掌握的基础概念

```
🔸 事件绑定：让元素能够响应用户操作
🔸 v-on语法：v-on:click 或简写 @click
🔸 事件对象：包含事件信息的"数据包"
🔸 修饰符：给事件添加特殊行为的"指令"
```

### 8.2 常用修饰符速查表


| 修饰符类型 | **修饰符** | **作用** | **使用场景** |
|-----------|-----------|---------|-------------|
| **事件控制** | `.stop` | 阻止冒泡 | 避免父元素事件触发 |
| | `.prevent` | 阻止默认行为 | 阻止链接跳转、表单提交 |
| | `.once` | 只触发一次 | 重要操作、初始化事件 |
| | `.self` | 只在自身触发 | 容器点击但不包括子元素 |
| **键盘事件** | `.enter` | 回车键 | 表单提交、确认操作 |
| | `.esc` | ESC键 | 取消操作、关闭弹窗 |
| | `.space` | 空格键 | 播放暂停、选择切换 |
| | `.tab` | Tab键 | 焦点切换 |
| **系统修饰** | `.ctrl` | Ctrl键 | 快捷键组合 |
| | `.alt` | Alt键 | 备选操作 |
| | `.shift` | Shift键 | 多选操作 |
| **鼠标事件** | `.left` | 左键 | 普通点击 |
| | `.right` | 右键 | 右键菜单 |
| | `.middle` | 中键 | 新标签页打开 |

### 8.3 最佳实践要点


**🔹 事件命名规范**
```javascript
// ✅ 推荐：动词 + 名词
handleClick、submitForm、showMenu

// ❌ 不推荐：不清晰的命名
click、submit、show
```

**🔹 参数传递技巧**
```vue
<!-- ✅ 明确传递事件对象 -->
<button @click="handleClick($event)">按钮</button>

<!-- ✅ 传递多个参数 -->
<button @click="handleClick(id, $event)">按钮</button>

<!-- ✅ 传递额外数据 -->
<button @click="handleClick({id: 1, name: 'test'})">按钮</button>
```

**🔹 修饰符使用原则**
```
简单优先：能用修饰符解决的不写额外代码
组合使用：多个修饰符可以链式组合
顺序重要：修饰符的顺序会影响执行结果
语义清晰：选择最能表达意图的修饰符
```

### 8.4 常见应用场景


**🎯 表单交互**
- 回车提交：`@keyup.enter`
- ESC取消：`@keyup.esc`
- 阻止默认提交：`@submit.prevent`

**🎯 菜单操作**
- 阻止冒泡：`@click.stop`
- 右键菜单：`@contextmenu.prevent`
- 快捷键：`@keyup.ctrl.s`

**🎯 列表操作**
- 单选：`@click`
- 多选：`@click.ctrl`
- 全选：`@click.shift`

**核心记忆口诀**：
```
事件绑定@开头，修饰符点号连
stop阻冒泡，prevent禁默认
once只一次，self仅自身
键盘enter回车键，esc取消很常见
ctrl组合做快捷，鼠标左右中分明
```
