## 1. 📊 图的基本概念


### 1.1 什么是图

**定义**：图(Graph)是由顶点(Vertex)和边(Edge)组成的数据结构，用于表示对象之间的关系。

**数学定义**：G = (V, E)
- V：顶点集合，非空有限集合
- E：边集合，顶点对的集合

```
现实世界中的图：
🌐 社交网络：人是顶点，关系是边
🗺️ 地图导航：城市是顶点，道路是边  
🔗 网页链接：网页是顶点，超链接是边
🏫 课程依赖：课程是顶点，先修关系是边
```

### 1.2 图与其他数据结构的区别

```
线性结构：  A → B → C → D
           只有前后关系

树结构：        A
              / \
             B   C        只有父子关系
            / \
           D   E

图结构：     A ←→ B
           ↙  ↘  ↓     可以有任意关系
          C ←→ D ←→ E    更加灵活复杂
```

### 1.3 图的基本组成

```
图的基本元素：

G = (V, E)
V = {A, B, C, D}     // 顶点集合
E = {(A,B), (B,C), (C,D), (A,D)}  // 边集合

图形表示：
    A ————— B
    |       |
    |       |  
    D ————— C

顶点A的相邻顶点：B, D
边(A,B)连接顶点A和B
```

## 2. 🔄 图的分类与特性


### 2.1 按边的方向分类


**无向图(Undirected Graph)**：
```
特征：边没有方向，表示双向关系
表示：(u,v) 等价于 (v,u)
示例：朋友关系、道路连接

    A ——— B
    |     |
    |     |
    D ——— C

边集：{(A,B), (B,C), (C,D), (A,D)}
```

**有向图(Directed Graph/Digraph)**：
```
特征：边有方向，表示单向关系
表示：<u,v> 不等价于 <v,u>
示例：网页链接、依赖关系

    A ——→ B
    ↑     ↓
    |     |
    D ←—— C

边集：{<A,B>, <B,C>, <C,D>, <D,A>}
```

### 2.2 按边的权重分类


**无权图(Unweighted Graph)**：
```
特征：边没有权重，只表示连接关系
应用：社交网络、网络拓扑

    A ——— B
    |     |
    D ——— C
```

**加权图(Weighted Graph)**：
```
特征：边带有权重，表示代价或距离
应用：最短路径、网络流

    A —5— B
    |     |
    3     2
    |     |
    D —4— C

边集：{(A,B,5), (B,C,2), (C,D,4), (A,D,3)}
```

### 2.3 特殊图类型


**完全图(Complete Graph)**：
```
定义：任意两个顶点之间都有边
无向完全图：K_n，有 n(n-1)/2 条边
有向完全图：有 n(n-1) 条边

K_4 完全图：
    A ——— B
   /|    /|
  / |   / |
 D——+——C  |
  \  |  \ |
   \ |   \|
    \——————

每个顶点度数：n-1
```

**稀疏图与稠密图**：
```
稀疏图：边数相对较少，|E| << |V|²
稠密图：边数相对较多，|E| ≈ |V|²

影响算法选择：
- 稀疏图适合邻接表
- 稠密图适合邻接矩阵
```

## 3. 📋 图的表示方法


### 3.1 邻接矩阵(Adjacency Matrix)


**基本概念**：
```
定义：用二维数组表示图，matrix[i][j] = 1表示顶点i和j之间有边

无向图邻接矩阵：
    A ——— B
    |     |
    D ——— C

    A B C D
A [ 0 1 0 1 ]
B [ 1 0 1 0 ]  
C [ 0 1 0 1 ]
D [ 1 0 1 0 ]

特点：矩阵关于主对角线对称
```

**实现代码**：
```java
class GraphMatrix {
    private int[][] matrix;
    private int vertexCount;
    
    public GraphMatrix(int n) {
        vertexCount = n;
        matrix = new int[n][n];
    }
    
    // 添加边
    public void addEdge(int i, int j) {
        matrix[i][j] = 1;
        matrix[j][i] = 1;  // 无向图
    }
    
    // 检查是否有边
    public boolean hasEdge(int i, int j) {
        return matrix[i][j] == 1;
    }
}
```

**加权图邻接矩阵**：
```
用权重值代替0和1：
    A —5— B
    |     |
    3     2
    |     |
    D —4— C

    A B C D
A [ 0 5 ∞ 3 ]
B [ 5 0 2 ∞ ]
C [ ∞ 2 0 4 ]
D [ 3 ∞ 4 0 ]

∞表示无边，0表示自环
```

### 3.2 邻接表(Adjacency List)


**基本概念**：
```
定义：为每个顶点维护一个链表，存储其相邻顶点

同样的图：
    A ——— B
    |     |
    D ——— C

邻接表表示：
A: [B, D]
B: [A, C]
C: [B, D]  
D: [A, C]
```

**实现代码**：
```java
class GraphList {
    private List<List<Integer>> adjList;
    private int vertexCount;
    
    public GraphList(int n) {
        vertexCount = n;
        adjList = new ArrayList<>();
        for (int i = 0; i < n; i++) {
            adjList.add(new ArrayList<>());
        }
    }
    
    // 添加边
    public void addEdge(int u, int v) {
        adjList.get(u).add(v);
        adjList.get(v).add(u);  // 无向图
    }
    
    // 获取邻接顶点
    public List<Integer> getNeighbors(int u) {
        return adjList.get(u);
    }
}
```

**加权图邻接表**：
```java
class WeightedEdge {
    int to;
    int weight;
    
    public WeightedEdge(int to, int weight) {
        this.to = to;
        this.weight = weight;
    }
}

class WeightedGraphList {
    private List<List<WeightedEdge>> adjList;
    
    public void addEdge(int u, int v, int weight) {
        adjList.get(u).add(new WeightedEdge(v, weight));
        adjList.get(v).add(new WeightedEdge(u, weight));
    }
}
```

### 3.3 两种表示方法对比


```
特性        | 邻接矩阵        | 邻接表
-----------|----------------|------------------
空间复杂度  | O(V²)          | O(V + E)
查找边     | O(1)           | O(度数)
添加边     | O(1)           | O(1)  
删除边     | O(1)           | O(度数)
遍历邻接点  | O(V)           | O(度数)
适用场景   | 稠密图         | 稀疏图
```

**选择建议**：
- **稠密图**：边数接近V²，选择邻接矩阵
- **稀疏图**：边数远小于V²，选择邻接表
- **频繁查边**：选择邻接矩阵
- **频繁遍历**：选择邻接表

## 4. 📝 图的基本术语


### 4.1 顶点相关术语


**度(Degree)**：
```
无向图：顶点的度 = 与该顶点相连的边数
有向图：入度 + 出度 = 总度数

示例：
    A ——— B
    |     |
    D ——— C

度数：deg(A) = 2, deg(B) = 2, deg(C) = 2, deg(D) = 2
握手定理：所有顶点度数之和 = 2 × 边数
```

**有向图的度**：
```
入度(In-degree)：指向该顶点的边数
出度(Out-degree)：从该顶点出发的边数

    A ——→ B
    ↑     ↓
    |     |
    D ←—— C

入度：in(A)=1, in(B)=1, in(C)=1, in(D)=1
出度：out(A)=1, out(B)=1, out(C)=1, out(D)=1
```

### 4.2 路径相关术语


**路径(Path)**：
```
定义：从顶点u到顶点v的顶点序列
简单路径：没有重复顶点的路径
路径长度：路径上边的数量

    A ——— B
    |     |
    D ——— C

从A到C的路径：
- A → B → C (长度为2)
- A → D → C (长度为2)
```

**回路/环(Cycle)**：
```
定义：起点和终点相同的路径
简单回路：除起点外没有重复顶点

    A ——— B
    |     |
    D ——— C

简单回路：A → B → C → D → A
```

### 4.3 连通性术语


**连通(Connected)**：
```
无向图：任意两顶点间存在路径
连通图：整个图是连通的
连通分量：最大连通子图

示例：
连通图：           非连通图：
    A — B             A — B    E
    |   |             |   |    |
    D — C             D — C    F
```

**强连通(Strongly Connected)**：
```
有向图：任意两顶点间都存在有向路径
强连通图：整个有向图是强连通的
强连通分量：最大强连通子图

    A ——→ B
    ↑     ↓
    |     |
    D ←—— C

强连通：A可达B，B也可达A
```

### 4.4 特殊顶点和边


**割点(Cut Vertex)**：
```
定义：删除该顶点及其关联边后，图的连通分量增加

    A — B — C
        |
        D

B是割点，删除B后图分为{A}和{C,D}两个分量
```

**桥(Bridge)**：
```
定义：删除该边后，图的连通分量增加

    A — B — C
        |
        D

边(B,C)是桥，删除后图分为{A,B,D}和{C}
```

## 5. 📋 核心要点总结


### 5.1 必须掌握的核心概念


```
🔸 图的本质：顶点和边的集合，表示对象间关系
🔸 图的分类：有向/无向、加权/无权、稀疏/稠密
🔸 表示方法：邻接矩阵适合稠密图，邻接表适合稀疏图
🔸 基本术语：度、路径、连通性是后续算法基础
🔸 选择原则：根据图的特性选择合适的表示方法
```

### 5.2 关键理解要点


**🔹 图与树的关系**：
```
树是特殊的图：连通、无环、n个顶点有n-1条边
图更一般：可以有环、可以不连通、边数任意
```

**🔹 表示方法的选择**：
```
邻接矩阵：空间固定O(V²)，查边快O(1)
邻接表：空间省O(V+E)，遍历快O(度数)
实际应用中邻接表更常用
```

**🔹 术语的重要性**：
```
度数 → 决定遍历复杂度
路径 → 最短路径算法基础  
连通性 → 图遍历和搜索基础
环 → 拓扑排序和检测算法
```


**核心记忆**：
- 图 = 顶点 + 边，关系建模工具
- 矩阵查边快，链表省空间
- 度数路径连通性，算法基础要记牢
- 有向无向要区分，加权无权看应用
