## 1. 🎯 什么是ORM框架


### 1.1 ORM的通俗理解


**ORM全称**：Object-Relational Mapping（对象关系映射）

**大白话解释**：
```
想象一下：
你和你的朋友说话 → 用中文交流 ✅
你和数据库说话 → 数据库只懂SQL语句 ❌

ORM就像一个"翻译官"：
你说人话(Java对象) → ORM翻译成SQL → 数据库理解并执行
```

**举个生活例子**：
```
传统方式（没有ORM）：
你想查询一个用户信息，需要自己写SQL：
SELECT id, name, age FROM user WHERE id = 1

有了ORM：
你只需要写Java代码：
User user = userMapper.selectById(1);

ORM自动帮你生成SQL，你根本不用写！
```

### 1.2 ORM解决的核心问题


**问题1：Java对象和数据库表不是一个东西**
```
Java世界：                数据库世界：
User对象                  user表
├── id (Long)            ├── id (BIGINT)
├── name (String)        ├── name (VARCHAR)
└── age (Integer)        └── age (INT)

问题：两边格式不一样，怎么对应起来？
ORM的作用：自动建立映射关系，帮你转换
```

**问题2：重复写SQL太麻烦**
```
没有ORM时，你要写很多重复代码：

// 增加用户
String sql = "INSERT INTO user(name, age) VALUES(?, ?)";

// 删除用户  
String sql = "DELETE FROM user WHERE id = ?";

// 修改用户
String sql = "UPDATE user SET name=?, age=? WHERE id=?";

// 查询用户
String sql = "SELECT * FROM user WHERE id = ?";

有了ORM：
userMapper.insert(user);    // 增
userMapper.deleteById(id);  // 删
userMapper.updateById(user);// 改
userMapper.selectById(id);  // 查

这些SQL都自动生成，省事！
```

**ORM框架大家族**：
```
┌─────────────────────────────────┐
│        ORM框架对比               │

│                                 │
│  Hibernate (JPA)                │

│                                 │
│  MyBatis                        │
│  └─ 半自动，你写SQL            │
│     很灵活但麻烦                │
│                                 │

│  └─ 增强版MyBatis              │
│     既灵活又省事                │
│                                 │
└─────────────────────────────────┘
```

**MyBatis-Plus的特点** `[核心优势]`：
- **基于MyBatis** - 继承MyBatis所有功能

- **只做增强** - 只增加功能，不做改变
- **自动化程度高** - 基本的CRUD都不用写

## 3. 🏗️ 单体与微服务架构适配


### 3.1 单体应用中的使用


**什么是单体应用**：
```
简单理解：所有功能在一个项目里

单体应用结构：
┌────────────────────────┐
│      一个大项目         │
│                        │
│  用户模块              │
│  订单模块              │
│  商品模块              │
│  ...                   │
│                        │
│  共用一个数据库        │
└────────────────────────┘
```

**在单体应用中使用MyBatis-Plus**：
```java
// 配置超级简单
@Configuration
public class MybatisPlusConfig {
    // 基本上什么都不用配置
    // MP自动扫描Mapper
}

// 使用也简单
@Service
public class UserService {
    @Autowired
    private UserMapper userMapper;
    
    public User getUser(Long id) {
        return userMapper.selectById(id);
    }
}
```

**优势分析**：
- ✅ 配置简单，开箱即用
- ✅ 统一的数据访问方式
- ✅ 代码量减少60%以上

### 3.2 微服务架构中的使用


**什么是微服务**：
```
简单理解：把大项目拆成多个小项目

微服务架构：
┌──────────┐  ┌──────────┐  ┌──────────┐
│用户服务  │  │订单服务  │  │商品服务  │
├──────────┤  ├──────────┤  ├──────────┤
│  用户DB  │  │  订单DB  │  │  商品DB  │
└──────────┘  └──────────┘  └──────────┘

每个服务独立部署，独立数据库
```

**在微服务中使用MyBatis-Plus**：

| 服务模块 | **使用方式** | **优势** |
|---------|------------|---------|
| 用户服务 | `独立配置MP` | `独立维护，互不影响` |
| 订单服务 | `独立配置MP` | `可以有不同的配置` |
| 商品服务 | `独立配置MP` | `服务隔离性好` |

**实际配置示例**：
```yaml
# 用户服务的配置
mybatis-plus:
  mapper-locations: classpath:mapper/user/**/*.xml
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl

# 订单服务的配置（可以不一样）
mybatis-plus:
  mapper-locations: classpath:mapper/order/**/*.xml
  global-config:
    db-config:
      logic-delete-value: 1
```

```xml

    SELECT id, name, age, email FROM user WHERE id = #{id}
</select>

<insert id="insert">
    INSERT INTO user(name, age, email) VALUES(#{name}, #{age}, #{email})
</insert>

<update id="update">
    UPDATE user SET name=#{name}, age=#{age} WHERE id=#{id}
</update>

<delete id="deleteById">
    DELETE FROM user WHERE id = #{id}
</delete>
```

**使用MyBatis-Plus**：
```java
// 什么都不用写！
```
