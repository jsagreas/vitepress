## 1. 🎯 页面布局基础概念


### 1.1 什么是页面布局


**通俗解释**：页面布局就像搭积木，把网页的各个部分（导航、内容、侧边栏等）合理地摆放在页面上。

```
典型网页结构：
┌─────────────────────────────┐
│         顶部导航栏            │
├─────────────────────────────┤
│  侧边栏  │     主要内容区      │
│   菜单   │    文章/产品等      │
│         │                  │
├─────────────────────────────┤
│         底部版权信息          │
└─────────────────────────────┘
```

### 1.2 Bootstrap布局核心思想


**🔸 容器包裹**：所有内容都要放在容器里
```html
<div class="container">  <!-- 固定最大宽度 -->
<div class="container-fluid">  <!-- 100%宽度 -->
```

**🔸 行列结构**：用行(row)和列(col)组织内容
```html
<div class="row">
  <div class="col-md-3">侧边栏</div>
  <div class="col-md-9">主内容</div>
</div>
```

## 2. 🧭 导航栏+内容布局


### 2.1 基础导航栏结构


**核心概念**：导航栏是网站的"方向盘"，帮用户快速找到想要的内容。

```html
<!-- 基础导航+内容布局 -->
<nav class="navbar navbar-expand-lg navbar-dark bg-primary">
  <div class="container">
    <a class="navbar-brand" href="#">我的网站</a>
    <button class="navbar-toggler" data-bs-toggle="collapse" data-bs-target="#navbarNav">
      <span class="navbar-toggler-icon"></span>
    </button>
    <div class="collapse navbar-collapse" id="navbarNav">
      <ul class="navbar-nav ms-auto">
        <li class="nav-item"><a class="nav-link" href="#">首页</a></li>
        <li class="nav-item"><a class="nav-link" href="#">产品</a></li>
        <li class="nav-item"><a class="nav-link" href="#">关于</a></li>
      </ul>
    </div>
  </div>
</nav>

<!-- 主要内容区 -->
<div class="container mt-4">
  <div class="row">
    <div class="col-12">
      <h1>欢迎来到我的网站</h1>
      <p>这里是主要内容区域...</p>
    </div>
  </div>
</div>
```

### 2.2 固定导航栏


**实用场景**：让导航栏始终显示在顶部，用户随时可以导航。

```html
<!-- 固定在顶部的导航栏 -->
<nav class="navbar navbar-expand-lg navbar-dark bg-primary fixed-top">
  <!-- 导航内容 -->
</nav>

<!-- 内容需要添加顶部间距，避免被导航栏遮挡 -->
<div class="container" style="margin-top: 80px;">
  <h1>内容不会被导航栏遮挡</h1>
</div>
```

## 3. 📋 左侧边栏+主内容布局


### 3.1 经典侧边栏布局


**使用场景**：管理后台、文档网站、博客等需要分类导航的页面。

```html
<div class="container-fluid">
  <div class="row">
    <!-- 左侧边栏 -->
    <div class="col-md-3 col-lg-2 bg-light vh-100">
      <div class="sidebar p-3">
        <h5>菜单</h5>
        <ul class="nav flex-column">
          <li class="nav-item">
            <a class="nav-link active" href="#">📊 仪表盘</a>
          </li>
          <li class="nav-item">
            <a class="nav-link" href="#">👥 用户管理</a>
          </li>
          <li class="nav-item">
            <a class="nav-link" href="#">📝 内容管理</a>
          </li>
          <li class="nav-item">
            <a class="nav-link" href="#">⚙️ 系统设置</a>
          </li>
        </ul>
      </div>
    </div>
    
    <!-- 主内容区 -->
    <div class="col-md-9 col-lg-10">
      <div class="main-content p-4">
        <h2>主要内容区域</h2>
        <p>这里显示具体的页面内容...</p>
      </div>
    </div>
  </div>
</div>
```

### 3.2 响应式侧边栏


**关键技巧**：在手机上隐藏侧边栏，通过按钮切换显示。

```html
<!-- 移动端显示的菜单按钮 -->
<button class="btn btn-primary d-md-none mb-3" data-bs-toggle="collapse" data-bs-target="#sidebar">
  菜单 ☰
</button>

<!-- 可折叠的侧边栏 -->
<div class="col-md-3 collapse d-md-block" id="sidebar">
  <!-- 侧边栏内容 -->
</div>
```

## 4. 🃏 卡片网格布局


### 4.1 基础卡片网格


**应用场景**：产品展示、文章列表、团队介绍等需要整齐排列的内容。

```html
<div class="container">
  <div class="row g-4">  <!-- g-4 设置卡片间距 -->
    <!-- 卡片1 -->
    <div class="col-lg-4 col-md-6">
      <div class="card h-100">  <!-- h-100 让卡片等高 -->
        <img src="product1.jpg" class="card-img-top" alt="产品图片">
        <div class="card-body">
          <h5 class="card-title">产品名称</h5>
          <p class="card-text">这是产品的简要描述...</p>
          <a href="#" class="btn btn-primary">查看详情</a>
        </div>
      </div>
    </div>
    
    <!-- 重复更多卡片 -->
    <div class="col-lg-4 col-md-6">
      <div class="card h-100">
        <!-- 卡片内容 -->
      </div>
    </div>
  </div>
</div>
```

### 4.2 响应式卡片排列


**排列规律**：
- 大屏幕：一行3个卡片 (`col-lg-4`)
- 中等屏幕：一行2个卡片 (`col-md-6`)
- 小屏幕：一行1个卡片 (默认`col-12`)

```
大屏幕 (≥992px):  [卡片] [卡片] [卡片]
中屏幕 (≥768px):  [卡片] [卡片]
小屏幕 (<768px):  [卡片]
                 [卡片]
```

## 9. 📱 响应式适配实践


### 9.1 关键断点策略


**Bootstrap断点对应**：
```
- sm (≥576px)：手机横屏
- md (≥768px)：平板竖屏  
- lg (≥992px)：平板横屏/小笔记本
- xl (≥1200px)：桌面显示器
- xxl (≥1400px)：大屏显示器
```

### 9.2 常用响应式模式


**🔸 列数响应式变化**
```html
<!-- 大屏4列，中屏2列，小屏1列 -->
<div class="col-xl-3 col-md-6 col-12">
  <div class="card">商品卡片</div>
</div>
```

**🔸 内容显示隐藏**
```html
<!-- 在小屏幕上隐藏 -->
<div class="d-none d-md-block">
  <p>这段文字在手机上不显示</p>
</div>

<!-- 只在小屏幕上显示 -->
<div class="d-block d-md-none">
  <button class="btn btn-primary">手机菜单</button>
</div>
```

**🔸 间距响应式调整**
```html
<!-- 大屏间距大，小屏间距小 -->
<div class="p-2 p-md-4">
  <h2>标题</h2>
</div>
```

## 10. 📲 移动端优化技巧


### 10.1 触摸友好设计


**按钮大小**：手机上的按钮要够大，方便手指点击
```html
<!-- 移动端友好的按钮 -->
<button class="btn btn-primary btn-lg w-100 mb-3">
  确认下单
</button>
```

**表单优化**：输入框要大，标签要清晰
```html
<div class="mb-3">
  <label class="form-label fs-6">手机号码</label>
  <input type="tel" class="form-control form-control-lg" 
         placeholder="请输入11位手机号">
</div>
```

### 10.2 导航优化


**折叠菜单**：手机上用汉堡菜单节省空间

```html
<nav class="navbar navbar-expand-md">

    <span class="navbar-toggler-icon"></span>
  </button>
  
  <div class="collapse navbar-collapse">
    <!-- 菜单项 -->
  </div>
</nav>
```

**底部导航**：移动应用常用的导航方式

## 11. 📋 核心要点总结


### 11.1 布局核心规律


**🔸 容器→行→列的层次结构**
```html
<div class="container">     ← 容器
  <div class="row">         ← 行
    <div class="col">       ← 列
      内容
    </div>
  </div>
</div>
```

**🔸 响应式优先级**
```
移动优先：先设计手机版 → 再适配大屏
断点继承：小断点的样式会继承到大断点
逐步增强：在大屏幕上添加更多功能
```

**🔸 常用布局模式**
- **导航+内容**：最基础的网站布局
- **侧边栏+主内容**：管理后台常用
- **卡片网格**：产品展示、文章列表
- **三栏布局**：传统网站布局

### 11.2 实用布局技巧


**🔸 等高卡片**：使用 `h-100` 让卡片高度一致
**🔸 垂直居中**：用 `align-items-center` 和 `justify-content-center`
**🔸 间距控制**：用 `g-*` 控制栅格间距，用 `m-*` `p-*` 控制元素间距
**🔸 响应式显示**：用 `d-none d-md-block` 控制不同屏幕的显示

### 11.3 移动端适配要点


```
✅ 导航要可折叠
✅ 按钮要够大 (btn-lg)
✅ 表单要简化
✅ 图片要响应式 (img-fluid)
✅ 文字要清晰 (合适的字号)
✅ 间距要适中 (不要太挤)
```

### 11.4 常见布局陷阱


**❌ 忘记设置viewport**
```html
<!-- 必须要有这行 -->
<meta name="viewport" content="width=device-width, initial-scale=1">
```

**❌ 直接在row里放内容**
```html
<!-- 错误 -->
<div class="row">
  <p>内容</p>  <!-- 应该放在col里 -->
</div>

<!-- 正确 -->
<div class="row">
  <div class="col">
    <p>内容</p>
  </div>
</div>
```

**❌ 过度嵌套容器**
```html
<!-- 避免容器套容器 -->
<div class="container">
  <div class="container">  <!-- 不需要 -->
    内容
  </div>
</div>
```

**核心记忆**：
- 先移动后桌面，响应式更自然
- 容器行列要规范，布局结构不能乱
- 卡片网格最实用，各种场景都能用
- 导航侧栏要会做，后台前台都需要
