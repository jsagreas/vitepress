## 1. 🏛️ 三栏布局（左中右）


### 1.1 什么是三栏布局

**通俗理解**：就像一个房子分成三个房间 - 左边房间、中间大厅、右边房间

```
┌─────────┬──────────────────┬─────────┐
│  左侧   │      中间内容     │  右侧   │
│  边栏   │      主要区域     │  边栏   │
│ (导航)  │     (文章内容)    │ (广告)  │
└─────────┴──────────────────┴─────────┘
```

### 1.2 基础三栏布局


```html
<!-- 经典三栏：左右固定，中间自适应 -->
<div class="container-fluid">
  <div class="row">
    <div class="col-2">
      <h5>左侧导航</h5>
      <ul class="nav flex-column">
        <li class="nav-item"><a href="#" class="nav-link">首页</a></li>
        <li class="nav-item"><a href="#" class="nav-link">产品</a></li>
      </ul>
    </div>
    
    <div class="col-8">
      <h2>主要内容区域</h2>
      <p>这里放置网站的主要内容，比如文章、产品列表等...</p>
    </div>
    
    <div class="col-2">
      <h5>右侧广告</h5>
      <div class="bg-light p-3">广告位</div>
    </div>
  </div>
</div>
```

### 1.3 响应式三栏布局


```html
<!-- 移动端变成单栏，桌面端三栏 -->
<div class="container">
  <div class="row">
    <!-- 移动端占满屏，桌面端占2列 -->
    <div class="col-12 col-lg-2">
      <h5>导航菜单</h5>
      <ul class="nav flex-column">
        <li class="nav-item"><a href="#" class="nav-link">首页</a></li>
      </ul>
    </div>
    
    <!-- 移动端占满屏，桌面端占8列 -->
    <div class="col-12 col-lg-8">
      <h2>文章标题</h2>
      <p>文章正文内容...</p>
    </div>
    
    <!-- 移动端占满屏，桌面端占2列 -->
    <div class="col-12 col-lg-2">
      <h5>相关推荐</h5>
      <div class="list-group">
        <a href="#" class="list-group-item">推荐文章1</a>
      </div>
    </div>
  </div>
</div>
```

## 2. 🏺 圣杯布局实现


### 2.1 圣杯布局是什么

**简单理解**：一种特殊的三栏布局，特点是==中间内容在HTML中写在最前面==，优先加载

```
HTML顺序：中间内容 → 左边栏 → 右边栏
显示效果：左边栏 ← 中间内容 → 右边栏
```

**为什么叫圣杯？** 因为这种布局曾经很难实现，像寻找圣杯一样困难！

### 2.2 Bootstrap圣杯布局


```html
<!-- 关键：使用order类来调整显示顺序 -->
<div class="container">
  <div class="row">
    <!-- 1. 中间内容写在最前面（SEO友好） -->
    <div class="col-12 col-md-8 order-md-2">
      <h1>主要内容</h1>
      <p>重要的文章内容最先加载...</p>
    </div>
    
    <!-- 2. 左侧导航 -->
    <div class="col-12 col-md-2 order-md-1">
      <h5>导航</h5>
      <ul class="nav flex-column">
        <li class="nav-item"><a href="#" class="nav-link">首页</a></li>
      </ul>
    </div>
    
    <!-- 3. 右侧边栏 -->
    <div class="col-12 col-md-2 order-md-3">
      <h5>广告</h5>
      <div class="bg-light p-3">广告内容</div>
    </div>
  </div>
</div>
```

**核心技巧**：
- `order-md-1`：桌面端排第1位
- `order-md-2`：桌面端排第2位  
- `order-md-3`：桌面端排第3位

## 4. 📌 左侧固定右侧自适应


### 4.1 固定像素宽度侧边栏


```html
<!-- 左侧固定200px，右侧自适应 -->
<div class="d-flex">
  <div class="bg-dark text-white" style="width: 200px; min-height: 100vh;">
    <h5 class="p-3">固定侧边栏</h5>
    <ul class="nav flex-column">
      <li class="nav-item">
        <a href="#" class="nav-link text-white">菜单1</a>
      </li>
    </ul>
  </div>
  
  <div class="flex-grow-1 p-4">
    <h2>主要内容区域</h2>
    <p>这个区域会自动填满剩余空间...</p>
  </div>
</div>
```

### 4.2 使用栅格实现固定比例


```html
<!-- 左侧始终占3列，右侧占9列 -->
<div class="container-fluid">
  <div class="row min-vh-100">
    <div class="col-3 bg-light">
      <h5 class="pt-3">固定侧边栏</h5>
      <ul class="nav flex-column">
        <li class="nav-item">
          <a href="#" class="nav-link">Dashboard</a>
        </li>
      </ul>
    </div>
    
    <div class="col-9">
      <div class="p-4">
        <h1>Dashboard</h1>
        <p>主要内容...</p>
      </div>
    </div>
  </div>
</div>
```

**实用技巧**：
- `min-vh-100`：最小高度100%视口高度
- `flex-grow-1`：自动占满剩余空间
- `container-fluid`：占满整个屏幕宽度

## 6. ⚖️ 等高卡片布局


### 6.1 为什么需要等高

**问题**：卡片内容长短不一，导致高度不齐，看起来不美观

```
不等高效果：
┌─────────┐ ┌─────────┐ ┌─────────┐
│ 短内容  │ │ 长内容  │ │ 中等    │
│         │ │ 很多文字│ │ 内容    │
└─────────┘ │ 还有更多│ │         │
            │ 内容... │ └─────────┘
            └─────────┘

等高效果：
┌─────────┐ ┌─────────┐ ┌─────────┐
│ 短内容  │ │ 长内容  │ │ 中等    │
│         │ │ 很多文字│ │ 内容    │
│         │ │ 还有更多│ │         │
│         │ │ 内容... │ │         │
└─────────┘ └─────────┘ └─────────┘
```

### 6.2 使用h-100实现等高


```html
<div class="container">
  <div class="row g-4">
    <div class="col-md-4">
      <div class="card h-100">
        <div class="card-body d-flex flex-column">
          <h5 class="card-title">短标题</h5>
          <p class="card-text">简短内容</p>
          <a href="#" class="btn btn-primary mt-auto">查看详情</a>
        </div>
      </div>
    </div>
    
    <div class="col-md-4">
      <div class="card h-100">
        <div class="card-body d-flex flex-column">
          <h5 class="card-title">很长的标题内容</h5>
          <p class="card-text">这是一段很长的描述内容，可能会占用多行...</p>
          <a href="#" class="btn btn-primary mt-auto">查看详情</a>
        </div>
      </div>
    </div>
    
    <div class="col-md-4">
      <div class="card h-100">
        <div class="card-body d-flex flex-column">
          <h5 class="card-title">中等标题</h5>
          <p class="card-text">中等长度的内容</p>
          <a href="#" class="btn btn-primary mt-auto">查看详情</a>
        </div>
      </div>
    </div>
  </div>
</div>
```

**核心技巧**：
- `h-100`：卡片高度100%
- `d-flex flex-column`：垂直弹性布局
- `mt-auto`：按钮自动推到底部

## 8. ⚠️ 常见布局陷阱避免


### 8.1 忘记container容器


```html
<!-- ❌ 错误：直接使用row -->
<div class="row">
  <div class="col-6">内容</div>
</div>

<!-- ✅ 正确：使用container包裹 -->
<div class="container">
  <div class="row">
    <div class="col-6">内容</div>
  </div>
</div>
```

### 8.2 列数超过12


```html
<!-- ❌ 错误：3+4+6=13列，会换行 -->
<div class="row">
  <div class="col-3">列1</div>
  <div class="col-4">列2</div>
  <div class="col-6">列3</div> <!-- 这列会掉到下一行 -->
</div>

<!-- ✅ 正确：3+4+5=12列 -->
<div class="row">
  <div class="col-3">列1</div>
  <div class="col-4">列2</div>
  <div class="col-5">列3</div>
</div>
```

### 8.3 忘记响应式断点


```html
<!-- ❌ 错误：移动端也是三列，太挤 -->
<div class="row">
  <div class="col-4">列1</div>
  <div class="col-4">列2</div>
  <div class="col-4">列3</div>
</div>

<!-- ✅ 正确：移动端单列，桌面端三列 -->
<div class="row">
  <div class="col-12 col-md-4">列1</div>
  <div class="col-12 col-md-4">列2</div>
  <div class="col-12 col-md-4">列3</div>
</div>
```

### 8.4 高度问题解决


```html
<!-- 解决父容器高度塌陷 -->
<div class="container-fluid min-vh-100">
  <div class="row h-100">
    <div class="col-3 bg-dark text-white">
      <h5>侧边栏</h5>
    </div>
    <div class="col-9">
      <h1>主内容</h1>
    </div>
  </div>
</div>
```

## 9. 📋 核心要点总结


### 9.1 必须掌握的布局模式


```
🔸 三栏布局：左-中-右经典结构，适合网站主体布局
🔸 圣杯布局：中间内容优先加载，SEO友好
🔸 两栏布局：侧边栏+主内容，适合管理后台
🔸 卡片网格：产品展示、文章列表的首选
🔸 响应式适配：移动优先，断点合理设置
```

### 9.2 关键技术要点


**响应式断点规律**：
```
col-12：手机端占满屏
col-md-6：平板端一行两个  
col-lg-4：桌面端一行三个
col-xl-3：大屏端一行四个
```

**等高布局技巧**：
```
父容器：h-100 (高度100%)
内容区：d-flex flex-column (垂直弹性)
按钮区：mt-auto (自动推到底部)
```

**常用组合技巧**：
```
container + row + col：基础栅格结构
g-4：统一间距
order-*：调整显示顺序
d-none d-lg-block：响应式显示隐藏
```

### 9.3 实际应用场景


- **企业官网**：三栏布局展示导航、内容、广告
- **电商平台**：卡片网格展示商品列表
- **管理后台**：左侧固定菜单，右侧主要内容
- **博客网站**：圣杯布局优化SEO，内容优先加载
- **移动应用**：响应式适配不同设备尺寸

**核心记忆口诀**：
- 布局先想容器，栅格不超十二
- 响应式要断点，移动优先原理  
- 等高用h-100，弹性布局排列
- 常见陷阱要避免，调试开发者工具
