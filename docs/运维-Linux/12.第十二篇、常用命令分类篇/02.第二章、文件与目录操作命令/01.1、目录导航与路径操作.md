## 1. 📍 pwd命令 - 显示当前位置


### 1.1 什么是pwd命令

**pwd** 全称是 **Print Working Directory**，简单说就是"显示我现在在哪里"。

```bash
$ pwd
/home/username/documents
```

**通俗理解**：就像你在大商场里迷路了，pwd就是那个"您现在的位置"指示牌。

### 1.2 pwd命令的两种形式

```bash
# 显示当前真实路径（推荐）
$ pwd

# 显示逻辑路径（包含符号链接）
$ pwd -L

# 显示物理路径（解析符号链接）
$ pwd -P
```

**实际场景举例**：
```
假设你进入了一个符号链接目录：
/home/user/项目 -> /var/www/html

pwd -L 显示：/home/user/项目
pwd -P 显示：/var/www/html
```

### 1.3 为什么需要pwd

- **确认位置**：避免在错误目录执行危险操作
- **脚本编程**：获取当前目录路径用于程序逻辑
- **调试排错**：确定命令执行的上下文环境

## 2. 🚀 cd命令 - 目录切换神器


### 2.1 cd命令基本概念

**cd** 全称是 **Change Directory**，就是"换个地方"的意思。

```bash
# 基本语法
cd [目标目录]
```

### 2.2 cd命令的常用方式


**🔸 直接切换到指定目录**
```bash
$ cd /home/username/documents
$ pwd
/home/username/documents
```

**🔸 切换到用户主目录**
```bash
# 以下三种方式都能回到家目录
$ cd
$ cd ~
$ cd $HOME
```

**🔸 返回上一级目录**
```bash
$ cd ..
```

**🔸 返回上次访问的目录**
```bash
$ cd -
# 相当于"返回键"，在两个目录间来回切换
```

### 2.3 cd命令的实际应用场景


**场景示例**：
```
当前位置：/home/user/projects/website/css
需要编辑：/home/user/projects/website/js/main.js

传统做法：cd /home/user/projects/website/js
智能做法：cd ../js  (相对路径更简洁)
```

**常见用法对比**：

| 目标 | 命令 | 说明 |
|------|------|------|
| **回到家目录** | `cd` | 最快捷的方式 |
| **上一级目录** | `cd ..` | 向上返回一层 |
| **上次目录** | `cd -` | 在两个目录间切换 |
| **根目录** | `cd /` | 系统最顶层 |

## 3. 🗺️ 路径系统详解


### 3.1 什么是路径

**路径** 就是文件或目录在系统中的"家庭住址"，告诉系统去哪里找它们。

### 3.2 绝对路径 vs 相对路径


**📍 绝对路径**
```
定义：从根目录(/)开始的完整路径
特点：无论在哪里都指向同一个位置
示例：/home/user/documents/report.txt

类比：就像写信时的完整地址
"中国北京市朝阳区XX路XX号XX室"
```

**📍 相对路径**
```
定义：相对于当前位置的路径
特点：会根据当前位置变化而变化
示例：../images/logo.png

类比：就像说"隔壁老王家"
具体指哪里要看你现在在哪里
```

### 3.3 路径选择的实用建议


**🟢 什么时候用绝对路径**
- 脚本编程中的关键文件
- 系统配置文件
- 跨目录层级较深的操作

**🟡 什么时候用相对路径**
- 同一项目内的文件操作
- 临时的快速操作
- 目录结构相对固定的场景

**路径示例对比**：
```
文件结构：
/home/user/
├── projects/
│   ├── website/
│   │   ├── css/
│   │   └── js/
│   └── backup/
└── documents/

当前在：/home/user/projects/website/css

访问js目录：
绝对路径：cd /home/user/projects/website/js
相对路径：cd ../js  ← 更简洁！
```

## 4. 🔤 特殊目录符号


### 4.1 核心特殊符号详解


**🏠 ~ (波浪号) - 用户主目录**
```bash
# 以下都表示用户主目录
cd ~
cd ~/documents  # 等同于 cd /home/username/documents
```
**通俗理解**：~ 就是你的"家"，无论走到哪里，`cd ~` 都能回家。

**📁 . (单点) - 当前目录**
```bash
# 表示当前目录
ls .         # 列出当前目录内容
cp file1 .   # 复制文件到当前目录
```
**通俗理解**：. 就是"这里"，表示你现在站的地方。

**⬆️ .. (双点) - 父目录**
```bash
cd ..           # 上一级目录
cd ../..        # 上两级目录
cd ../../home   # 上两级再进入home
```
**通俗理解**：.. 就是"楼上"，每个 .. 向上一层。

### 4.2 特殊符号的组合使用


**🔸 路径拼接示例**
```
当前位置：/home/user/projects/website/css

cd ../js          → /home/user/projects/website/js
cd ../../backup   → /home/user/projects/backup  
cd ~/documents    → /home/user/documents
```

**🔸 实用技巧**
```bash
# 快速在兄弟目录间切换
cd ../images  # 从css切换到images
cd ../js      # 从images切换到js
cd -          # 返回images目录
```

## 5. ⚡ 高级导航技巧


### 5.1 cd - 命令详解


**返回上次目录的妙用**：
```bash
$ cd /var/log
$ cd /home/user/documents
$ cd -                    # 返回 /var/log
$ cd -                    # 返回 /home/user/documents
```

**实际应用场景**：
```
场景：在两个相距很远的目录间频繁切换
目录A：/var/www/html/project/frontend/src/components
目录B：/home/user/backup/project/config

传统做法：每次都要输入完整路径
高效做法：用 cd - 在两个目录间跳转
```

### 5.2 路径记忆与快速访问


**🔸 使用历史命令**
```bash
# 查看cd命令历史
history | grep "cd "

# 使用!!重复上次命令
cd /very/long/path/to/directory
!!  # 重复执行上次的cd命令
```

**🔸 创建别名快捷方式**
```bash
# 在 ~/.bashrc 中添加
alias proj='cd /home/user/projects'
alias web='cd /var/www/html'
alias logs='cd /var/log'

# 使用方法
$ proj    # 直接跳转到projects目录
$ web     # 直接跳转到web目录
```

## 6. 📚 目录栈操作


### 6.1 什么是目录栈

**目录栈** 就像一个"书签系统"，帮你记住访问过的目录，方便来回跳转。

```
想象场景：
你在看书时，用书签标记了几个重要页面
目录栈就是这样的"书签系统"
```

### 6.2 pushd - 压入目录栈


**基本用法**：
```bash
# 当前在 /home/user
$ pushd /var/log          # 跳转到 /var/log 并记住 /home/user
/var/log ~

$ pushd /etc              # 跳转到 /etc 并记住之前的位置  
/etc /var/log ~

$ dirs                    # 查看目录栈
/etc /var/log ~
```

### 6.3 popd - 弹出目录栈


**基本用法**：
```bash
$ popd                    # 返回到栈顶的上一个目录
/var/log ~

$ popd                    # 再次返回
~
```

### 6.4 目录栈的实际应用


**适用场景**：
- **多目录项目开发**：在源码、测试、文档目录间切换
- **系统管理**：在配置、日志、备份目录间跳转
- **复杂的文件整理**：记住多个重要位置

**实用技巧**：
```bash
# 查看目录栈并带编号
dirs -v

# 切换到栈中指定位置（从0开始）
pushd +2    # 切换到栈中第3个目录
```

## 7. ⚙️ 环境变量配置


### 7.1 CDPATH环境变量详解


**CDPATH** 就像给cd命令设置了"搜索路径"，让你在任何地方都能快速跳转到常用目录。

### 7.2 CDPATH的工作原理


```bash
# 设置CDPATH
export CDPATH="/home/user:/var:/usr/local"

# 现在无论在哪里，都可以直接cd到这些路径下的目录
$ cd projects     # 自动找到 /home/user/projects
$ cd log          # 自动找到 /var/log  
$ cd bin          # 自动找到 /usr/local/bin
```

**工作流程**：
```
1. cd命令首先在当前目录查找
2. 如果没找到，依次在CDPATH路径中查找
3. 找到第一个匹配的就跳转过去
```

### 7.3 CDPATH配置最佳实践


**🔸 在 ~/.bashrc 中配置**
```bash
# 添加常用的父级目录
export CDPATH=".:~:/home/user/projects:/var:/usr/local"

# 使配置生效
source ~/.bashrc
```

**🔸 实际应用示例**
```bash
# 配置前
cd /home/user/projects/website    # 需要输入完整路径

# 配置后  
cd website                        # 直接输入目录名即可
```

### 7.4 注意事项


::: warning 使用警告
- CDPATH会改变cd的默认行为
- 可能导致脚本兼容性问题
- 建议只在个人环境中使用
:::

## 8. 🔍 路径补全与通配符


### 8.1 Tab键自动补全


**基础补全功能**：
```bash
$ cd /ho<Tab>           # 自动补全为 /home/
$ cd /home/us<Tab>      # 自动补全为 /home/user/  
$ cd proj<Tab>          # 补全项目目录名
```

**智能补全技巧**：
- **单击Tab**：如果唯一匹配，直接补全
- **双击Tab**：显示所有可能的匹配项

### 8.2 通配符在路径中的应用


**🔸 * 通配符 - 匹配任意字符**
```bash
$ cd /home/*/documents     # 匹配任意用户的documents目录
$ ls proj*                 # 列出所有以proj开头的目录
```

**🔸 ? 通配符 - 匹配单个字符**
```bash
$ cd log?                  # 匹配log1, loga等4字符目录
$ ls file?.txt             # 匹配file1.txt, filea.txt等
```

**🔸 [] 通配符 - 匹配指定范围**
```bash
$ cd dir[123]              # 匹配dir1, dir2, dir3
$ ls [abc]*                # 匹配以a,b,c开头的文件/目录
```

### 8.3 高级补全配置


**bash-completion增强**：
```bash
# Ubuntu/Debian安装
sudo apt install bash-completion

# 功能增强后支持：
$ cd proj<Tab><Tab>        # 显示所有项目目录
$ cd --<Tab>               # 显示cd命令的所有选项
```

### 8.4 路径补全的实用技巧


**🟢 效率提升技巧**
1. **部分输入+Tab**：输入几个字母就按Tab
2. **利用唯一性**：输入能唯一标识的最少字符
3. **双Tab查看选项**：不确定时双击Tab查看所有可能

**路径补全示例**：
```
目录结构：
projects/
├── project-web/
├── project-api/
└── project-mobile/

输入：cd proj<Tab>          # 显示3个选项
输入：cd project-w<Tab>     # 直接补全为project-web/
```

## 9. 📋 核心要点总结


### 9.1 必须掌握的基础命令


```bash
🔸 pwd              # 显示当前位置 - 迷路时的指南针
🔸 cd               # 切换目录 - 移动的基本方式  
🔸 cd ~             # 回到家目录 - 一键回家
🔸 cd ..            # 上一级目录 - 向上爬楼梯
🔸 cd -             # 返回上次目录 - 来回切换神器
```

### 9.2 路径概念理解


**🔹 记忆要点**
```
绝对路径：从/开头，完整地址，全球定位
相对路径：不从/开头，相对位置，本地定位

特殊符号记忆：
~  = 家(home)
.  = 这里(here) 
.. = 楼上(up)
-  = 上次(last)
```

### 9.3 实用技能总结


| 技能 | 命令示例 | 适用场景 | 🔥热度 |
|------|---------|----------|--------|
| **快速回家** | `cd ~` | 任何时候想回主目录 | 🔥🔥🔥🔥🔥 |
| **上级目录** | `cd ..` | 向上返回时 | 🔥🔥🔥🔥🔥 |
| **来回切换** | `cd -` | 两个目录频繁切换 | 🔥🔥🔥🔥 |
| **路径补全** | `<Tab>` | 减少输入，避免错误 | 🔥🔥🔥🔥 |
| **目录栈** | `pushd/popd` | 多目录记忆跳转 | 🔥🔥 |

### 9.4 新手常见错误


::: danger 避免这些错误
❌ **cd后面有空格**：`cd /home/user` ✅ 不是 `cd  /home/user`
❌ **路径中的空格**：用引号包围 `cd "My Documents"`
❌ **相对路径混乱**：记住 . 是当前，.. 是上级
❌ **符号链接困惑**：用 `pwd -P` 查看真实路径
:::

### 9.5 效率提升建议


**🟢 日常使用建议**
1. **多用Tab补全** - 减少拼写错误
2. **善用cd -** - 在常用目录间快速切换
3. **设置有用的alias** - 为长路径创建快捷方式
4. **熟记特殊符号** - ~、.、.. 是最常用的

**🟡 进阶使用建议**  
1. **配置CDPATH** - 为常用目录设置搜索路径
2. **使用目录栈** - pushd/popd管理多个工作目录
3. **掌握通配符** - 批量操作时很有用

**💡 学习顺序建议**
```
第1周：掌握 pwd, cd, cd ~, cd .., cd -
第2周：理解绝对路径和相对路径概念  
第3周：熟练使用Tab补全和基本通配符
第4周：学习目录栈和CDPATH配置
```

**核心记忆**：
- pwd = 我在哪里？
- cd = 去哪里？  
- Tab = 帮我补全
- 路径 = 地址系统
