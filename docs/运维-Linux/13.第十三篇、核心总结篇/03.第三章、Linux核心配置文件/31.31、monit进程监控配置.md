## 1. 🎯 monit进程监控概述


### 1.1 什么是monit


**简单理解**：monit就像一个24小时不休息的保安，时刻监控着系统中的各种服务和资源。

> 💡 **通俗比喻**：
> 如果把服务器比作一栋大楼，monit就是那个巡逻的保安，定时检查每个房间（进程），发现问题立即报告并尝试解决

**monit的核心功能**：
- **进程监控**：监控服务进程的运行状态
- **资源监控**：监控CPU、内存、磁盘使用情况
- **自动恢复**：服务异常时自动重启
- **告警通知**：问题发生时及时发送通知
- **Web管理**：提供直观的Web管理界面

### 1.2 为什么需要monit


**传统手动监控的问题**：
```
问题场景：
凌晨3点，网站突然无法访问
→ 管理员在睡觉，无人发现
→ 用户投诉才知道问题
→ 紧急起床处理，业务已受损

使用monit后：
服务异常 → monit立即发现 → 自动重启服务 → 发送告警邮件
→ 问题在几分钟内自动解决
```

**monit的优势**：
- **7×24小时监控**：全天候自动监控
- **快速响应**：秒级发现问题
- **自动处理**：无需人工干预即可恢复服务
- **轻量高效**：资源占用极少
- **配置简单**：易于理解和配置

### 1.3 monit监控架构


```
monit监控体系架构：

┌─────────────────────┐
│    Web Interface    │ ← 管理界面
├─────────────────────┤
│   monit 守护进程     │ ← 核心监控引擎
├─────────────────────┤
│   监控对象管理       │
├─────┬─────┬─────────┤
│进程  │资源 │文件系统  │ ← 监控目标
│监控  │监控 │监控     │
└─────┴─────┴─────────┘
       ↓
┌─────────────────────┐
│   告警通知系统       │ ← 邮件/短信通知
└─────────────────────┘
```

## 3. ⚙️ 主配置文件详解


### 3.1 配置文件位置


**不同系统的配置文件路径**：
```
CentOS/RHEL：
主配置文件：/etc/monitrc
配置目录：/etc/monit.d/

Ubuntu/Debian：
主配置文件：/etc/monit/monitrc  
配置目录：/etc/monit/conf.d/
```

### 3.2 配置文件权限


> ⚠️ **重要安全要求**：
> monit配置文件必须设置严格的权限，否则服务无法启动

```bash
# 设置正确的权限
chmod 600 /etc/monitrc
chown root:root /etc/monitrc

# 如果权限不正确，启动时会报错：
# 'monitrc' file has wrong permissions
```

### 3.3 主配置文件结构


**基础配置结构**：
```bash
# /etc/monitrc 基本结构

# 全局设置

set daemon 30                    # 检查间隔30秒
set log syslog                   # 日志输出到syslog

# Web界面配置

set httpd port 2812 and
    use address localhost        # 只允许本地访问
    allow localhost              # 允许本地连接
    allow admin:monit            # 设置用户名密码

# 邮件服务器配置

set mailserver smtp.example.com port 587
    username "alert@example.com"
    password "your_password"
    using tls

# 全局邮件通知设置

set alert admin@example.com

# 引入其他配置文件

include /etc/monit.d/*
```

**监控间隔设置**：

## 4. 🔍 进程监控配置


### 4.1 进程监控基本语法


**监控配置的基本结构**：
```bash
check process <service_name> with pidfile <pid_file>
    start program = "<start_command>"
    stop program = "<stop_command>"
    if failed port <port> then restart
    if cpu > 80% for 2 cycles then restart
    if memory > 500 MB for 2 cycles then restart
```

> 📖 **语法要素解释**：
> - **check process**：定义进程监控
> - **with pidfile**：指定进程PID文件位置
> - **start/stop program**：定义启动和停止命令
> - **if条件**：定义监控规则和响应动作

### 4.2 常见服务监控配置


**监控Nginx服务**：
```bash
# /etc/monit.d/nginx
check process nginx with pidfile /var/run/nginx.pid
    group www-data
    start program = "/usr/sbin/service nginx start"
    stop program = "/usr/sbin/service nginx stop"
    if failed port 80 protocol http then restart
    if cpu > 60% for 2 cycles then alert
    if memory > 100 MB for 2 cycles then restart
    if 3 restarts within 5 cycles then timeout
```

**监控Apache服务**：
```bash
# /etc/monit.d/apache
check process apache with pidfile /var/run/httpd/httpd.pid
    start program = "/usr/sbin/service httpd start"
    stop program = "/usr/sbin/service httpd stop"
    if failed port 80 protocol http then restart
    if failed port 443 protocol https then restart
    if cpu > 80% for 3 cycles then restart
    if memory > 200 MB then restart
```

**监控MySQL数据库**：
```bash
# /etc/monit.d/mysql
check process mysql with pidfile /var/run/mysqld/mysqld.pid
    group database
    start program = "/usr/sbin/service mysqld start"
    stop program = "/usr/sbin/service mysqld stop"
    if failed port 3306 protocol mysql then restart
    if cpu > 80% for 2 cycles then restart
    if memory > 500 MB for 2 cycles then restart
    if failed unixsocket /var/lib/mysql/mysql.sock then restart
```

### 4.3 进程监控条件详解


**端口检查条件**：
```bash
# HTTP端口检查
if failed port 80 protocol http then restart

# HTTPS端口检查  
if failed port 443 protocol https then restart

# TCP端口检查
if failed port 22 protocol tcp then restart

# 带超时的端口检查
if failed port 80 protocol http with timeout 15 seconds then restart
```

**资源使用条件**：
```bash
# CPU使用率检查
if cpu > 80% for 2 cycles then restart     # CPU超过80%持续2个周期
if cpu > 90% for 1 cycle then alert       # CPU超过90%立即告警

# 内存使用检查
if memory > 500 MB then restart           # 内存使用超过500MB
if memory > 80% then restart              # 内存使用率超过80%

# 负载平均值检查
if loadavg (1min) > 4 then restart        # 1分钟负载超过4
if loadavg (5min) > 2 then restart        # 5分钟负载超过2
```

### 4.4 自定义监控脚本


**使用自定义脚本监控**：
```bash
# /etc/monit.d/custom-app
check program custom-check with path "/usr/local/bin/check-app.sh"
    if status != 0 then restart
    every 3 cycles                         # 每3个周期检查一次

# 自定义检查脚本示例
#!/bin/bash
# /usr/local/bin/check-app.sh
ps aux | grep "my-application" | grep -v grep
if [ $? -eq 0 ]; then
    exit 0      # 进程存在
else  
    exit 1      # 进程不存在
fi
```

## 5. 📊 系统资源监控


### 5.1 系统负载监控


**负载平均值监控**：
```bash
# 监控系统负载
check system $HOST
    if loadavg (1min) > 4 then alert
    if loadavg (5min) > 2 then alert  
    if loadavg (15min) > 1 then alert
    if cpu usage > 95% for 10 cycles then alert
    if memory usage > 75% then alert
```

> 📖 **负载平均值说明**：
> - **1分钟负载**：反映当前系统繁忙程度
> - **5分钟负载**：反映近期系统负载趋势  
> - **15分钟负载**：反映系统长期负载情况
> - **理想值**：单核系统负载应小于1，多核按核心数倍数计算

### 5.2 CPU使用率监控


**CPU监控配置**：
```bash
check system localhost
    if cpu usage (user) > 70% for 2 cycles then alert
    if cpu usage (system) > 30% for 2 cycles then alert  
    if cpu usage (wait) > 20% for 2 cycles then alert
    if cpu usage > 80% for 3 cycles then alert
```

**CPU使用类型说明**：
| CPU类型 | 含义 | 正常范围 | 异常情况 |
|---------|------|----------|----------|
| **user** | 用户进程CPU使用 | < 70% | 应用程序负载过重 |
| **system** | 系统内核CPU使用 | < 30% | 系统调用频繁或I/O瓶颈 |
| **wait** | I/O等待时间 | < 20% | 磁盘或网络I/O瓶颈 |

### 5.3 内存使用监控


**内存监控配置**：
```bash
check system localhost  
    if memory usage > 80% then alert
    if swap usage > 25% then alert
    
# 详细内存监控
check system $HOST
    if memory usage > 75% for 2 cycles then alert
    if memory usage > 85% then exec "/usr/local/bin/cleanup-cache.sh"
    if swap usage > 50% for 3 cycles then alert
```

**内存清理脚本示例**：
```bash
#!/bin/bash
# /usr/local/bin/cleanup-cache.sh
echo "开始清理系统缓存..."
sync && echo 3 > /proc/sys/vm/drop_caches
echo "缓存清理完成"
```

## 6. 💾 文件系统监控


### 6.1 磁盘空间监控


**基本磁盘空间监控**：
```bash
# 监控根分区
check filesystem rootfs with path /
    if space usage > 80% then alert
    if space usage > 90% then exec "/usr/local/bin/cleanup-logs.sh"
    if inode usage > 80% then alert

# 监控数据分区
check filesystem datafs with path /data
    if space usage > 85% then alert
    if space usage > 95% for 2 cycles then exec "/usr/local/bin/emergency-cleanup.sh"
```

**多分区监控配置**：
```bash
# 监控主要系统分区
check filesystem rootfs with path /
    if space usage > 80% then alert
    if space usage > 90% then exec "/usr/local/bin/cleanup-root.sh"
    
check filesystem varfs with path /var  
    if space usage > 75% then alert
    if space usage > 85% then exec "/usr/local/bin/cleanup-var-logs.sh"
    
check filesystem tmpfs with path /tmp
    if space usage > 90% then exec "/bin/rm -rf /tmp/*"
```

### 6.2 文件权限监控


**重要文件权限监控**：
```bash
# 监控重要配置文件
check file passwd with path /etc/passwd
    if failed checksum then alert
    if failed permission 644 then alert
    if failed uid root then alert
    if failed gid root then alert
    
# 监控SSH配置文件
check file sshd_config with path /etc/ssh/sshd_config  
    if failed checksum then exec "/usr/sbin/service sshd restart"
    if failed permission 600 then alert
    if changed timestamp then alert
```

### 6.3 日志文件监控


**日志文件大小监控**：
```bash
# 监控系统日志大小
check file messages with path /var/log/messages
    if size > 100 MB then exec "/usr/local/bin/rotate-log.sh /var/log/messages"
    
# 监控应用日志
check file app-log with path /var/log/application.log
    if size > 500 MB then exec "/usr/local/bin/archive-log.sh"
    if not exist then alert
```

**日志轮转脚本示例**：
```bash
#!/bin/bash
# /usr/local/bin/rotate-log.sh
LOGFILE=$1
if [ -f "$LOGFILE" ]; then
    mv "$LOGFILE" "${LOGFILE}.$(date +%Y%m%d)"
    touch "$LOGFILE"
    chmod 644 "$LOGFILE"
    echo "日志文件 $LOGFILE 已轮转"
fi
```

## 8. 📧 告警通知配置


### 8.1 邮件通知设置


**基本邮件配置**：
```bash
# 邮件服务器设置
set mailserver smtp.company.com port 587
    username "monit@company.com"
    password "your-password"  
    using tls
    
# 默认邮件格式设置
set mail-format {
    from: monit@company.com
    subject: [MONIT] $HOST - $SERVICE $EVENT
    message: 
服务器: $HOST
服务: $SERVICE  
事件: $EVENT
时间: $DATE
描述: $DESCRIPTION

请及时处理！
}
```

**多收件人配置**：
```bash
# 全局告警邮件
set alert admin@company.com
set alert ops-team@company.com only on { timeout, nonexist }
set alert dba@company.com only on { resource, timeout }

# 针对特定服务的告警
check process nginx with pidfile /var/run/nginx.pid
    alert webmaster@company.com on { timeout, resource }
    alert ops@company.com
```

### 8.2 告警过滤设置


**告警事件过滤**：
```bash
# 只在特定事件时发送邮件
set alert admin@company.com only on { 
    nonexist,      # 服务不存在
    timeout,       # 服务超时  
    resource,      # 资源超限
    icmp,          # 网络连接问题
    connection     # 连接失败
}

# 排除特定事件
set alert ops@company.com not on { pid, ppid }
```

### 8.3 告警频率控制


**防止告警轰炸**：
```bash
# 在服务配置中添加告警间隔
check process apache with pidfile /var/run/httpd.pid
    start program = "/usr/sbin/service httpd start"  
    stop program = "/usr/sbin/service httpd stop"
    if failed port 80 then restart
    if 3 restarts within 5 cycles then timeout
    alert admin@company.com with reminder on 10 cycles  # 每10个周期提醒一次
```

## 9. 🖥️ Web界面配置


### 9.1 Web界面基本设置


**启用Web界面**：
```bash
# 基本Web界面配置
set httpd port 2812 and
    use address 0.0.0.0          # 允许所有IP访问
    allow localhost              # 允许本地访问
```

> ⚠️ **安全提醒**：
> 生产环境中应该限制Web界面的访问IP范围，避免使用0.0.0.0

### 9.2 用户权限管理


**多用户权限设置**：
```bash
set httpd port 2812 and
    use address localhost
    allow localhost
    allow admin:admin123 read-write    # 管理员权限
    allow viewer:view123 read-only     # 只读权限  
    allow ops:ops123                   # 普通用户权限
```

**权限级别说明**：
| 权限类型 | 说明 | 可执行操作 |
|----------|------|------------|
| **read-write** | 完全权限 | 查看、启动、停止、重启服务 |
| **read-only** | 只读权限 | 只能查看监控状态 |
| **默认权限** | 普通用户 | 查看和控制部分服务 |

### 9.3 SSL/TLS加密设置


**HTTPS访问配置**：
```bash
set httpd port 2812 and
    use address localhost
    ssl enable
    pemfile /etc/ssl/private/monit.pem
    allow localhost
    allow admin:monit
```

**生成SSL证书**：
```bash
# 生成自签名证书
openssl req -new -x509 -days 365 -nodes -out /etc/ssl/private/monit.pem -keyout /etc/ssl/private/monit.pem

# 设置权限
chmod 600 /etc/ssl/private/monit.pem
chown root:root /etc/ssl/private/monit.pem
```

## 10. 🚀 实践案例与优化


### 10.1 Web服务器完整监控方案


**完整的Web服务器监控配置**：
```bash
# /etc/monit.d/web-server
# 监控Nginx主进程
check process nginx with pidfile /var/run/nginx.pid
    group www
    start program = "/usr/sbin/service nginx start"  
    stop program = "/usr/sbin/service nginx stop"
    if failed port 80 protocol http then restart
    if failed port 443 protocol https then restart
    if cpu > 80% for 2 cycles then alert
    if memory > 200 MB for 2 cycles then restart
    if 3 restarts within 5 cycles then timeout

# 监控PHP-FPM进程  
check process php-fpm with pidfile /var/run/php-fpm/php-fpm.pid
    group www
    start program = "/usr/sbin/service php-fpm start"
    stop program = "/usr/sbin/service php-fpm stop"  
    if failed unixsocket /var/run/php-fpm/www.sock then restart
    if cpu > 60% for 2 cycles then alert
    if memory > 300 MB then restart

# 监控网站可用性
check host website with address example.com
    if failed port 80 protocol http 
       request "/" 
       with content = "Welcome"
       with timeout 10 seconds
       for 2 cycles then alert
```

```bash
# /etc/monit.d/database
```

### 10.3 性能优化建议


**监控频率优化**：

## 11. 📋 核心要点总结


### 11.1 必须掌握的核心概念


```
🔸 monit本质：轻量级的系统监控工具，提供进程、资源、文件系统监控
🔸 配置结构：主配置文件+分模块配置文件，权限必须设为600
🔸 监控类型：进程监控、系统资源监控、文件系统监控、网络服务监控
🔸 告警机制：邮件通知、自动重启、自定义脚本执行
🔸 Web界面：提供直观的监控管理界面，支持用户权限控制
🔸 自动化：7×24小时自动监控，异常时自动处理和通知
```

### 11.2 关键理解要点


**🔹 监控策略的设计思路**
```
监控层次：
- 系统层监控：CPU、内存、磁盘、负载
- 服务层监控：进程状态、端口连接、响应时间
- 应用层监控：自定义检查、业务逻辑验证

响应策略：
- 预警阶段：资源使用达到阈值时发送告警
- 自动恢复：服务异常时自动重启
- 紧急处置：多次重启失败后停止监控并通知
```

**🔹 配置文件的最佳实践**
```
文件组织：
- 主配置文件：全局设置、Web界面、邮件配置
- 分模块配置：按服务类型分别配置
- 命名规范：使用有意义的服务名和文件名

安全考虑：
- 配置文件权限必须设为600
- Web界面访问限制IP范围
- 邮件密码不要明文存储
```

**🔹 告警通知的合理配置**
```
告警分级：
- 紧急告警：服务完全不可用
- 警告信息：资源使用率偏高
- 信息通知：服务重启成功

频率控制：
- 避免告警轰炸
- 设置合理的检查周期
- 使用告警间隔和提醒机制
```

### 11.3 实际应用价值


**🎯 核心应用场景**
- **Web服务监控**：监控Nginx、Apache等Web服务器状态
- **数据库监控**：监控MySQL、PostgreSQL等数据库服务
- **系统资源监控**：监控服务器CPU、内存、磁盘使用情况
- **自动故障恢复**：服务异常时自动重启，减少人工干预

**🔧 运维效率提升**
- **24小时监控**：无需人工值守，系统自动监控
- **快速响应**：秒级发现问题，分钟级自动恢复
- **统一管理**：Web界面集中管理所有监控项
- **历史记录**：完整的监控日志便于问题分析

### 11.4 配置优化指南


**性能优化**：
```
检查频率优化：
- 关键服务：30秒或更短
- 普通服务：1-2分钟  
- 非关键服务：5分钟或更长

阈值设置优化：
- 根据服务器配置调整资源阈值
- 根据业务特点设置告警条件
- 避免过于敏感的告警设置
```

**监控覆盖**：
```
必须监控的项目：
✅ 关键服务进程状态
✅ 系统资源使用情况
✅ 重要端口连接状态
✅ 磁盘空间使用率

可选监控的项目：
- 应用程序特定指标
- 自定义业务逻辑检查
- 第三方服务依赖检查
```

### 11.5 故障处理流程


**标准处理流程**：
```
问题发现：
monit检测到异常 → 记录日志 → 尝试自动恢复

自动恢复：
执行预定义的恢复操作 → 验证恢复效果 → 发送通知

人工介入：
自动恢复失败 → 发送紧急告警 → 运维人员介入处理
```

**核心记忆**：
- monit是系统守护神，24小时不间断监控
- 配置简单功能强，进程资源全覆盖  
- 自动恢复加告警，故障处理更高效
- Web管理很直观，运维工作更轻松
