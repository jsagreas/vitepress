## 1. 🗂️ usr/sbin目录概述


### 1.1 usr/sbin是什么

🎯 **简单理解**：usr/sbin就像系统的"高级工具箱"，存放给管理员用的专业工具

```
生活中的类比：
家用工具箱 → 普通用户的命令（/bin, /usr/bin）
专业维修工具 → 系统管理员的命令（/sbin, /usr/sbin）

usr/sbin的特点：
- 需要管理员权限才能使用
- 主要用于系统管理和维护
- 功能复杂，影响范围大
```

### 1.2 目录结构与权限特点

**📋 usr/sbin的基本特征**

| 特征 | **说明** | **示例** |
|------|---------|---------|
| 🔸 **权限要求** | `通常需要root权限` | `大部分命令需要sudo` |
| 🔸 **用户群体** | `系统管理员专用` | `运维、DBA、网管` |
| 🔸 **功能范围** | `系统级操作` | `网络、用户、服务管理` |
| 🔸 **影响范围** | `全系统影响` | `配置变更影响所有用户` |

**🔍 目录内容概览**
```bash
ls -la /usr/sbin | head -10
# 可以看到各种系统管理工具

du -sh /usr/sbin
# 查看目录大小，通常有几百MB
```

### 1.3 与其他sbin目录的区别

**🔄 系统中的sbin目录对比**

```
/sbin vs /usr/sbin 的区别：

/sbin（系统根目录的sbin）：
- 系统启动必需的工具
- 在根文件系统中，单用户模式可用
- 如：init, mount, fsck

/usr/sbin（用户区域的sbin）：
- 系统管理的扩展工具
- 需要/usr文件系统挂载后才可用
- 如：nginx, apache2, mysqld
```

**💡 实际应用场景**
```
系统启动流程中的工具分布：

启动阶段1（单用户模式）：
使用 /sbin 中的工具
- /sbin/init（系统初始化）
- /sbin/mount（挂载文件系统）

启动阶段2（多用户模式）：
使用 /usr/sbin 中的工具  
- /usr/sbin/sshd（SSH服务）
- /usr/sbin/nginx（Web服务）
```

## 2. ⚙️ 系统管理工具集


### 2.1 系统信息管理工具

🔍 **系统状态查看与管理**

**核心系统信息工具**
- **`lscpu`** - 查看CPU详细信息
- **`lsblk`** - 显示块设备信息
- **`lshw`** - 显示硬件详细信息
- **`dmidecode`** - 读取DMI（Desktop Management Interface）信息
- **`hwinfo`** - 全面的硬件信息探测

```bash
# CPU信息详细查看
lscpu
# 显示：架构、CPU核心数、缓存大小、频率等

# 存储设备结构查看
lsblk -f
# 显示：磁盘分区、文件系统、挂载点

# 内存信息查看
dmidecode -t memory | grep -i size
# 显示：内存条大小、频率、类型
```

**🎯 实际应用场景**
```
性能优化场景：
1. 使用lscpu了解CPU架构，优化编译参数
2. 使用lsblk查看存储布局，规划数据分布
3. 使用dmidecode确认硬件兼容性

故障排查场景：
1. 硬件故障时使用lshw全面检查
2. 内存问题时使用dmidecode查看内存规格
3. 存储问题时使用lsblk查看分区状态
```

### 2.2 进程与资源管理工具

⚡ **系统资源监控与控制**

**关键资源管理命令**
- **`htop`** / **`atop`** - 增强版进程监控
- **`iotop`** - I/O使用情况监控
- **`nethogs`** - 网络流量按进程统计
- **`powertop`** - 电源管理和功耗分析
- **`sysstat`** - 系统性能统计工具集

```bash
# I/O性能监控
iotop -o
# 只显示有I/O活动的进程

# 网络使用按进程分析
nethogs eth0
# 查看每个进程的网络使用情况

# 系统资源历史统计
sar -u 5 3
# 每5秒采样一次，共3次，显示CPU使用率
```

**📊 性能监控策略**
```
分层监控方法：

第1层 - 系统整体：
使用 htop 查看整体资源使用
使用 sar 查看历史趋势

第2层 - 专项分析：
使用 iotop 分析磁盘瓶颈
使用 nethogs 分析网络占用

第3层 - 深度诊断：
使用 atop 进行详细分析
使用 powertop 优化功耗
```

### 2.3 系统配置管理工具

🔧 **系统参数调优工具**

**系统调优核心工具**
- **`sysctl`** - 内核参数管理

- **`systemd-analyze`** - 系统启动分析

## 4. 👥 用户管理工具


### 4.1 用户账户管理

👤 **用户创建、修改、删除**

**用户管理核心命令**
- **`useradd`** - 创建用户账户
- **`usermod`** - 修改用户属性
- **`userdel`** - 删除用户账户
- **`passwd`** - 密码管理
- **`chage`** - 密码策略管理

```bash
# 创建用户并设置属性
useradd -m -s /bin/bash -G wheel username
# -m: 创建家目录, -s: 指定shell, -G: 附加组

# 修改用户属性
usermod -aG docker username
# 将用户添加到docker组

# 设置密码过期策略
chage -M 90 -W 7 username
# -M: 密码最大有效期90天, -W: 提前7天警告
```

**🔐 密码安全策略**
```
企业级密码策略：

密码复杂度要求：
- 最少8位字符
- 包含大小写字母、数字、特殊字符
- 不能包含用户名或常见字典词

密码生命周期管理：
- 最大有效期：90天
- 最小更改间隔：1天
- 提前警告期：7天
- 失效后宽限期：7天

账户锁定策略：
- 连续失败3次锁定账户
- 锁定时间：30分钟
- 管理员账户特殊处理
```

### 4.2 组权限管理

👥 **用户组与权限控制**

**组管理工具集**
- **`groupadd`** - 创建用户组
- **`groupmod`** - 修改组属性
- **`groupdel`** - 删除用户组
- **`gpasswd`** - 组密码和成员管理
- **`newgrp`** - 临时切换用户组

```bash
# 创建系统组和普通组
groupadd -r system_group    # 系统组（GID < 1000）
groupadd normal_group       # 普通组（GID >= 1000）

# 组成员管理
gpasswd -a username groupname    # 添加用户到组
gpasswd -d username groupname    # 从组中删除用户
```

**⚖️ 权限管理最佳实践**
```
最小权限原则：
1. 用户只获得完成工作所需的最小权限
2. 定期审查和回收不必要的权限
3. 使用组权限而不是直接给用户权限

角色分离原则：
开发人员组：读取代码、部署测试环境
运维人员组：系统管理、生产环境部署
安全人员组：日志审计、权限管理
```

### 4.3 访问控制与审计

🔍 **登录控制与用户行为审计**

**访问控制工具**
- **`pam_tally2`** - 登录失败计数和锁定
- **`last`** - 用户登录历史
- **`w`** / **`who`** - 当前登录用户
- **`sudo`** - 权限提升管理
- **`su`** - 用户身份切换

```bash
# 查看登录失败次数
pam_tally2 --user username

# 重置登录失败计数
pam_tally2 --user username --reset

# 查看详细登录历史
last -a | head -20
```

**🔒 sudo权限精细化配置**
```
/etc/sudoers 配置示例：

# 允许wheel组成员执行所有命令
%wheel ALL=(ALL) ALL

# 限制特定用户只能重启服务
username ALL=(root) NOPASSWD: /bin/systemctl restart nginx

# 允许组成员管理特定服务，但需要密码
%webadmin ALL=(root) /bin/systemctl * nginx, /bin/systemctl * apache2
```

## 6. 📊 系统监控工具


### 6.1 实时性能监控

⏱️ **系统资源实时监控**

**实时监控工具集**
- **`top`** / **`htop`** - 进程监控
- **`vmstat`** - 虚拟内存统计
- **`iostat`** - I/O统计信息
- **`pidstat`** - 进程统计信息
- **`nmon`** - 综合性能监控

```bash
# 虚拟内存和CPU监控
vmstat 5 3
# 每5秒显示一次，共3次

# I/O性能监控
iostat -x 2
# 每2秒显示扩展I/O统计

# 按进程监控资源使用
pidstat -u -d -r 5
# 监控CPU、磁盘、内存使用情况
```

**📈 性能指标解读**
```
关键性能指标：

CPU指标：
- %user: 用户空间CPU使用率
- %system: 内核空间CPU使用率  
- %iowait: 等待I/O的CPU时间
- load average: 系统负载平均值

内存指标：
- available: 可用内存
- buffer/cache: 缓冲区和缓存
- swap使用率: 交换分区使用情况

I/O指标：
- %util: 设备利用率
- await: 平均等待时间
- svctm: 平均服务时间
```

### 6.2 历史数据监控

📊 **长期趋势分析工具**

**历史监控工具**
- **`sar`** - 系统活动报告
- **`collectd`** - 系统统计数据收集
- **`munin`** - 网络资源监控
- **`cacti`** - 网络图形化监控
- **`zabbix-agent`** - 企业级监控代理

```bash
# 查看历史CPU使用情况

# 查看10号的CPU统计

# 查看内存使用历史

# 每秒采样，共60次网络设备统计
```

**🔍 监控数据分析方法**
```
趋势分析方法：
```

### 8.3 自定义服务创建

🛠️ **创建和管理自定义服务**

**服务单元文件结构**
```ini
# /etc/systemd/system/myapp.service
[Unit]
Description=My Custom Application
After=network.target
Requires=network.target

[Service]
Type=forking
User=myapp
Group=myapp
ExecStart=/opt/myapp/bin/myapp --daemon
ExecReload=/bin/kill -HUP $MAINPID
PIDFile=/var/run/myapp.pid
Restart=always

[Install]
WantedBy=multi-user.target
```
