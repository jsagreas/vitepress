## 1. 🖥️ /sys文件系统概述


### 1.1 什么是/sys文件系统


**/sys文件系统**是Linux内核提供的一个**虚拟文件系统**，专门用于向用户空间展示内核对象、属性和设备信息。

> 💡 **简单理解**：就像Windows的设备管理器，但是以文件形式展现，让你可以用文本命令查看和修改系统硬件信息

**核心特点**：
```
📁 虚拟文件系统：不占用实际磁盘空间，内容由内核动态生成
🔄 实时更新：文件内容实时反映系统当前状态
📝 可读可写：大部分文件可读，部分可写用于配置
🎯 统一接口：为用户程序提供统一的内核信息访问方式
```

### 1.2 /sys与其他虚拟文件系统的区别


```
系统文件系统对比：

/proc文件系统：
└── 主要功能：进程信息 + 系统统计信息
└── 内容特点：运行时进程状态、内存使用等
└── 典型文件：/proc/cpuinfo, /proc/meminfo

/sys文件系统：
└── 主要功能：硬件设备 + 内核对象信息  
└── 内容特点：设备属性、驱动状态、内核参数
└── 典型目录：/sys/devices, /sys/class, /sys/kernel

/dev文件系统：
└── 主要功能：设备文件节点
└── 内容特点：字符设备、块设备的访问入口
└── 典型文件：/dev/sda1, /dev/tty0
```

### 1.3 /sys的目录结构概览


```
/sys/
├── block/           ← 块设备信息（硬盘、光驱等）
├── bus/             ← 总线信息（PCI、USB等）
├── class/           ← 设备类别（网卡、声卡等）
├── dev/             ← 设备号映射
├── devices/         ← 设备层次结构（核心目录）
├── firmware/        ← 固件信息
├── fs/              ← 文件系统相关参数
├── kernel/          ← 内核参数和调试信息
├── module/          ← 内核模块信息
└── power/           ← 电源管理相关
```

## 2. 🔍 内核对象信息接口


### 2.1 内核对象的概念


**内核对象**是Linux内核中的数据结构，代表系统中的各种资源和组件。

🏠 **生活类比**：
> 就像一个工厂的管理系统，每个设备、每条生产线、每个工人都有一个档案，内核对象就是这些"档案"，记录了它们的状态和属性。

### 2.2 查看内核对象信息


**常用的内核对象查看方法**：

#### 📊 系统基本信息

```bash
# 查看内核版本信息
cat /sys/kernel/version

# 查看内核命令行参数
cat /proc/cmdline

# 查看系统运行时间
cat /proc/uptime
```

#### 🔧 内核模块信息

```bash
# 查看已加载的内核模块
cat /sys/kernel/modules

# 查看特定模块信息
ls /sys/module/e1000e/    # 网卡驱动模块信息
```

**实际示例**：
```
查看内核版本：
$ cat /sys/kernel/version
Linux version 5.15.0-56-generic (buildd@ubuntu) ...

模块参数查看：
$ cat /sys/module/bluetooth/parameters/disable_esco
N    # 表示未禁用esco功能
```

### 2.3 内核对象的层次结构


内核对象按照**层次化**的方式组织，形成树状结构：

```
内核对象层次示例：

系统根对象
├── 设备控制器对象
│   ├── PCI控制器
│   │   ├── 网络适配器
│   │   └── 声音适配器  
│   └── USB控制器
│       ├── USB键盘
│       └── USB鼠标
└── 文件系统对象
    ├── ext4文件系统
    └── tmpfs文件系统
```

## 3. 🖨️ 硬件设备信息管理


### 3.1 设备信息的重要性


硬件设备信息是系统管理的基础，通过/sys可以实时查看和管理所有硬件组件的状态。

**为什么需要了解设备信息**：
- ⚡ **故障诊断**：快速定位硬件问题
- 🔧 **性能调优**：根据硬件特性优化配置  
- 📊 **监控管理**：实时监控硬件状态
- 🛠️ **驱动管理**：检查驱动加载情况

### 3.2 查看硬件设备信息


#### 💾 存储设备信息

```bash
# 查看所有块设备
ls /sys/block/

# 查看硬盘详细信息
cat /sys/block/sda/size          # 扇区数量
cat /sys/block/sda/queue/scheduler # IO调度器
```

**实际输出示例**：
```
$ ls /sys/block/
sda  sdb  sr0    # sda、sdb是硬盘，sr0是光驱

$ cat /sys/block/sda/size
1953525168        # 扇区数，每扇区512字节

$ cat /sys/block/sda/queue/scheduler
noop deadline [cfq]    # []中的是当前使用的调度器
```

#### 🌐 网络设备信息

```bash
# 查看网络接口
ls /sys/class/net/

# 查看网卡状态
cat /sys/class/net/eth0/operstate    # 网卡工作状态
cat /sys/class/net/eth0/speed        # 连接速度
cat /sys/class/net/eth0/address      # MAC地址
```

### 3.3 设备层次结构浏览


#### 📁 /sys/devices目录详解


`/sys/devices`是最重要的目录，包含了系统中所有设备的层次结构信息：

```
/sys/devices/
├── LNXSYSTM:00/          ← ACPI系统设备
├── pci0000:00/           ← PCI设备根
│   ├── 0000:00:02.0/     ← 显卡设备
│   ├── 0000:00:1f.2/     ← SATA控制器
│   └── 0000:00:1f.3/     ← 音频控制器
├── platform/             ← 平台设备
├── system/               ← 系统级设备
└── virtual/              ← 虚拟设备
```

**查看设备详细信息**：
```bash
# 查看PCI设备信息
lspci | head -5

# 查看特定PCI设备属性
ls /sys/devices/pci0000:00/0000:00:02.0/
cat /sys/devices/pci0000:00/0000:00:02.0/vendor
cat /sys/devices/pci0000:00/0000:00:02.0/device
```

### 3.4 设备类别分类


#### 🏷️ /sys/class目录详解


设备按功能分类，便于管理：

| 设备类别 | 目录路径 | 包含设备类型 | 常见用途 |
|---------|----------|--------------|----------|
| 🌐 **网络设备** | `/sys/class/net/` | `eth0, wlan0, lo` | `网络配置、流量监控` |
| 🔊 **音频设备** | `/sys/class/sound/` | `card0, pcmC0D0p` | `音频配置、声卡管理` |
| 💾 **存储设备** | `/sys/class/block/` | `sda, sdb, nvme0n1` | `磁盘管理、性能调优` |
| 🖨️ **输入设备** | `/sys/class/input/` | `mouse0, event0` | `鼠标键盘配置` |
| 💡 **LED设备** | `/sys/class/leds/` | `power, caps_lock` | `指示灯控制` |

## 4. ⚙️ 内核参数配置


### 4.1 内核参数的作用


**内核参数**用于调整Linux内核的行为和性能特征，就像汽车的各种开关和调节器。

🚗 **汽车类比**：
> 内核参数就像汽车的各种设置：空调温度、座椅位置、后视镜角度等，可以根据需要调整来获得最佳体验。

### 4.2 内核参数的分类


#### 🔧 /sys/kernel目录结构

```
/sys/kernel/
├── debug/               ← 内核调试接口
├── mm/                  ← 内存管理参数
├── sched_domain/        ← 调度域配置  
├── security/            ← 安全子系统参数
├── tracing/             ← 内核追踪功能
├── config              ← 内核编译配置(只读)
├── kexec_crash_size    ← 崩溃转储配置
└── version             ← 内核版本信息(只读)
```

### 4.3 常用内核参数配置


#### 📊 内存管理参数


::: warning 注意事项
修改内核参数前建议备份原始设置，错误的参数可能导致系统不稳定
:::

```bash
# 查看内存相关参数
cat /sys/kernel/mm/transparent_hugepage/enabled
cat /sys/kernel/mm/transparent_hugepage/defrag

# 查看当前设置
echo [always] madvise never    # []表示当前激活选项
```

**参数含义**：
- **always**: 总是使用透明大页
- **madvise**: 仅在程序明确请求时使用  
- **never**: 禁用透明大页

#### ⚡ 调度器参数

```bash
# 查看调度相关参数
cat /sys/kernel/sched_child_runs_first
cat /sys/kernel/sched_wakeup_granularity_ns

# 示例输出和含义
0           # 子进程不优先运行
4000000     # 唤醒粒度4毫秒
```

### 4.4 动态修改内核参数


#### ✍️ 临时修改(重启后失效)

```bash
# 启用透明大页
echo always > /sys/kernel/mm/transparent_hugepage/enabled

# 修改调度参数
echo 1 > /sys/kernel/sched_child_runs_first
```

#### 💾 永久修改(写入配置文件)

```bash
# 编辑sysctl配置
sudo vim /etc/sysctl.conf

# 添加参数(如果/sys对应sysctl参数)
vm.swappiness = 10
kernel.sched_child_runs_first = 1

# 使配置生效
sudo sysctl -p
```

- [ ] 理解内核参数的作用
- [ ] 掌握查看参数的方法
- [ ] 了解临时修改与永久修改的区别
- [ ] 知道修改参数的风险和注意事项

## 5. 🚗 设备驱动信息


### 5.1 设备驱动的重要作用


**设备驱动**是操作系统和硬件设备之间的"翻译官"，负责让软件能够正确使用硬件功能。

🎭 **翻译官类比**：
> 就像联合国会议的翻译官，驱动程序把操作系统的"语言"翻译给硬件理解，同时把硬件的"反馈"翻译给操作系统。

### 5.2 查看驱动加载状态


#### 📋 已加载驱动列表

```bash
# 查看所有已加载的驱动模块
lsmod | head -10

# 查看特定驱动的详细信息
modinfo e1000e    # 网卡驱动信息
```

**实际输出示例**：
```
$ lsmod | head -5
Module                  Size  Used by
nvidia_drm             69632  8      # NVIDIA显卡驱动
nvidia_modeset       1048576  10     # NVIDIA模式设置
nvidia              35282944  620    # NVIDIA核心驱动  
e1000e                286720  0      # Intel网卡驱动
```

### 5.3 驱动程序详细信息


#### 🔍 通过/sys查看驱动信息


```bash
# 查看总线上的驱动
ls /sys/bus/pci/drivers/     # PCI驱动
ls /sys/bus/usb/drivers/     # USB驱动

# 查看特定驱动管理的设备
ls /sys/bus/pci/drivers/e1000e/
```

**驱动与设备的绑定关系**：
```
PCI总线驱动示例：

/sys/bus/pci/drivers/e1000e/
├── 0000:00:19.0 -> ../../../../devices/pci0000:00/...
├── bind          ← 手动绑定设备文件
├── unbind        ← 手动解绑设备文件
├── new_id        ← 添加新的设备ID支持
└── remove_id     ← 移除设备ID支持
```

### 5.4 驱动程序管理操作


#### 🔄 动态加载和卸载驱动

```bash
# 加载驱动模块
sudo modprobe bluetooth      # 加载蓝牙驱动

# 卸载驱动模块  
sudo modprobe -r bluetooth   # 卸载蓝牙驱动

# 强制卸载(谨慎使用)
sudo rmmod bluetooth
```

#### ⚠️ 驱动故障排查

```bash
# 查看内核日志中的驱动消息
dmesg | grep -i error
dmesg | grep -i fail

# 查看特定设备的驱动状态
cat /sys/class/net/eth0/operstate
ls -l /sys/class/net/eth0/device/driver/
```

**💡 故障排查技巧**
```
驱动问题诊断流程：

1. 确认硬件被识别
   └── lspci 或 lsusb 查看设备列表

2. 检查驱动是否加载
   └── lsmod | grep 设备名

3. 查看设备状态  
   └── /sys/class/对应类别/设备名/

4. 检查内核日志
   └── dmesg | tail -20
```

## 6. 🔋 电源管理接口


### 6.1 电源管理的重要性


**电源管理**不仅关系到笔记本的续航时间，对服务器的能耗控制也很重要。

💡 **节能类比**：
> 就像现代家庭的智能电器，不用时自动进入节能模式，Linux的电源管理让CPU、硬盘等设备在不需要时降低功耗。

### 6.2 CPU电源管理


#### ⚡ CPU频率调节

```bash
# 查看CPU调频策略
cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_governor

# 查看CPU当前频率
cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_cur_freq

# 查看CPU支持的频率范围
cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_min_freq
cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_max_freq
```

**调频策略说明**：
| 策略名称 | 特点描述 | 适用场景 | 功耗特点 |
|---------|----------|----------|----------|
| **performance** | `始终最高频率` | `高性能计算` | `功耗最高` |
| **powersave** | `始终最低频率` | `超长待机` | `功耗最低` |  
| **ondemand** | `按需调节` | `日常办公` | `平衡功耗` |
| **conservative** | `缓慢调节` | `稳定场景` | `较低功耗` |

#### 🔄 动态调整CPU策略

```bash
# 切换到节能模式
echo powersave > /sys/devices/system/cpu/cpu0/cpufreq/scaling_governor

# 切换到性能模式  
echo performance > /sys/devices/system/cpu/cpu0/cpufreq/scaling_governor

# 批量设置所有CPU核心
for cpu in /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor; do
    echo ondemand > $cpu
done
```

### 6.3 系统休眠与唤醒


#### 💤 系统睡眠状态


Linux支持多种睡眠状态：

```
睡眠状态层次：

S0 - 正常工作状态
├── 所有设备正常工作
└── 功耗最高

S1 - 轻度睡眠
├── CPU停止执行指令
├── 内存保持供电
└── 快速唤醒

S3 - 深度睡眠(STR)  
├── 内存内容保存到RAM
├── CPU和大部分设备断电
└── 唤醒速度中等

S4 - 休眠状态(STD)
├── 内存内容保存到硬盘
├── 系统完全断电
└── 唤醒速度慢，但最省电
```

#### 🛌 查看和控制系统睡眠

```bash
# 查看支持的睡眠状态
cat /sys/power/state

# 查看当前内存睡眠模式
cat /sys/power/mem_sleep

# 进入睡眠状态(需要root权限)
echo mem > /sys/power/state      # 进入内存睡眠
echo disk > /sys/power/state     # 进入磁盘休眠
```

### 6.4 设备电源管理


#### 🖥️ 设备唤醒控制

```bash
# 查看哪些设备可以唤醒系统
cat /proc/acpi/wakeup

# 查看USB设备的电源管理
ls /sys/bus/usb/devices/*/power/

# 查看特定设备的自动挂起设置
cat /sys/bus/usb/devices/1-1/power/autosuspend_delay_ms
```

**🚨 常见误区**
> ❌ **错误理解**：认为所有设备都支持电源管理
> ✅ **正确理解**：只有支持ACPI的设备才能进行电源管理

## 7. 🌐 网络设备配置


### 7.1 网络设备管理概述


Linux通过/sys提供了丰富的网络设备管理接口，可以查看和配置网卡的各种参数。

### 7.2 网络接口信息查看


#### 📡 基本网络接口信息

```bash
# 查看所有网络接口
ls /sys/class/net/

# 查看网卡基本状态
cat /sys/class/net/eth0/operstate    # up/down状态
cat /sys/class/net/eth0/carrier      # 载波检测(1=连接,0=断开)  
cat /sys/class/net/eth0/mtu          # 最大传输单元
```

**实际示例输出**：
```
$ ls /sys/class/net/
eth0  lo  wlan0

$ cat /sys/class/net/eth0/operstate
up           # 网卡处于活动状态

$ cat /sys/class/net/eth0/speed  
1000         # 连接速度1000Mbps(千兆网卡)
```

#### 🔧 网卡详细配置信息

```bash
# MAC地址和硬件信息
cat /sys/class/net/eth0/address       # MAC地址
cat /sys/class/net/eth0/addr_len      # 地址长度(以太网为6)
cat /sys/class/net/eth0/type          # 接口类型(1=以太网)

# 传输统计信息
cat /sys/class/net/eth0/statistics/rx_bytes    # 接收字节数
cat /sys/class/net/eth0/statistics/tx_bytes    # 发送字节数
cat /sys/class/net/eth0/statistics/rx_packets  # 接收数据包数  
cat /sys/class/net/eth0/statistics/tx_packets  # 发送数据包数
```

### 7.3 网络性能监控


#### 📊 实时网络统计

```bash
# 创建简单的网络流量监控脚本
#!/bin/bash
interface="eth0"
while true; do
    rx_bytes=$(cat /sys/class/net/$interface/statistics/rx_bytes)
    tx_bytes=$(cat /sys/class/net/$interface/statistics/tx_bytes)
    
    echo "接收: $(($rx_bytes / 1024 / 1024)) MB"
    echo "发送: $(($tx_bytes / 1024 / 1024)) MB"
    sleep 5
done
```

#### 🔍 网络错误诊断

```bash
# 查看网络错误统计
cat /sys/class/net/eth0/statistics/rx_errors     # 接收错误
cat /sys/class/net/eth0/statistics/tx_errors     # 发送错误
cat /sys/class/net/eth0/statistics/rx_dropped    # 接收丢包
cat /sys/class/net/eth0/statistics/tx_dropped    # 发送丢包
```

### 7.4 无线网络管理


#### 📶 无线网卡特有属性

```bash
# 无线网卡相关信息(如果存在)
ls /sys/class/net/wlan0/wireless/
cat /sys/class/net/wlan0/wireless/status
cat /sys/class/net/wlan0/wireless/link
```

**🎯 网络诊断检查点**
1. **连接状态**: operstate是否为up
2. **物理连接**: carrier是否为1
3. **传输统计**: 是否有数据收发
4. **错误检查**: errors和dropped是否异常

## 8. 🚀 系统调优参数


### 8.1 系统调优的意义


**系统调优**是根据实际应用场景，调整内核参数来获得最佳性能的过程。

🎯 **赛车调校类比**：
> 就像F1赛车在不同赛道需要调整悬挂、轮胎气压等参数，Linux系统也需要根据工作负载调整各种参数来获得最佳性能。

### 8.2 内存子系统调优


#### 💾 虚拟内存参数

```bash
# 查看内存管理关键参数
cat /sys/kernel/mm/transparent_hugepage/enabled
cat /sys/kernel/mm/transparent_hugepage/defrag
cat /sys/kernel/mm/transparent_hugepage/khugepaged/pages_to_scan
```

**透明大页配置说明**：
- **enabled**: 控制是否启用透明大页
- **defrag**: 控制内存碎片整理策略
- **pages_to_scan**: khugepaged每次扫描的页面数

#### 📊 内存回收参数

```bash
# 查看页面回收相关参数
cat /proc/sys/vm/swappiness           # swap使用倾向(0-100)
cat /proc/sys/vm/dirty_ratio          # 脏页回写比例
cat /proc/sys/vm/dirty_background_ratio # 后台回写触发比例
```

**参数调优建议**：
| 应用场景 | swappiness值 | 说明 |
|---------|-------------|------|
| **数据库服务器** | `1-10` | `尽量避免swap，保持内存响应` |
| **桌面系统** | `60(默认)` | `平衡内存和swap使用` |
| **内存充足系统** | `10-20` | `优先使用物理内存` |

### 8.3 I/O子系统调优


#### 💿 磁盘I/O调度器

```bash
# 查看当前I/O调度器
cat /sys/block/sda/queue/scheduler

# 查看I/O调度器参数
ls /sys/block/sda/queue/iosched/
```

**I/O调度器对比**：
| 调度器 | 特点 | 适用场景 | 延迟特性 |
|--------|------|----------|----------|
| **noop** | `先进先出，无优化` | `SSD固态硬盘` | `延迟稳定` |
| **deadline** | `保证读写请求延迟上限` | `数据库应用` | `延迟可控` |
| **cfq** | `完全公平队列` | `桌面多任务` | `延迟较高` |

#### ⚡ I/O队列参数调优

```bash
# 查看队列深度
cat /sys/block/sda/queue/nr_requests

# 查看读写合并
cat /sys/block/sda/queue/read_ahead_kb
cat /sys/block/sda/queue/max_sectors_kb
```

### 8.4 网络子系统调优


#### 🌐 网络缓冲区调优

```bash
# TCP缓冲区参数
cat /proc/sys/net/core/rmem_default    # 默认接收缓冲区
cat /proc/sys/net/core/rmem_max        # 最大接收缓冲区
cat /proc/sys/net/core/wmem_default    # 默认发送缓冲区
cat /proc/sys/net/core/wmem_max        # 最大发送缓冲区
```

#### 📊 网络性能参数

```bash
# 网络队列长度
cat /proc/sys/net/core/netdev_max_backlog

# TCP窗口缩放
cat /proc/sys/net/ipv4/tcp_window_scaling

# TCP拥塞控制算法
cat /proc/sys/net/ipv4/tcp_congestion_control
```

### 8.5 实用调优脚本示例


#### 🔧 数据库服务器优化脚本

```bash
#!/bin/bash
# 数据库服务器优化脚本

# 内存优化
echo 1 > /proc/sys/vm/swappiness                    # 减少swap使用
echo madvise > /sys/kernel/mm/transparent_hugepage/enabled  # 启用大页

# I/O优化  
echo deadline > /sys/block/sda/queue/scheduler      # 使用deadline调度器
echo 4096 > /sys/block/sda/queue/nr_requests        # 增加队列深度

# 网络优化
echo 65536 > /proc/sys/net/core/rmem_default        # 增加网络缓冲区
echo 65536 > /proc/sys/net/core/wmem_default

echo "数据库服务器优化完成"
```

**⭐ 调优重要原则**
```
系统调优黄金法则：

1. 先测试后应用：在测试环境验证效果
2. 逐步调整：一次只改一个参数  
3. 监控对比：调优前后进行性能对比
4. 做好备份：记录原始参数值
5. 文档记录：记录修改原因和效果
```

## 9. 📋 核心要点总结


### 9.1 必须掌握的核心概念


```
🔸 /sys文件系统：内核向用户空间展示设备和对象信息的虚拟文件系统
🔸 设备管理：通过/sys/devices、/sys/class查看和管理硬件设备
🔸 驱动信息：通过/sys/bus、/sys/module了解驱动加载状态
🔸 电源管理：CPU调频、系统睡眠、设备电源控制
🔸 网络配置：网卡状态查看、性能监控、参数调整
🔸 系统调优：内存、I/O、网络子系统的性能优化
🔸 参数配置：内核参数的查看、修改和持久化
```

### 9.2 关键理解要点


**🔹 /sys的本质作用**
```
信息展示：实时反映内核和硬件状态
配置接口：提供参数修改的标准途径  
调试工具：系统问题诊断的重要手段
管理平台：设备和驱动管理的统一界面
```

**🔹 实际应用价值**
```
故障诊断：快速定位硬件和驱动问题
性能优化：根据系统特点进行调优
监控管理：实时监控系统健康状态
自动化运维：脚本化系统管理操作
```

### 9.3 实用技能掌握


**📊 日常管理任务**
- **设备检查**: `ls /sys/class/net/` 查看网络设备
- **驱动状态**: `lsmod` + `/sys/module/` 检查驱动
- **性能监控**: 通过统计文件监控I/O和网络
- **电源管理**: CPU调频和系统睡眠控制
- **参数调优**: 根据应用场景优化内核参数

**🔧 故障排查流程**
1. **确认现象**: 通过/sys查看设备状态
2. **检查驱动**: 确认相关驱动是否正常加载
3. **分析日志**: 结合dmesg查看错误信息  
4. **测试修复**: 尝试重新加载驱动或调整参数
5. **验证效果**: 确认问题是否解决
