## 1. 🌐 IPv6地址分配基础概念


### 1.1 什么是IPv6地址分配


**简单理解**：IPv6地址分配就像给每个网络设备分配一个"网络身份证号码"，让设备能在互联网上被找到和通信。

```
类比理解：
IPv4时代 = 门牌号不够用了（地址枯竭）
IPv6时代 = 新的超大号码系统（地址充足）

就像从4位门牌号升级到32位门牌号
不仅号码够用，分配方式也更智能了
```

### 1.2 IPv6地址分配的特点


**核心特点对比**：

| 特性 | **IPv4** | **IPv6** |
|------|---------|---------|
| **地址长度** | `32位（4字节）` | `128位（16字节）` |
| **表示方法** | `192.168.1.1` | `2001:db8::1` |
| **分配方式** | `主要依赖DHCP` | `多种自动配置方式` |
| **地址数量** | `约43亿个` | `约340万亿亿亿个` |
| **配置复杂度** | `需要手动或DHCP` | `可以自动配置` |

### 1.3 IPv6地址的基本分类


**按使用范围分类**：
```
┌─────────────────────────────────────┐
│           IPv6地址类型               │
├─────────────────────────────────────┤
│ 🌍 全局单播地址                      │ ← 互联网通信用
│   (Global Unicast Address)         │
├─────────────────────────────────────┤
│ 🔗 链路本地地址                      │ ← 同一网段通信用
│   (Link-Local Address)             │
├─────────────────────────────────────┤
│ 🏠 站点本地地址（已废弃）              │ ← 局域网通信用
│   (Site-Local Address)             │
├─────────────────────────────────────┤
│ 🔄 组播地址                          │ ← 一对多通信用
│   (Multicast Address)              │
└─────────────────────────────────────┘
```

## 2. 🌍 全局单播地址分配机制


### 2.1 什么是全局单播地址


**通俗解释**：全局单播地址就像你的手机号码，在全世界范围内都是唯一的，任何人都可以通过这个号码找到你。

**地址结构解析**：
```
IPv6全局单播地址结构：
┌─────────────────┬─────────────────┬─────────────────┐
│   网络前缀       │    子网ID       │    接口ID       │
│   (48-64位)     │   (16-0位)      │    (64位)       │
└─────────────────┴─────────────────┴─────────────────┘

实际示例：
2001:0db8:85a3:0042:1000:8a2e:0370:7334
│               │    │              │
网络部分        子网  主机部分
```

### 2.2 地址分配层次结构


**分配流程图**：
```
IANA (全球)
    │
    ├── APNIC (亚太地区)
    │     │
    │     ├── 中国互联网络信息中心 (CNNIC)
    │     │     │
    │     │     ├── 电信运营商
    │     │     │     │
    │     │     │     ├── 企业/组织
    │     │     │     │     │
    │     │     │     │     └── 最终用户设备
    │     │     │     │
    │     │     │     └── 家庭用户
    │     │     │           │
    │     │     │           └── 家庭设备
```

### 2.3 实际分配示例


**运营商分配给用户的典型情况**：
```bash
# 运营商给家庭用户分配的前缀（通常是/56或/64）
家庭网络前缀: 2001:db8:1234:5600::/56

# 用户可以划分出256个子网：
子网1: 2001:db8:1234:5601::/64  (客厅WiFi)
子网2: 2001:db8:1234:5602::/64  (书房网络)
子网3: 2001:db8:1234:5603::/64  (智能家居)
...

# 每个子网可以容纳 2^64 个设备
```

## 3. 🔗 链路本地地址自动配置


### 3.1 链路本地地址的作用


**简单理解**：链路本地地址就像局域网内的"临时通行证"，只在同一个网段内有效，用于设备之间的基本通信。

**重要特点**：
- **自动生成**：设备启动时自动创建，无需手动配置
- **本地有效**：只能在同一链路（网段）内使用
- **必须存在**：每个IPv6接口都必须有链路本地地址
- **不可路由**：路由器不会转发这类地址的数据包

### 3.2 链路本地地址的格式


**地址结构**：
```
链路本地地址格式：
┌──────────────┬──────────────┬────────────────────────────┐
│   FE80::/10  │   54个0位    │        64位接口标识符        │
│  (固定前缀)   │             │                            │
└──────────────┴──────────────┴────────────────────────────┘

实际示例：
FE80::1234:5678:9ABC:DEF0
│     │
固定   接口ID
前缀
```

### 3.3 自动配置过程


**配置步骤解析**：
```
设备启动时的配置流程：

步骤1: 生成临时链路本地地址
├── 使用FE80::/10前缀
└── 基于MAC地址生成接口ID

步骤2: 重复地址检测（DAD）
├── 发送邻居请求消息
├── 检查是否有地址冲突
└── 确认地址唯一性

步骤3: 地址状态变更
├── 从"临时"变为"可用"
└── 开始正常网络通信

实际命令查看：
$ ip addr show eth0
# 会看到类似：inet6 fe80::a00:27ff:fe4e:66a1/64 scope link
```

### 3.4 Linux系统中的链路本地地址


**查看和管理命令**：
```bash
# 查看所有接口的IPv6地址
ip -6 addr show

# 查看特定接口
ip -6 addr show eth0

# 输出示例：
# inet6 fe80::a00:27ff:fe4e:66a1/64 scope link
#        ↑ 链路本地地址            ↑ 作用域
```

## 4. 🤖 无状态地址自动配置(SLAAC)


### 4.1 SLAAC是什么


**通俗解释**：SLAAC就像"智能门牌号分配系统"，设备插上网线后，路由器会广播"这个小区的门牌号规则"，设备自己就能算出自己的门牌号，不需要专门的分配员（DHCP服务器）。

**核心优势**：
- ✅ **真正的即插即用**：设备连网即可自动获得地址
- ✅ **无需服务器**：不依赖DHCP服务器
- ✅ **分布式管理**：每个设备自主配置
- ✅ **减少网络负载**：避免集中式地址分配的瓶颈

### 4.2 SLAAC工作原理


**工作流程详解**：
```
SLAAC地址配置流程：

设备端：                              路由器端：
  │                                      │
  ├─[1] 生成链路本地地址                  │
  │                                      │
  ├─[2] 发送路由器请求(RS)───────────────→│
  │                                      │
  │                              ┌─[3] 发送路由器通告(RA)
  │←─────────────────────────────┘      │
  │   包含网络前缀信息                    │
  │                                      │
  ├─[4] 结合前缀+接口ID生成全局地址        │
  │                                      │
  ├─[5] 执行重复地址检测(DAD)             │
  │                                      │
  └─[6] 地址配置完成，开始通信            │
```

### 4.3 路由器通告(RA)消息解析


**RA消息包含的关键信息**：
```
路由器通告消息内容：
┌─────────────────────────────────────┐
│ 🌐 网络前缀信息                      │ ← 如：2001:db8::/64
├─────────────────────────────────────┤
│ ⏰ 前缀有效期                        │ ← 地址可用时间
├─────────────────────────────────────┤
│ 🔧 配置标志                          │ ← 告知配置方式
│   • M标志：是否使用DHCPv6            │
│   • O标志：是否从DHCPv6获取其他配置   │
├─────────────────────────────────────┤
│ 🚪 默认网关信息                      │ ← 路由器地址
├─────────────────────────────────────┤
│ 📊 网络参数                          │ ← 跳数限制等
└─────────────────────────────────────┘
```

### 4.4 SLAAC实际配置示例


**Linux系统中的SLAAC**：
```bash
# 启用接口的IPv6自动配置
echo 1 > /proc/sys/net/ipv6/conf/eth0/autoconf

# 查看自动配置的地址
ip -6 addr show eth0

# 典型输出：
# inet6 2001:db8:1234:5678:a00:27ff:fe4e:66a1/64 scope global dynamic
#       ↑ 全局地址（SLAAC生成）                    ↑ 动态获得

# 查看路由器通告信息
rdisc6 eth0
```

**SLAAC地址构成分析**：
```
自动生成的IPv6地址组成：
2001:db8:1234:5678:a00:27ff:fe4e:66a1
│                │                  │
网络前缀(RA提供)   接口ID(设备生成)
64位              64位

接口ID生成方式：
├── 基于MAC地址的EUI-64格式
├── 随机生成（隐私扩展）
└── 稳定私有地址算法
```

## 5. 🏢 有状态地址配置(DHCPv6)


### 5.1 DHCPv6的作用和定位


**与SLAAC的关系**：DHCPv6不是SLAAC的替代品，而是补充。就像SLAAC是"自助门牌号系统"，DHCPv6是"专业门牌号分配员"，提供更精确的控制和额外服务。

**DHCPv6的优势**：
- **精确控制**：管理员可以指定具体的地址分配策略
- **额外配置**：提供DNS服务器、域名等网络参数
- **集中管理**：便于企业网络的统一管理
- **地址回收**：可以回收和重新分配地址

### 5.2 DHCPv6的两种工作模式


**模式对比**：

| 模式 | **有状态模式** | **无状态模式** |
|------|---------------|---------------|
| **地址分配** | `DHCPv6分配地址` | `SLAAC分配地址` |
| **其他配置** | `DHCPv6提供DNS等` | `DHCPv6提供DNS等` |
| **服务器负载** | `较高（管理地址）` | `较低（仅提供配置）` |
| **适用场景** | `企业网络、需要严格管理` | `家庭网络、简单环境` |

### 5.3 DHCPv6配置过程


**有状态DHCPv6配置流程**：
```
客户端请求过程：

客户端                               DHCPv6服务器
  │                                      │
  ├─[1] 发送Solicit消息 ─────────────────→│ 寻找DHCPv6服务器
  │                                      │
  │                              ┌─[2] 发送Advertise消息
  │←─────────────────────────────┘      │ 响应可用性
  │                                      │
  ├─[3] 发送Request消息 ──────────────────→│ 请求具体地址
  │                                      │
  │                              ┌─[4] 发送Reply消息
  │←─────────────────────────────┘      │ 分配地址和配置
  │                                      │
  └─[5] 地址配置完成                      │
```

### 5.4 实际配置示例


**Linux下的DHCPv6客户端配置**：
```bash
# 安装DHCPv6客户端
sudo apt-get install dhcpcd5  # Ubuntu/Debian
sudo yum install dhcp-client  # CentOS/RHEL

# 启动DHCPv6客户端
dhcpcd -6 eth0

# 或使用dhclient
dhclient -6 eth0

# 查看获得的地址
ip -6 addr show eth0
# 输出类似：
# inet6 2001:db8:1234:5678::100/64 scope global dynamic
```

**路由器配置指示**：
```
路由器通告中的DHCPv6标志：

M标志 = 1：使用有状态DHCPv6获取地址
O标志 = 1：使用无状态DHCPv6获取其他配置信息

标志组合含义：
├── M=0, O=0：仅使用SLAAC
├── M=0, O=1：SLAAC + DHCPv6配置信息
├── M=1, O=0：DHCPv6地址 + RA配置信息  
└── M=1, O=1：完全使用DHCPv6
```

## 6. 🔒 临时地址与隐私扩展


### 6.1 隐私问题的产生


**为什么需要隐私扩展**：传统的基于MAC地址生成的IPv6地址会暴露设备的硬件信息，就像每次出门都带着身份证号码一样，容易被追踪。

**隐私风险示例**：
```
传统地址生成（基于MAC地址）：
MAC地址：00:1A:2B:3C:4D:5E
IPv6地址：2001:db8::21a:2bff:fe3c:4d5e
                   ↑
                包含MAC信息，可追踪设备

隐私扩展地址（随机生成）：
IPv6地址：2001:db8::8a3f:9e2d:4c1b:7f6e
                   ↑
                随机生成，无法追踪
```

### 6.2 临时地址的工作机制


**地址生成和轮换**：
```
隐私扩展地址的生命周期：

时间轴：  0天    1天    3天    7天    14天
         │      │      │      │       │
地址1：   ├生成──┼优选──┼废弃──┼────────┼删除
地址2：   │      ├生成──┼优选──┼废弃────┼删除
地址3：   │      │      ├生成──┼优选────┼废弃

说明：
- 每个地址有"优选期"和"废弃期"
- 新连接使用优选地址
- 旧连接继续使用废弃地址直到结束
- 定期生成新地址确保隐私
```

### 6.3 Linux中的隐私扩展配置


**启用和配置隐私扩展**：
```bash
# 查看当前隐私扩展状态
cat /proc/sys/net/ipv6/conf/all/use_tempaddr
# 0=禁用, 1=启用但优选公共地址, 2=启用且优选临时地址

# 启用隐私扩展（优选临时地址）
echo 2 > /proc/sys/net/ipv6/conf/all/use_tempaddr

# 设置临时地址的有效期（秒）
echo 86400 > /proc/sys/net/ipv6/conf/all/temp_valid_lft    # 1天
echo 3600 > /proc/sys/net/ipv6/conf/all/temp_prefered_lft  # 1小时

# 查看接口上的临时地址
ip -6 addr show eth0 | grep "scope global temporary"
```

### 6.4 地址类型识别


**不同类型地址的特征**：
```bash
# 查看地址时的关键标识
ip -6 addr show eth0

输出解析：
inet6 2001:db8::21a:2bff:fe3c:4d5e/64 scope global         # 永久地址
inet6 2001:db8::8a3f:9e2d:4c1b:7f6e/64 scope global temporary  # 临时地址
inet6 fe80::21a:2bff:fe3c:4d5e/64 scope link               # 链路本地地址
                                              ↑
                                          关键标识词
```

## 7. 🎯 地址选择算法与优先级


### 7.1 为什么需要地址选择


**实际场景**：现代设备通常同时拥有多个IPv6地址，系统需要智能选择最合适的地址进行通信。

**常见的多地址情况**：
```
一个网络接口的多个地址：
eth0: 
├── fe80::a00:27ff:fe4e:66a1/64        (链路本地地址)
├── 2001:db8::a00:27ff:fe4e:66a1/64    (SLAAC全局地址)  
├── 2001:db8::8a3f:9e2d:4c1b:7f6e/64  (临时地址)
└── 2001:db8::100/64                   (DHCPv6地址)

问题：访问外网时应该选择哪个源地址？
```

### 7.2 地址选择的基本原则


**RFC 6724地址选择算法**：

| 优先级 | **规则** | **含义** | **示例** |
|--------|---------|---------|---------|
| **最高** | `作用域匹配` | `优选相同作用域的地址` | `全局访问全局，本地访问本地` |
| **高** | `标签匹配` | `IPv6优于IPv4` | `双栈环境下优选IPv6` |
| **中** | `前缀匹配` | `更长的公共前缀优先` | `同网段通信优选同网段地址` |
| **低** | `临时地址` | `隐私保护时优选临时地址` | `上网浏览优选临时地址` |

### 7.3 地址选择实际示例


**选择过程演示**：
```
场景：从内网设备访问外网服务器

本地地址候选：
├── fe80::a00:27ff:fe4e:66a1     (链路本地) → 作用域不匹配，排除
├── 2001:db8::a00:27ff:fe4e:66a1 (全局永久) → 可用
└── 2001:db8::8a3f:9e2d:4c1b:7f6e (全局临时) → 可用，隐私保护更好

目标地址：2001:470:1f0b:18f8::2 (外网服务器)

选择结果：使用临时地址作为源地址（隐私保护）
```

### 7.4 Linux系统中的地址选择配置


**查看和调整地址选择策略**：
```bash
# 查看地址选择策略表
ip addrlabel show

# 典型输出：
# prefix ::1/128 label 0
# prefix ::/0 label 1
# prefix 2002::/16 label 2
# prefix fc00::/7 label 5
# prefix fec0::/10 label 11
# prefix 3ffe::/16 label 12

# 调整IPv6地址优先级（使临时地址优先）
echo 2 > /proc/sys/net/ipv6/conf/all/use_tempaddr
```

## 8. 🔍 重复地址检测(DAD)机制


### 8.1 DAD机制的重要性


**为什么需要DAD**：即使在IPv6的巨大地址空间中，地址冲突仍然可能发生，特别是在使用某些地址生成算法时。DAD就像"新搬家时检查门牌号是否重复"的机制。

**可能发生冲突的情况**：
- 多个设备使用相同的MAC地址（虚拟机克隆）
- 手动配置了相同的地址
- 某些随机算法产生了相同结果
- 网络配置错误导致地址重叠

### 8.2 DAD检测过程


**检测流程详解**：
```
重复地址检测流程：

设备A配置新地址：2001:db8::1
         │
         ├─[1] 将地址标记为"暂定"状态
         │
         ├─[2] 发送邻居请求消息(NS)
         │     目标：2001:db8::1
         │     源：:: (未指定地址)
         │
设备B(如果存在)：│
         │    ├─[3] 收到NS，发现查询自己的地址
         │    └─[4] 发送邻居通告消息(NA)
         │
设备A：   ├─[5] 收到NA响应？
         │     ├─YES → 地址冲突，标记为"重复"
         │     └─NO  → 等待超时
         │
         └─[6] 超时无响应 → 地址唯一，标记为"可用"
```

### 8.3 DAD的技术细节


**关键参数和时间**：
```
DAD检测参数：
┌─────────────────────────────────────┐
│ 📊 检测次数：1次（默认）              │
│ ⏱️ 等待时间：1秒                     │
│ 🚫 检测期间：地址不可用                │
│ 🔄 失败处理：禁用地址或生成新地址      │
└─────────────────────────────────────┘

实际时间线：
0秒：开始DAD检测
0.1秒：发送第1个NS消息
1.0秒：检测完成，地址可用
```

### 8.4 Linux系统中的DAD配置


**DAD相关的系统配置**：
```bash
# 查看DAD配置
cat /proc/sys/net/ipv6/conf/eth0/dad_transmits
# 默认值：1（发送1次检测消息）

# 禁用DAD（不推荐）
echo 0 > /proc/sys/net/ipv6/conf/eth0/dad_transmits

# 查看地址状态（包括DAD状态）
ip -6 addr show eth0
# tentative：正在进行DAD检测
# dadfailed：DAD检测失败
# deprecated：地址已废弃

# 监控DAD过程
tcpdump -i eth0 icmp6 and 'icmp6[0] == 135'  # 监控NS消息
```

### 8.5 DAD失败的处理


**冲突检测到后的处理策略**：
```
DAD失败后的系统行为：

自动配置地址（SLAAC）：
├── 禁用当前地址
├── 生成新的随机接口ID
└── 重新尝试DAD检测

手动配置地址：
├── 标记地址为"dadfailed"
├── 地址不可用
└── 需要管理员手动解决

临时地址：
├── 立即生成新的临时地址
└── 继续使用其他可用地址
```

## 9. 📅 地址租约管理与更新


### 9.1 地址生命周期管理


**地址的生命周期概念**：IPv6地址不是"永久有效"的，而是有"保质期"的，就像驾驶证需要定期更新一样。

**生命周期参数**：
```
IPv6地址的两个关键时间：
┌─────────────────────────────────────┐
│ ⏰ 优选生命期 (Preferred Lifetime)    │
│   └── 地址处于"最佳使用状态"的时间    │
├─────────────────────────────────────┤  
│ 🕐 有效生命期 (Valid Lifetime)       │
│   └── 地址完全有效可用的总时间        │
└─────────────────────────────────────┘

时间关系：优选生命期 ≤ 有效生命期
```

### 9.2 地址状态转换


**地址状态变化过程**：
```
地址生命周期状态图：

配置 → 暂定 → 优选 → 废弃 → 无效
 │      │      │      │      │
 │   [DAD]   [正常使用] [仅维持] [删除]
 │   检测期   新连接使用 现有连接  地址移除
 │            优选此地址 可继续用
 │
手动配置
自动获得

实际时间示例：
0天：地址获得，进入优选状态
30天：优选期满，进入废弃状态（不用于新连接）
60天：有效期满，地址删除（所有连接断开）
```

### 9.3 租约更新机制


**SLAAC地址的更新**：
```
路由器通告更新周期：

路由器：                           客户端设备：
  │                                  │
  ├─每隔3-600秒发送RA消息───────────→ │ 接收RA，更新地址生命期
  │ (包含最新的生命期信息)            │
  │                                  │
  │                          ┌─当生命期<30%时
  │ ←─────────────────────────┤ 发送RS请求最新RA
  │ 立即发送RA响应             │
  │ ─────────────────────────→ │ 重置生命期计时器

设备端处理：
生命期剩余 > 50%：继续使用
生命期剩余 30-50%：发送RS请求更新
生命期剩余 < 30%：准备获取新地址
```

### 9.4 DHCPv6租约管理


**有状态地址的租约更新**：
```bash
# DHCPv6租约信息通常保存在：
/var/lib/dhcp/dhclient6.leases

# 典型租约信息内容：
lease6 {
    interface "eth0";
    ia-na "00:00:00:00";
    iaaddr 2001:db8::100 {
        preferred-life 3600;    # 优选期：1小时
        max-life 7200;         # 有效期：2小时
        binding state active;
    }
    server-id 00:01:00:01:xx:xx:xx:xx;
}

# 手动更新租约
dhclient -6 -r eth0  # 释放当前租约
dhclient -6 eth0     # 重新获取租约
```

### 9.5 生命周期监控和管理


**实际监控命令**：
```bash
# 查看地址的详细信息（包括生命期）
ip -6 addr show eth0

# 输出示例解读：
# inet6 2001:db8::1/64 scope global dynamic
#    valid_lft 86394sec preferred_lft 14394sec
#                ↑ 有效剩余时间    ↑ 优选剩余时间

# 实时监控地址变化
ip monitor address

# 检查路由器通告信息
rdisc6 eth0
```

## 10. 📋 核心要点总结


### 10.1 必须掌握的核心概念


```
🔸 IPv6地址分配的多样性：不再依赖单一的DHCP，支持多种自动配置方式
🔸 SLAAC的革命性：真正的即插即用，设备自主配置全局地址
🔸 DHCPv6的补充作用：提供精确控制和额外配置信息
🔸 隐私扩展的重要性：保护用户隐私，防止设备追踪
🔸 地址生命周期管理：地址有"保质期"，需要定期更新维护
```

### 10.2 关键技术对比


| 特性 | **SLAAC** | **DHCPv6有状态** | **DHCPv6无状态** |
|------|-----------|-----------------|-----------------|
| **地址分配** | `自动生成` | `服务器分配` | `SLAAC生成` |
| **配置信息** | `从RA获取` | `从服务器获取` | `从服务器获取` |
| **服务器依赖** | `仅需路由器` | `需要专用服务器` | `需要专用服务器` |
| **管理复杂度** | `低` | `高` | `中` |
| **适用环境** | `家庭、小型网络` | `企业、需要管控` | `混合环境` |

### 10.3 实际应用指导


**选择配置方式的决策树**：
```
网络环境分析：
├── 家庭/小型办公网络
│   └── 推荐：SLAAC + 隐私扩展
│       ├── 配置简单，即插即用
│       └── 自动隐私保护
│
├── 企业网络
│   └── 推荐：DHCPv6有状态
│       ├── 集中管理，安全可控
│       └── 地址分配可追踪
│
└── 混合环境
    └── 推荐：SLAAC + DHCPv6无状态
        ├── 地址自动配置
        └── 配置信息集中管理
```
