## 1. 🔍 IPv6故障排查概述


### 1.1 IPv6故障的特点


**与IPv4故障的区别：**
IPv6网络故障往往比IPv4更加复杂，主要原因包括：

```
复杂性因素：
├─ 地址类型多样化
│  ├─ 本地链路地址 (fe80::/10)
│  ├─ 全局单播地址 (2000::/3)
│  ├─ 唯一本地地址 (fc00::/7)
│  └─ 多播地址 (ff00::/8)
├─ 双栈共存问题
│  ├─ IPv4/IPv6优先级冲突
│  ├─ DNS解析选择问题
│  └─ 应用程序兼容性
└─ 自动配置复杂性
   ├─ SLAAC机制
   ├─ DHCPv6配置
   └─ 路由器通告
```

### 1.2 故障排查方法论


**🔧 系统性排查思路：**

| 排查层次 | 检查内容 | 常用工具 | 关键指标 |
|---------|----------|----------|----------|
| **物理层** | 链路状态 | `ip link` | 接口UP/DOWN |
| **数据链路层** | 邻居发现 | `ip -6 neigh` | 邻居状态 |
| **网络层** | 地址配置 | `ip -6 addr` | 地址有效性 |
| **传输层** | 端口监听 | `ss -6` | 服务绑定 |
| **应用层** | 服务响应 | `curl -6` | 应用可达性 |

### 1.3 排查工具箱


**🛠️ 必备工具清单：**

```bash
# 基础诊断工具
ping6          # IPv6连通性测试
traceroute6    # IPv6路径跟踪
ip -6          # IPv6网络配置查看
ss -6          # IPv6网络连接状态
tcpdump        # 数据包分析

# 高级诊断工具
ndisc6         # 邻居发现测试
rdisc6         # 路由器发现测试
addr6          # IPv6地址分析
nc -6          # IPv6网络连接测试
```

## 2. 🔗 连接故障诊断


### 2.1 基础连接测试


**🎯 连接性测试流程：**

**第一步：本地接口检查**
```bash
# 查看IPv6网络接口状态
ip -6 addr show

# 预期输出示例：
# 2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500
#    inet6 2001:db8::100/64 scope global
#    inet6 fe80::a00:27ff:fe4e:66a1/64 scope link
```

> 💡 **检查要点**
> - 接口是否UP状态
> - 是否有全局IPv6地址
> - 本地链路地址是否正常

**第二步：本地连通性测试**
```bash
# 测试本地回环
ping6 ::1

# 测试本地链路地址
ping6 fe80::a00:27ff:fe4e:66a1%eth0
```

**第三步：网关连通性测试**
```bash
# 查看默认网关
ip -6 route show default

# 测试网关连通性
ping6 fe80::1%eth0  # 本地链路网关
ping6 2001:db8::1   # 全局网关
```

### 2.2 连接故障类型分析


**🔍 故障类型判断表：**

```
故障现象分类：

完全无法连接 ━━━┳━━━ 接口未启用
                ┣━━━ 无IPv6地址  
                ┗━━━ 硬件故障

部分无法连接 ━━━┳━━━ 本地链路正常，全局地址异常
                ┣━━━ 网关不可达
                ┗━━━ DNS解析问题

连接不稳定 ━━━━┳━━━ 路由震荡
                ┣━━━ 负载均衡问题
                ┗━━━ 隧道不稳定
```

### 2.3 连接故障诊断实例


**❌ 故障案例：无法获取IPv6地址**

```bash
# 问题现象
$ ip -6 addr show eth0
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP>
    inet6 fe80::a00:27ff:fe4e:66a1/64 scope link
    # 缺少全局IPv6地址

# 诊断步骤
$ ip -6 route show
fe80::/64 dev eth0 proto kernel metric 256
# 缺少默认路由

# 检查路由器通告
$ rdisc6 eth0
# 如果无响应，说明路由器未发送RA
```

**✅ 解决方案：**
```bash
# 方案1：重启网络接口触发SLAAC
sudo ip link set eth0 down
sudo ip link set eth0 up

# 方案2：手动配置静态地址
sudo ip -6 addr add 2001:db8::100/64 dev eth0
sudo ip -6 route add default via 2001:db8::1

# 方案3：重启网络服务
sudo systemctl restart networking
```

## 3. 🌐 双栈解析问题排查


### 3.1 DNS解析优先级问题


**🔄 双栈DNS解析机制：**

现代操作系统在双栈环境中会同时查询A记录（IPv4）和AAAA记录（IPv6），但存在优先级选择问题。

```
DNS查询流程：
客户端发起查询
     ↓
┌─── A记录查询 (IPv4)
└─── AAAA记录查询 (IPv6)
     ↓
系统根据优先级选择
     ↓
┌─── IPv6优先 (RFC 6724)
└─── IPv4回退机制
```

### 3.2 解析问题诊断方法


**🔍 诊断DNS解析：**

```bash
# 分别测试IPv4和IPv6解析
nslookup -type=A www.example.com      # IPv4解析
nslookup -type=AAAA www.example.com   # IPv6解析

# 使用dig更详细的信息
dig www.example.com A     # IPv4记录
dig www.example.com AAAA  # IPv6记录

# 测试实际连接行为
curl -4 http://www.example.com/       # 强制IPv4
curl -6 http://www.example.com/       # 强制IPv6
curl -v http://www.example.com/       # 查看选择的协议
```

**📊 解析问题分类：**

| 问题类型 | 现象 | 原因 | 解决方案 |
|---------|------|------|----------|
| **AAAA记录缺失** | IPv6连接失败 | DNS服务器未配置AAAA | 配置AAAA记录 |
| **IPv6优先但不可达** | 连接超时 | IPv6路径有问题 | 调整优先级 |
| **DNS64问题** | 部分服务不可达 | DNS64配置错误 | 检查DNS64设置 |

### 3.3 解析优先级调整


**⚙️ 系统优先级配置：**

```bash
# 查看当前优先级配置
cat /etc/gai.conf

# 调整优先级（优先IPv4）
echo "precedence ::ffff:0:0/96  100" >> /etc/gai.conf

# 临时禁用IPv6（不推荐）
echo 1 > /proc/sys/net/ipv6/conf/all/disable_ipv6
```

> ⚠️ **注意事项**
> 修改优先级可能影响系统全局网络行为，建议在测试环境验证

**📱 应用层解决方案：**

```bash
# 浏览器测试
firefox --new-instance -p ipv4only  # 禁用IPv6的配置文件

# 应用程序配置
export CURL_CA_BUNDLE=""
export CURLOPT_IPRESOLVE=CURL_IPRESOLVE_V4  # curl使用IPv4
```

## 4. 🛣️ 路由不可达问题


### 4.1 IPv6路由表诊断


**🗺️ 路由表分析：**

```bash
# 查看完整IPv6路由表
ip -6 route show table all

# 查看默认路由
ip -6 route show default

# 查看特定目的地路由
ip -6 route get 2001:db8::100
```

**路由表读取示例：**
```
2001:db8::/64 dev eth0 proto kernel metric 256 pref medium
default via fe80::1 dev eth0 proto ra metric 1024 pref medium

解释：
├─ 目的网络: 2001:db8::/64
├─ 出接口: eth0  
├─ 协议: kernel (内核自动生成)
├─ 度量值: 256 (越小优先级越高)
└─ 优先级: medium
```

### 4.2 路由不可达类型


**🚫 常见路由问题：**

```
路由问题分类：

缺少默认路由 ━━━┳━━━ 路由器未发送RA
                ┣━━━ DHCPv6未获取路由
                ┗━━━ 手动配置错误

路由环路 ━━━━━━┳━━━ 多个网关配置
                ┣━━━ 路由协议错误
                ┗━━━ 静态路由冲突

MTU问题 ━━━━━━┳━━━ 路径MTU发现失败
                ┣━━━ 隧道MTU设置错误
                ┗━━━ 分片被阻止
```

### 4.3 路由故障诊断实例


**❌ 故障案例：远程网络不可达**

```bash
# 现象：无法访问远程IPv6网站
$ ping6 2001:4860:4860::8888
connect: Network is unreachable

# 诊断步骤1：检查本地路由
$ ip -6 route show
2001:db8::/64 dev eth0 proto kernel metric 256
fe80::/64 dev eth0 proto kernel metric 256
# 缺少默认路由！

# 诊断步骤2：检查网关发现
$ rdisc6 eth0
Soliciting ff02::2 (ff02::2) on eth0...
# 超时无响应

# 诊断步骤3：检查邻居发现
$ ip -6 neigh show
# 应该能看到网关的邻居条目
```

**✅ 解决方案：**

```bash
# 方案1：手动添加默认路由
sudo ip -6 route add default via fe80::1 dev eth0

# 方案2：重新获取路由器通告
sudo sysctl -w net.ipv6.conf.eth0.accept_ra=1
sudo ip -6 route flush dev eth0
echo 1 > /proc/sys/net/ipv6/conf/eth0/accept_ra

# 方案3：使用DHCPv6获取路由
sudo dhclient -6 eth0
```

### 4.4 路径MTU问题


**📏 MTU问题诊断：**

IPv6要求最小MTU为1280字节，路径MTU发现对IPv6更加重要。

```bash
# 测试路径MTU
ping6 -s 1450 2001:4860:4860::8888  # 可能失败
ping6 -s 1280 2001:4860:4860::8888  # 应该成功

# 查看接口MTU
ip link show eth0
# mtu 1500

# 手动设置MTU
sudo ip link set eth0 mtu 1280
```

**🔧 MTU问题解决：**

| 问题 | 现象 | 解决方案 |
|------|------|----------|
| **隧道MTU过大** | 大包丢失 | 调整隧道MTU |
| **路径MTU发现失败** | 连接建立后断开 | 允许ICMPv6通过防火墙 |
| **应用MTU配置错误** | 部分数据传输失败 | 调整应用MTU设置 |

## 5. 🌐 IPv6服务访问异常


### 5.1 服务绑定问题


**🔌 服务绑定诊断：**

很多服务可能只绑定IPv4，导致IPv6访问失败。

```bash
# 检查服务监听状态
ss -6 -tuln

# 示例输出分析：
tcp6  LISTEN  0  128  :::80     :::*     # 正确：监听所有IPv6
tcp6  LISTEN  0  128  ::1:22   :::*     # 问题：只监听本地
tcp   LISTEN  0  128  0.0.0.0:443  *:*  # 问题：只监听IPv4
```

**常见服务配置检查：**

```bash
# Apache HTTP服务器
grep -i "listen" /etc/apache2/sites-enabled/*
# 应该看到: Listen [::]:80

# Nginx
grep -i "listen" /etc/nginx/sites-enabled/*
# 应该看到: listen [::]:80 ipv6only=on;

# SSH服务
grep -i "addressfamily\|listenaddress" /etc/ssh/sshd_config
# AddressFamily any 或 inet6
```

### 5.2 防火墙配置问题


**🛡️ IPv6防火墙检查：**

```bash
# 检查ip6tables规则
sudo ip6tables -L -v -n

# 常见防火墙问题：
# 1. 完全阻止IPv6流量
# 2. 阻止ICMPv6（影响邻居发现）
# 3. 端口转发未配置IPv6

# 临时允许所有IPv6流量（测试用）
sudo ip6tables -P INPUT ACCEPT
sudo ip6tables -P FORWARD ACCEPT
sudo ip6tables -P OUTPUT ACCEPT
```

> ⚠️ **安全提醒**
> 生产环境不要使用全部ACCEPT策略，应该配置具体的允许规则

### 5.3 应用程序兼容性


**🔧 应用程序IPv6支持检查：**

```bash
# 测试应用程序IPv6支持
curl -6 -v http://[::1]:8080/         # 本地测试
telnet -6 ::1 22                      # 端口连接测试
nc -6 -v ::1 80                       # netcat测试

# Java应用程序IPv6配置
java -Djava.net.preferIPv6Addresses=true MyApp

# Python应用程序
python3 -c "import socket; print(socket.has_ipv6)"  # 检查IPv6支持
```

**📋 应用程序配置指南：**

| 应用类型 | 配置要点 | 示例配置 |
|---------|----------|----------|
| **Web服务器** | 监听IPv6地址 | `Listen [::]:80` |
| **数据库** | 允许IPv6连接 | `bind_address = ::` |
| **代理服务** | 双栈代理配置 | `proxy_bind [::]:8080` |

## 6. 🔧 地址配置冲突解决


### 6.1 地址冲突检测


**⚠️ IPv6地址冲突类型：**

IPv6虽然地址空间巨大，但仍可能发生地址冲突，特别是在以下情况：

```
地址冲突场景：
├─ 重复地址检测(DAD)失败
├─ 静态地址配置错误
├─ SLAAC与DHCPv6冲突
├─ 临时地址池冲突
└─ VPN隧道地址重叠
```

**🔍 冲突检测方法：**

```bash
# 查看重复地址检测状态
ip -6 addr show | grep -E "(tentative|dadfailed)"

# 手动触发DAD检测
echo 1 > /proc/sys/net/ipv6/conf/eth0/dad_transmits

# 监控邻居发现冲突
tcpdump -i eth0 icmp6 and icmp6[0] == 135  # NS消息
tcpdump -i eth0 icmp6 and icmp6[0] == 136  # NA消息
```

### 6.2 SLAAC与DHCPv6共存


**🔄 自动配置冲突解决：**

当SLAAC和DHCPv6同时启用时，可能产生多个地址和配置冲突。

```bash
# 查看地址获取方式
ip -6 addr show eth0 | grep -E "(global|temporary)"

# 示例输出分析：
inet6 2001:db8::100/64 scope global     # 静态配置
inet6 2001:db8::a00:27ff:fe4e:66a1/64 scope global dynamic  # SLAAC
inet6 2001:db8::200/64 scope global     # DHCPv6
```

**⚙️ 配置策略调整：**

```bash
# 禁用SLAAC，只使用DHCPv6
echo 0 > /proc/sys/net/ipv6/conf/eth0/autoconf
echo 0 > /proc/sys/net/ipv6/conf/eth0/accept_ra

# 禁用DHCPv6，只使用SLAAC
echo 1 > /proc/sys/net/ipv6/conf/eth0/autoconf
echo 1 > /proc/sys/net/ipv6/conf/eth0/accept_ra
killall dhclient
```

### 6.3 临时地址管理


**🔄 隐私扩展地址问题：**

IPv6隐私扩展会生成临时地址，可能导致连接问题。

```bash
# 查看隐私扩展配置
cat /proc/sys/net/ipv6/conf/eth0/use_tempaddr
# 0: 禁用
# 1: 启用但优先使用永久地址  
# 2: 启用且优先使用临时地址

# 调整临时地址策略
echo 0 > /proc/sys/net/ipv6/conf/eth0/use_tempaddr  # 禁用
echo 1 > /proc/sys/net/ipv6/conf/eth0/use_tempaddr  # 保守启用
```

**📊 地址选择策略：**

```
地址选择算法 (RFC 6724):
1. 避免不可用地址
2. 优先匹配作用域
3. 避免已弃用地址  
4. 优先本地地址
5. 优先匹配标签
6. 优先更高优先级
7. 优先临时地址(如启用)
8. 优先更长匹配前缀
```

## 7. 🚇 隧道连接故障


### 7.1 IPv6隧道类型与问题


**🔧 常见隧道技术：**

| 隧道类型 | 用途 | 常见问题 | 诊断方法 |
|---------|------|----------|----------|
| **6in4** | 静态隧道 | 配置错误、防火墙阻止 | `ping6 隧道端点` |
| **6to4** | 自动隧道 | 中继路由器问题 | `traceroute6 2002::` |
| **Teredo** | NAT穿越 | NAT类型不兼容 | `teredo-client -v` |
| **6rd** | ISP部署 | 边界路由器配置 | 检查ISP设置 |

### 7.2 6in4隧道故障诊断


**🔍 6in4隧道检查流程：**

```bash
# 检查隧道接口状态
ip link show sit0
ip -6 addr show sit0

# 测试隧道端点连通性
ping -c 3 隧道服务器IPv4地址  # 底层IPv4连通性
ping6 -c 3 隧道服务器IPv6地址  # 隧道IPv6连通性

# 检查隧道配置
ip tunnel show sit0
# 应该显示：remote <服务器IP> local <本地IP>

# 手动重建隧道
sudo ip tunnel del sit0
sudo ip tunnel add sit0 mode sit remote <服务器IP> local <本地IP>
sudo ip link set sit0 up
sudo ip -6 addr add <分配的IPv6地址>/64 dev sit0
sudo ip -6 route add default dev sit0
```

### 7.3 隧道性能问题


**📈 性能诊断与优化：**

```bash
# 测试隧道带宽
iperf3 -6 -c <IPv6服务器>
iperf3 -4 -c <IPv4服务器>  # 对比

# 检查隧道MTU
ping6 -s 1450 <目标>  # 测试大包传输
ip link set sit0 mtu 1280  # 调整MTU

# 监控隧道流量
tcpdump -i sit0 -v  # 隧道接口
tcpdump -i eth0 proto 41  # 底层IPv4接口上的41号协议
```

**⚡ 隧道优化建议：**

```
性能优化策略：
├─ MTU优化
│  ├─ 设置适当的隧道MTU
│  └─ 启用路径MTU发现
├─ 多隧道负载均衡
│  ├─ 配置多个隧道服务器
│  └─ 实现故障切换
└─ 本地IPv6优化
   ├─ 启用本地IPv6服务
   └─ 减少隧道依赖
```

### 8.1 IPv6性能测试


**📊 性能测试工具集：**

```bash
# 基础连通性和延迟
```

### 8.2 性能瓶颈识别


**🔍 常见性能问题：**

```
性能问题分类：

解析延迟 ━━━━━━┳━━━ DNS服务器慢
                ┣━━━ 双栈解析冲突
                ┗━━━ DNS64处理延迟
```

### 9.1 典型错误码解析


**❌ 常见错误码分析：**

| 错误信息 | 含义 | 可能原因 | 解决方案 |
|---------|------|----------|----------|
| `Network unreachable` | 网络不可达 | 缺少路由 | 添加默认路由 |

| `No route to host` | 主机不可达 | 防火墙/路由问题 | 检查防火墙规则 |
