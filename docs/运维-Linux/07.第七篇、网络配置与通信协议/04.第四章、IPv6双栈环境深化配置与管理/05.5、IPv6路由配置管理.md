## 1. 🌐 IPv6路由基础概念


### 1.1 什么是IPv6路由


> **💡 核心理解**
> IPv6路由就像是网络世界的"导航系统"。当你的数据包需要从一个地方到达另一个地方时，路由器就是那个"导航员"，告诉数据包应该走哪条路。

**📋 路由的本质作用**：
- **寻路功能**：为数据包找到到达目标的最佳路径
- **转发决策**：决定数据包下一步该发送到哪里
- **网络连通**：让不同网段的设备能够相互通信

### 1.2 IPv6路由vs IPv4路由的区别


```
IPv4路由示例：
目标网络: 192.168.1.0/24
下一跳: 10.0.0.1
接口: eth0

IPv6路由示例：  
目标网络: 2001:db8::/32
下一跳: fe80::1%eth0
接口: eth0
```

**🔍 主要区别**：

| 特性 | **IPv4** | **IPv6** |
|------|----------|----------|
| **地址长度** | `32位` | `128位` |
| **路由表大小** | `相对较小` | `可能非常大` |
| **链路本地地址** | `无` | `fe80::/10` |
| **自动配置** | `需要DHCP` | `支持无状态自动配置` |

### 1.3 IPv6路由的工作原理


```
数据包路由流程：

[源主机] → [路由器A] → [路由器B] → [目标主机]
   ↓           ↓           ↓           ↓
查看目标IP → 查路由表 → 查路由表 → 接收数据
            ↓           ↓
        转发到下一跳  转发到下一跳
```

> **🔄 路由决策过程**
> 1. **检查目标地址**：数据包要去哪里？
> 2. **查询路由表**：有没有到达目标的路径？
> 3. **选择最佳路由**：如果有多条路径，选哪一条？
> 4. **转发数据包**：把数据包发送给下一跳

## 2. ⚙️ IPv6静态路由配置


### 2.1 什么是静态路由


> **💡 核心理解**
> 静态路由就像在地图上手工画出的固定路线。管理员人工告诉路由器："去这个地方，就走这条路"。一旦设置好，除非管理员手动修改，否则路径不会改变。

**📌 静态路由的特点**：
- **手动配置**：需要管理员逐条添加
- **路径固定**：不会自动调整
- **配置简单**：适合小型网络
- **资源占用少**：不需要路由协议开销

### 2.2 基本静态路由配置


**🔧 命令格式**：
```bash
# 基本语法
ip -6 route add <目标网络> via <下一跳地址> dev <接口名>

# 或者使用route命令
route -A inet6 add <目标网络> gw <下一跳地址> dev <接口名>
```

**💻 实际配置示例**：

```bash
# 添加到特定网络的路由
ip -6 route add 2001:db8:1::/64 via 2001:db8::1 dev eth0

# 添加到远程网络的路由
ip -6 route add 2001:db8:2::/48 via fe80::1%eth0

# 使用接口作为下一跳（点对点链路）
ip -6 route add 2001:db8:3::/64 dev ppp0
```

### 2.3 详细配置步骤解析


**步骤1：确定网络拓扑**
```
本地网络: 2001:db8:100::/64
目标网络: 2001:db8:200::/64
网关地址: 2001:db8:100::1
```

**步骤2：执行配置命令**
```bash
# 方法1：使用全局IPv6地址作为下一跳
ip -6 route add 2001:db8:200::/64 via 2001:db8:100::1

# 方法2：使用链路本地地址（推荐）
ip -6 route add 2001:db8:200::/64 via fe80::1%eth0
```

> **⚠️ 重要提醒**
> 使用链路本地地址时，必须指定出接口（用%符号），因为同一个链路本地地址可能在多个接口上存在。

**步骤3：验证配置**
```bash
# 查看路由是否添加成功
ip -6 route show | grep 2001:db8:200

# 测试连通性
ping6 2001:db8:200::100
```

### 2.4 常见配置场景


**🏢 企业网络场景**：
```bash
# 总部网络访问分支机构
ip -6 route add 2001:db8:branch::/48 via 2001:db8:main::1

# 分支机构访问总部
ip -6 route add 2001:db8:main::/48 via 2001:db8:branch::1

# 访问DMZ区域
ip -6 route add 2001:db8:dmz::/64 via 2001:db8:internal::254
```

**🏠 家庭网络场景**：
```bash
# 内网设备访问IPv6互联网（通过路由器）
ip -6 route add ::/0 via fe80::1%eth0

# 访问邻居网络（如果有直连）
ip -6 route add 2001:db8:neighbor::/64 via fe80::neighbor%eth1
```

## 3. 🎯 默认路由设置与管理


### 3.1 什么是默认路由


> **💡 核心理解**
> 默认路由就像是"万能钥匙"。当路由器不知道某个目标地址该怎么走时，就会说："不管了，全部走默认路由！"这通常指向上级路由器或互联网网关。

**🔑 默认路由的作用**：
- **兜底机制**：处理路由表中没有明确路径的目标
- **简化配置**：不需要为每个可能的目标配置路由
- **互联网接入**：通常指向ISP提供的网关

### 3.2 默认路由配置方法


**📝 基本配置语法**：
```bash
# IPv6默认路由配置
ip -6 route add default via <网关地址> dev <接口名>

# 或者使用::/0表示默认路由
ip -6 route add ::/0 via <网关地址> dev <接口名>
```

**💻 实际配置示例**：

```bash
# 方法1：使用全局IPv6地址
ip -6 route add default via 2001:db8::1 dev eth0

# 方法2：使用链路本地地址（常用）
ip -6 route add default via fe80::1%eth0

# 方法3：直接指定接口（PPP连接）
ip -6 route add default dev ppp0
```

### 3.3 多默认路由与负载均衡


> **🔍 深入思考**
> 有时候我们可能有多个互联网出口，这时候可以配置多个默认路由来实现负载均衡或故障切换。

```bash
# 配置多个默认路由
ip -6 route add default via 2001:db8::1 dev eth0 metric 100
ip -6 route add default via 2001:db8::2 dev eth1 metric 200

# 第一条路由优先级更高（metric值越小优先级越高）
```

**📊 路由优先级对比**：

| Metric值 | **优先级** | **使用场景** |
|----------|------------|--------------|
| `100` | `最高` | `主要出口` |
| `200` | `次高` | `备用出口` |
| `300` | `较低` | `应急出口` |

### 3.4 默认路由的管理操作


**🔄 常用管理命令**：

```bash
# 查看当前默认路由
ip -6 route show default

# 删除默认路由
ip -6 route del default via 2001:db8::1 dev eth0

# 修改默认路由（先删除再添加）
ip -6 route del default
ip -6 route add default via fe80::new_gateway%eth0
```

**📋 自动配置检查**：
```bash
# 检查是否启用了路由器发现
cat /proc/sys/net/ipv6/conf/all/accept_ra

# 如果值为1，系统会自动从路由器广告中学习默认路由
# 如果值为0，需要手动配置
```

## 4. 🛤️ 多路径路由配置


### 4.1 什么是多路径路由


> **💡 核心理解**
> 多路径路由就像城市里有多条路可以到达同一个目的地。系统可以同时使用这些路径，实现负载分担，或者在某条路堵塞时自动切换到其他路径。

**🎯 多路径路由的优势**：
- **负载均衡**：流量分散到多条路径上
- **提高带宽**：多条路径的带宽可以叠加
- **增强可靠性**：一条路径故障时其他路径继续工作
- **优化性能**：根据网络状况选择最佳路径

### 4.2 ECMP（等价多路径）配置


**🔧 基本配置方法**：
```bash
# 添加多条等价路由到同一目标
ip -6 route add 2001:db8:target::/64 \
    nexthop via 2001:db8::1 dev eth0 weight 1 \
    nexthop via 2001:db8::2 dev eth1 weight 1
```

**📊 详细配置示例**：

```bash
# 场景：有两个上游ISP，希望负载均衡
# ISP1: 2001:db8:isp1::1 (eth0接口)
# ISP2: 2001:db8:isp2::1 (eth1接口)

# 配置等权重的双路径默认路由
ip -6 route add default \
    nexthop via 2001:db8:isp1::1 dev eth0 weight 1 \
    nexthop via 2001:db8:isp2::1 dev eth1 weight 1
```

### 4.3 不等权重多路径配置


**⚖️ 权重设置原理**：
```bash
# 配置不同权重的多路径
ip -6 route add 2001:db8:remote::/48 \
    nexthop via 2001:db8::1 dev eth0 weight 3 \
    nexthop via 2001:db8::2 dev eth1 weight 1

# 结果：75%流量走eth0，25%流量走eth1
```

> **📊 权重计算说明**
> - 总权重 = 3 + 1 = 4
> - eth0占比 = 3/4 = 75%
> - eth1占比 = 1/4 = 25%

**💻 实际应用场景**：

```bash
# 场景1：主备链路配置
ip -6 route add default \
    nexthop via fe80::primary%eth0 weight 10 \
    nexthop via fe80::backup%eth1 weight 1

# 场景2：按带宽比例分配
# 假设eth0是1Gbps，eth1是100Mbps
ip -6 route add 2001:db8:servers::/64 \
    nexthop via 2001:db8::gw1 dev eth0 weight 10 \
    nexthop via 2001:db8::gw2 dev eth1 weight 1
```

### 4.4 多路径路由管理


**🔍 查看多路径配置**：
```bash
# 详细查看路由信息
ip -6 route show 2001:db8:target::/64

# 输出示例：
# 2001:db8:target::/64 
#     nexthop via 2001:db8::1 dev eth0 weight 1
#     nexthop via 2001:db8::2 dev eth1 weight 1
```

**🛠️ 修改多路径配置**：
```bash
# 删除原有路由
ip -6 route del 2001:db8:target::/64

# 重新配置新的多路径
ip -6 route add 2001:db8:target::/64 \
    nexthop via 2001:db8::1 dev eth0 weight 2 \
    nexthop via 2001:db8::2 dev eth1 weight 1 \
    nexthop via 2001:db8::3 dev eth2 weight 1
```

## 5. 🔍 路由表查看与管理


### 5.1 路由表的基本概念


> **💡 核心理解**
> 路由表就像是一本"地址簿"，记录着"要去某个地方，应该走哪条路"。每个路由器都维护着自己的路由表，用来做转发决策。

**📋 路由表包含的信息**：
- **目标网络**：要到达的网络地址和掩码
- **下一跳**：数据包下一步应该发给谁
- **出接口**：从哪个网络接口发出
- **度量值**：路径的"代价"，用于选择最佳路径
- **路由来源**：这条路由是怎么来的（静态、动态等）

### 5.2 路由表查看命令


**🔧 基本查看命令**：

```bash
# 查看完整的IPv6路由表
ip -6 route show

# 只看默认路由
ip -6 route show default

# 查看特定网络的路由
ip -6 route show 2001:db8::/32

# 使用传统route命令查看
route -A inet6 -n
```

**📊 路由表输出解析**：
```
2001:db8:100::/64 dev eth0 proto kernel scope link src 2001:db8:100::10 metric 256
fe80::/64 dev eth0 proto kernel scope link metric 256
default via fe80::1 dev eth0 proto ra metric 1024
```

> **🔍 字段含义解释**
> - **proto kernel**: 内核自动生成的路由
> - **proto ra**: 从路由器广告学到的路由  
> - **scope link**: 链路范围内有效
> - **metric**: 路由优先级（数值越小优先级越高）

### 5.3 详细路由信息查看


**📖 获取更详细信息**：

```bash
# 查看路由的详细信息
ip -6 route show table all

# 查看特定路由的详细信息
ip -6 route get 2001:db8:remote::100

# 输出示例：
# 2001:db8:remote::100 from :: via fe80::1 dev eth0 src 2001:db8:100::10 metric 1024
```

**🔎 路由来源标识**：

| **Proto标识** | **含义** | **说明** |
|---------------|----------|----------|
| `kernel` | `内核路由` | `系统自动生成` |
| `static` | `静态路由` | `手动配置` |
| `ra` | `路由器广告` | `自动发现` |
| `dhcp` | `DHCP获得` | `动态分配` |

### 5.4 路由表管理操作


**🛠️ 路由添加与删除**：

```bash
# 添加静态路由
ip -6 route add 2001:db8:new::/64 via 2001:db8::1 dev eth0

# 删除特定路由
ip -6 route del 2001:db8:new::/64

# 批量删除某个接口的所有路由
ip -6 route flush dev eth1

# 刷新整个路由表（危险操作！）
ip -6 route flush table main
```

**⚠️ 管理注意事项**：

> **🚨 重要警告**
> - 删除默认路由可能导致网络中断
> - 刷新路由表会清除所有路由，包括本地路由
> - 在远程操作时要特别小心，避免失去连接

**📋 安全管理实践**：
```bash
# 1. 操作前备份当前路由表
ip -6 route show > /tmp/ipv6_routes_backup.txt

# 2. 使用at命令设置自动恢复（5分钟后执行）
echo "ip -6 route flush table main; cat /tmp/ipv6_routes_backup.txt | while read route; do ip -6 route add \$route; done" | at now + 5 minutes

# 3. 然后进行路由修改操作
ip -6 route add 2001:db8:test::/64 via 2001:db8::1

# 4. 确认无问题后，取消自动恢复
atrm [job_number]
```

## 6. ⚖️ 路由优先级与策略


### 6.1 路由优先级机制


> **💡 核心理解**
> 路由优先级就像交通规则中的"优先通行权"。当有多条路可以到达同一目标时，系统会根据优先级规则选择"最佳"路径。在Linux中，这个优先级主要由metric值决定。

**🎯 优先级决策因素**：
1. **路由类型**：直连路由 > 静态路由 > 动态路由
2. **Metric值**：数值越小优先级越高
3. **管理距离**：不同协议有不同的可信度
4. **路径特性**：带宽、延迟、可靠性等

### 6.2 Metric值配置与管理


**📊 Metric值的作用原理**：

```bash
# 配置不同metric值的路由
ip -6 route add 2001:db8:target::/64 via 2001:db8::1 dev eth0 metric 100
ip -6 route add 2001:db8:target::/64 via 2001:db8::2 dev eth1 metric 200

# 系统会优先选择metric=100的路径
```

**🔢 常用Metric值范围**：

| **Metric范围** | **用途** | **典型应用** |
|----------------|----------|--------------|
| `1-100` | `高优先级路由` | `主要链路、核心路径` |
| `100-500` | `正常优先级` | `一般静态路由` |
| `500-1000` | `较低优先级` | `备用链路` |
| `1000+` | `最低优先级` | `应急路径、自动路由` |

**💻 实际配置示例**：

```bash
# 主备链路配置
# 主链路（高优先级）
ip -6 route add default via 2001:db8:main::1 dev eth0 metric 50

# 备份链路（低优先级）  
ip -6 route add default via 2001:db8:backup::1 dev eth1 metric 100

# 应急链路（最低优先级）
ip -6 route add default via 2001:db8:emergency::1 dev eth2 metric 200
```

### 6.3 高级路由策略配置


**🎛️ 基于策略的路由**：

> **🔍 深入思考**
> 有时候我们不只是想要"最短路径"，还要考虑其他因素，比如某些流量走专用链路，某些用户使用特定出口等。这就需要基于策略的路由。

```bash
# 查看当前路由策略
ip -6 rule show

# 典型输出：
# 0:	from all lookup local
# 32766:	from all lookup main
# 32767:	from all lookup default
```

**🔧 自定义路由策略**：

```bash
# 创建自定义路由表
echo "200 custom_table" >> /etc/iproute2/rt_tables

# 为特定源地址配置路由策略
ip -6 rule add from 2001:db8:vip::/64 table custom_table prio 100

# 在自定义表中添加路由
ip -6 route add default via 2001:db8:special::1 dev eth0 table custom_table

# 为特定目标配置策略
ip -6 rule add to 2001:db8:server::/64 table custom_table prio 200
```

### 6.4 路由策略应用场景


**🏢 企业网络应用**：

```bash
# 场景1：VIP用户走专用链路
# VIP用户网段：2001:db8:vip::/64
# 专用出口：2001:db8:premium::1

# 配置VIP用户策略
ip -6 rule add from 2001:db8:vip::/64 table vip_table prio 100
ip -6 route add default via 2001:db8:premium::1 dev eth0 table vip_table

# 场景2：内网服务器访问走内网路由
ip -6 rule add to 2001:db8:internal::/48 table internal_table prio 50
ip -6 route add 2001:db8:internal::/48 via 2001:db8:lan::1 dev eth1 table internal_table
```

**🏠 家庭网络应用**：

```bash
# 场景：游戏流量走低延迟链路，其他流量走高带宽链路

# 为游戏服务器配置专用路由
ip -6 rule add to 2001:db8:game::/32 table game_table prio 100
ip -6 route add default via 2001:db8:lowlatency::1 dev eth0 table game_table

# 普通流量走主链路
ip -6 route add default via 2001:db8:main::1 dev eth1 metric 200
```

**📋 策略管理命令**：

```bash
# 查看所有路由策略
ip -6 rule show

# 删除特定策略  
ip -6 rule del from 2001:db8:vip::/64 table custom_table

# 修改策略优先级
ip -6 rule del from 2001:db8:test::/64 table test_table
ip -6 rule add from 2001:db8:test::/64 table test_table prio 50
```

## 7. 📊 路由聚合与汇总


### 7.1 什么是路由聚合


> **💡 核心理解**
> 路由聚合就像整理书架一样。原本书架上有很多本书（具体路由），现在我们用一个书箱（聚合路由）来代表这一堆书。这样查找时只需要找到书箱，而不用逐本查找，大大提高了效率。

**🎯 路由聚合的作用**：
- **减少路由表大小**：多条路由合并为一条
- **提高查找效率**：路由表越小，查找越快
- **节省内存**：减少路由器的内存占用
- **简化管理**：减少配置复杂性

### 7.2 IPv6地址聚合原理


**📐 聚合的数学原理**：

```
原始路由（需要4条）：
2001:db8:1000::/64  →  2001:db8:1000:0000::/64
2001:db8:1001::/64  →  2001:db8:1001:0000::/64  
2001:db8:1002::/64  →  2001:db8:1002:0000::/64
2001:db8:1003::/64  →  2001:db8:1003:0000::/64

聚合后路由（只需1条）：
2001:db8:1000::/62  →  覆盖上述4个网段
```

> **🧮 聚合计算方法**
> 1. 找出所有路由的公共前缀
> 2. 确定最长的公共前缀长度
> 3. 新的掩码长度 = 公共前缀长度

**📊 聚合示例详解**：

```
地址二进制分析：
2001:db8:1000:: = ...1000 0000 0000 0000
2001:db8:1001:: = ...1000 0000 0000 0001  
2001:db8:1002:: = ...1000 0000 0000 0010
2001:db8:1003:: = ...1000 0000 0000 0011

公共前缀：...1000 0000 0000 00（前62位相同）
聚合路由：2001:db8:1000::/62
```

### 7.3 路由聚合配置实践


**🔧 基本聚合配置**：

```bash
# 删除原有的具体路由
ip -6 route del 2001:db8:1000::/64 via 2001:db8::1
ip -6 route del 2001:db8:1001::/64 via 2001:db8::1  
ip -6 route del 2001:db8:1002::/64 via 2001:db8::1
ip -6 route del 2001:db8:1003::/64 via 2001:db8::1

# 添加聚合路由
ip -6 route add 2001:db8:1000::/62 via 2001:db8::1 dev eth0
```

**💻 实际应用场景**：

```bash
# 场景：企业分支机构网络聚合
# 分支机构有多个部门网段：
# - 销售部：2001:db8:branch:100::/64
# - 技术部：2001:db8:branch:101::/64  
# - 行政部：2001:db8:branch:102::/64
# - 财务部：2001:db8:branch:103::/64

# 在总部路由器上配置聚合路由
ip -6 route add 2001:db8:branch:100::/62 via 2001:db8:wan::branch_gw

# 这一条路由就覆盖了分支机构的所有部门
```

### 7.4 聚合配置的注意事项


**⚠️ 聚合配置陷阱**：

> **🚨 黑洞路由风险**
> 聚合时要确保聚合范围内的所有网段都真实存在且可达，否则可能产生"黑洞"，导致某些流量丢失。

```bash
# 错误示例：
# 假设只有这些网段真实存在：
# 2001:db8:1000::/64, 2001:db8:1002::/64
# 但配置了聚合路由：
ip -6 route add 2001:db8:1000::/62 via 2001:db8::1

# 问题：发往2001:db8:1001::/64的流量会被错误转发
# 解决：使用更精确的聚合或添加拒绝路由
ip -6 route add 2001:db8:1001::/64 via ::1 dev lo  # 黑洞路由
```

**✅ 最佳聚合实践**：

```bash
# 1. 分层聚合策略
# 区域级聚合
ip -6 route add 2001:db8:asia::/48 via 2001:db8::asia_gw

# 国家级聚合  
ip -6 route add 2001:db8:asia:cn::/56 via 2001:db8::china_gw

# 城市级聚合
ip -6 route add 2001:db8:asia:cn:bj::/60 via 2001:db8::beijing_gw

# 2. 验证聚合效果
# 聚合前路由表大小
ip -6 route show | wc -l

# 聚合后路由表大小  
ip -6 route show | wc -l

# 3. 测试连通性
ping6 2001:db8:asia:cn:bj:1::100  # 测试聚合路由是否正常工作
```

## 8. 📋 核心要点总结


### 8.1 必须掌握的基本概念


```
🔸 IPv6路由基础：数据包转发的"导航系统"，指引流量到达目标
🔸 静态路由：手工配置的固定路径，适合小型稳定网络
🔸 默认路由：处理未知目标的"万能出口"，通常指向上级网关  
🔸 多路径路由：利用多条路径实现负载均衡和冗余备份
🔸 路由优先级：metric值决定路径选择，数值越小优先级越高
🔸 路由聚合：将多条具体路由合并为一条，提高效率
```

### 8.2 关键理解要点


**🔹 IPv6路由的特殊性**
```
与IPv4相比的差异：
• 地址空间更大：128位地址需要更大的路由表
• 链路本地地址：fe80::/10用于邻居发现和路由
• 无状态自动配置：可以自动获得默认路由
• 更好的层次化：地址结构更适合聚合
```

**🔹 配置策略选择**
```
配置方法对比：
静态路由：
✅ 配置简单，资源占用少
❌ 需要手动维护，不适应网络变化

多路径路由：  
✅ 提供冗余和负载均衡
❌ 配置相对复杂

路由聚合：
✅ 显著减少路由表大小
❌ 可能产生黑洞路由风险
```

**🔹 实际应用指导**
```
选择原则：
🎯 小型网络：使用静态路由 + 默认路由
🎯 中型网络：静态路由 + 适度聚合
🎯 大型网络：动态路由协议（OSPFv3、BGP4+）
🎯 多出口环境：配置多路径路由和策略路由
```

### 8.3 实践操作要点


**🛠️ 配置操作流程**
```
标准配置步骤：
1. 规划网络拓扑和地址分配
2. 配置基本静态路由  
3. 设置默认路由
4. 根据需要配置多路径
5. 验证路由可达性
6. 考虑聚合优化
```

**⚠️ 常见问题与解决**
```
典型问题：
• 链路本地地址必须指定接口：使用%接口名
• 路由冲突：检查metric值和路由表优先级
• 连通性问题：使用ping6和traceroute6排查
• 配置丢失：重启后需要持久化配置
```

**📊 性能优化建议**
```
优化策略：
🚀 合理使用路由聚合减少表大小
🚀 设置合适的metric值避免次优路径
🚀 定期清理无用路由条目
🚀 使用策略路由优化特定流量
```

### 8.4 安全与维护考虑


**🔒 安全配置**
```
安全要点：
• 避免配置过于宽泛的路由
• 定期审查路由表的合理性
• 监控异常路由变化
• 备份重要的路由配置
```

**🔄 日常维护**
```
维护任务：
📅 定期检查路由表状态
📅 验证关键路径连通性  
📅 监控路由表大小变化
📅 更新路由配置文档
```

**💡 故障排查思路**
```
排查步骤：
1. 检查本地路由表：ip -6 route show
2. 测试直连网段：ping6 邻居地址
3. 测试默认网关：ping6 网关地址  
4. 跟踪路由路径：traceroute6 目标地址
5. 检查接口状态：ip -6 addr show
```

**核心记忆口诀**：
- IPv6路由如导航，静态配置要手工
- 默认路由兜底用，多路径来做冗余
- Metric小者优先走，聚合减表提效率
- 策略路由更灵活，维护监控不可缺
