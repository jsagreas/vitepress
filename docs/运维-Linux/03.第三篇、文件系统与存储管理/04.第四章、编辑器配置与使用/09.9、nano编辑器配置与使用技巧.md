## 1. 📝 nano编辑器基本概念


### 1.1 什么是nano编辑器


**简单定义**：nano是Linux系统中一个**轻量级文本编辑器**，专为新手设计，操作简单直观。

```
nano的特点：
┌─────────────────────────────────┐
│ 简单易学：快捷键提示在屏幕底部    │
│ 轻量快速：启动速度快，占用资源少  │
│ 功能完整：支持搜索、替换、语法高亮 │
│ 适合新手：比vim更容易上手        │
└─────────────────────────────────┘
```

### 1.2 nano vs 其他编辑器


| 编辑器 | **学习难度** | **功能丰富度** | **适用场景** |
|--------|------------|--------------|-------------|
| 📝 **nano** | `极简单` | `基础够用` | `快速编辑、新手友好` |
| ⚡ **vim** | `较困难` | `功能强大` | `专业开发、高效编辑` |
| 📋 **emacs** | `复杂` | `功能最全` | `重度文本处理` |
| 🖥️ **gedit** | `简单` | `图形界面` | `桌面环境编辑` |

### 1.3 nano界面结构


```
nano编辑器界面布局：

┌─ 顶部标题栏 ─────────────────────┐
│  GNU nano 5.6.1  filename.txt   │
├─ 主编辑区域 ─────────────────────┤
│  这里是文件内容                   │
│  可以正常编辑文字                 │
│  ...                            │
├─ 状态栏 ─────────────────────────┤
│  [ line 1/10, col 1/50 ]        │
├─ 快捷键提示区 ─────────────────────┤
│ ^G Help  ^O Write Out ^W Where Is│
│ ^K Cut   ^J Justify   ^C Location│
└─────────────────────────────────┘

说明：^表示Ctrl键，M表示Alt键
```

## 2. ⚙️ 配置文件详解


### 2.1 配置文件位置与优先级


**配置文件查找顺序**：nano会按以下顺序查找配置文件

```
配置文件优先级：
1. ~/.nanorc          ← 用户个人配置（最高优先级）
2. /etc/nanorc        ← 系统全局配置
3. 内置默认配置        ← 系统默认配置

查找过程：
用户启动nano → 检查~/.nanorc → 如果不存在 → 检查/etc/nanorc → 使用默认配置
```

### 2.2 创建个人配置文件


**第一次创建配置**：

```bash
# 创建个人配置文件
nano ~/.nanorc

# 查看系统配置文件（可作为参考）
cat /etc/nanorc
```

**配置文件基本结构**：

```bash
# ~/.nanorc - nano编辑器个人配置文件
# 注释行以#开头

# 基础设置
set autoindent
set tabsize 4

# 界面设置  
set linenumbers
set mouse

# 语法高亮
include "/usr/share/nano/*.nanorc"
```

### 2.3 系统配置文件说明


**系统配置路径**：
- 🔧 **`/etc/nanorc`** - 影响所有用户的全局配置
- 📁 **`/usr/share/nano/`** - 存放语法高亮文件
- 📋 **`/etc/nanorc.d/`** - 额外配置文件目录（某些发行版）

> **💡 提示**：个人配置文件`~/.nanorc`会覆盖系统配置，建议在个人配置中进行定制。

## 3. 🛠️ 基础配置选项


### 3.1 自动缩进配置


**什么是自动缩进**：当你按回车换行时，nano会自动保持与上一行相同的缩进级别。

```bash
# 启用自动缩进
set autoindent

# 实际效果演示：
原始状态：
    function test() {
        console.log("hello");

按回车后自动效果：
    function test() {
        console.log("hello");
        |← 光标自动缩进到这个位置
```

### 3.2 制表符大小设置


**tabsize的含义**：控制Tab键显示的宽度，影响代码对齐效果。

```bash
# 设置Tab键宽度为4个空格
set tabsize 4

# 不同tabsize效果对比：
tabsize 2:      tabsize 4:      tabsize 8:
if (true) {     if (true) {     if (true) {
  return;           return;               return;
}               }               }
```

**常用配置建议**：
- 🐍 **Python开发**：`set tabsize 4`
- 🌐 **Web前端**：`set tabsize 2` 
- 📋 **一般文本**：`set tabsize 4`

### 3.3 软换行配置


**softwrap的作用**：当一行文字太长时，在屏幕边缘自动换行显示（不是真正的换行符）。

```bash
# 启用软换行
set softwrap

# 对比效果：
不启用softwrap：
这是一行很长的文字会超出屏幕边界不会自动换行需要左右滚动才能看全→

启用softwrap：
这是一行很长的文字会超出屏幕边界但是会在屏
幕边缘自动折行显示，方便阅读
```

### 3.4 其他实用基础配置


```bash
# ~/.nanorc 推荐基础配置
set autoindent          # 自动缩进
set tabsize 4          # Tab键4个空格宽度  
set softwrap           # 长行自动折行显示
set linenumbers        # 显示行号
set mouse              # 启用鼠标支持
set smooth             # 平滑滚动
set titlecolor black,yellow  # 标题栏颜色
set numbercolor cyan    # 行号颜色
```

**配置效果展示**：
```
带行号的nano界面：
  1  #!/bin/bash
  2  echo "Hello World"
  3  
  4  function greeting() {
  5      echo "Welcome to nano!"
  6  }
     ^G Help    ^O Write Out    ^W Where Is
```

## 4. 🎨 语法高亮配置


### 4.1 语法高亮原理


**什么是语法高亮**：根据文件类型，用不同颜色显示关键字、字符串、注释等，让代码更易读。

```
没有语法高亮：
function hello() {
    console.log("Hello World!");
    return true;
}

有语法高亮：
function hello() {          ← function显示为蓝色
    console.log("Hello World!");  ← 字符串显示为绿色
    return true;            ← return显示为蓝色
}                          ← 注释可能显示为灰色
```

### 4.2 include语法文件


**加载系统语法文件**：

```bash
# 加载所有系统语法高亮文件
include "/usr/share/nano/*.nanorc"

# 只加载特定语言的语法文件
include "/usr/share/nano/python.nanorc"
include "/usr/share/nano/html.nanorc"
include "/usr/share/nano/css.nanorc"
```

### 4.3 语法文件目录结构


**系统语法文件位置**：`/usr/share/nano/`目录包含各种语言的语法定义

```
/usr/share/nano/目录结构：
├── python.nanorc      ← Python语法高亮
├── html.nanorc        ← HTML语法高亮
├── css.nanorc         ← CSS语法高亮
├── javascript.nanorc  ← JavaScript语法高亮
├── c.nanorc          ← C语言语法高亮
├── bash.nanorc       ← Bash脚本语法高亮
├── php.nanorc        ← PHP语法高亮
├── xml.nanorc        ← XML语法高亮
└── ...
```

**查看可用语法文件**：

```bash
# 列出所有可用的语法高亮文件
ls /usr/share/nano/

# 查看特定语法文件内容
cat /usr/share/nano/python.nanorc
```

### 4.4 自定义语法高亮


**创建自定义语法文件**：

```bash
# 创建自定义语法文件
nano ~/.config/nano/my-syntax.nanorc

# 语法文件基本格式：
syntax "描述名称" "文件扩展名正则"
color 颜色 "匹配规则"

# 示例：自定义配置文件语法
syntax "config" "\.(conf|config|cfg)$"
color red "^#.*"           # 注释行显示红色
color blue "^[a-z_]*="     # 配置项显示蓝色
color green "".*""         # 字符串显示绿色
```

## 5. ⌨️ 快捷键定制


### 5.1 bind命令详解


**bind命令作用**：重新定义或添加快捷键绑定，让nano更符合你的使用习惯。

**基本语法**：
```bash
bind 快捷键 功能 菜单
```

**常用菜单类型**：
- `main` - 主编辑界面
- `search` - 搜索界面  
- `replace` - 替换界面
- `help` - 帮助界面

### 5.2 实用快捷键定制示例


```bash
# ~/.nanorc 快捷键定制示例

# 设置Ctrl+S为保存（类似常见编辑器）
bind ^S writeout main

# 设置Ctrl+Q为退出
bind ^Q exit main  

# 设置Ctrl+F为搜索（更直观）
bind ^F whereis main

# 设置Ctrl+H为替换
bind ^H replace main

# 设置Alt+上/下箭头快速移动
bind M-Up prevbuf main      # Alt+上箭头：上一个缓冲区
bind M-Down nextbuf main    # Alt+下箭头：下一个缓冲区
```

### 5.3 unbind取消绑定


**取消不需要的快捷键**：

```bash
# 取消默认快捷键绑定
unbind ^J main      # 取消Ctrl+J的默认功能
unbind ^T main      # 取消Ctrl+T的默认功能
unbind M-J main     # 取消Alt+J的默认功能

# 取消后可以重新绑定到其他功能
bind ^J gotoline main   # 重新绑定Ctrl+J为跳转行
```

### 5.4 查看快捷键绑定


```bash
# 在nano中查看所有快捷键
# 按 Ctrl+G 进入帮助界面，可以看到所有快捷键说明

# 查看配置文件中的绑定
grep "bind\|unbind" ~/.nanorc
```

**实用快捷键配置推荐**：

```bash
# 推荐的快捷键配置（更符合现代编辑器习惯）
bind ^S writeout main       # Ctrl+S 保存
bind ^Q exit main          # Ctrl+Q 退出  
bind ^F whereis main       # Ctrl+F 搜索
bind ^H replace main       # Ctrl+H 替换
bind ^G gotoline main      # Ctrl+G 跳转行
bind ^Z undo main          # Ctrl+Z 撤销
bind ^Y redo main          # Ctrl+Y 重做
```

## 6. 🔍 搜索与替换功能


### 6.1 基本搜索操作


**Ctrl+W搜索功能**：nano中最基本也是最常用的搜索功能

```
搜索操作步骤：
1. 按下 Ctrl+W
2. 输入要搜索的内容
3. 按回车开始搜索
4. 按 Ctrl+W 继续搜索下一个
```

**搜索过程示例**：
```
文件内容：
function getUserInfo() {
    const user = getUser();
    return user.info;
}

搜索步骤：
1. Ctrl+W → 底部出现: Search:
2. 输入 "user" → Search: user
3. 回车 → 光标跳到第一个"user"位置
4. Ctrl+W 回车 → 跳到下一个"user"位置
```

### 6.2 搜索选项与模式


**搜索时的选项**：按下Ctrl+W后，可以使用以下选项：

```bash
搜索选项快捷键：
Alt+C    # 区分大小写搜索
Alt+R    # 正则表达式搜索  
Alt+W    # 整词匹配搜索
Alt+B    # 向后搜索（反向搜索）
```

**实际应用示例**：

```
文件内容：
user = "admin"
username = "john" 
User = "guest"

普通搜索 "user"：    匹配 user, username, User
整词搜索 "user"：    只匹配 user（不匹配username）  
大小写敏感 "user"：  只匹配 user, username（不匹配User）
```

### 6.3 替换功能详解


**Ctrl+\替换操作**：查找并替换文本内容的强大功能

```
替换操作步骤：
1. 按下 Ctrl+\
2. 输入要查找的文本
3. 回车确认
4. 输入替换后的文本  
5. 回车确认
6. 选择替换选项：Y(是) N(否) A(全部) Ctrl+C(取消)
```

**替换过程演示**：

```
原始文件：
let name = "老王";
console.log("你好，老王！");  
alert("老王，欢迎回来！");

替换操作：
1. Ctrl+\ → Search to replace:
2. 输入 "老王" → Search to replace: 老王
3. 回车 → Replace with:  
4. 输入 "小李" → Replace with: 小李
5. 回车 → Replace this instance? (Y/N/A/^C)

选择A（全部替换）后的结果：
let name = "小李";
console.log("你好，小李！");
alert("小李，欢迎回来！");
```

### 6.4 正则表达式搜索替换


**启用正则表达式模式**：在搜索或替换时按 `Alt+R` 开启正则模式

**常用正则表达式示例**：

```bash
# 搜索以数字开头的行
^[0-9]

# 搜索邮箱地址
[a-zA-Z0-9]+@[a-zA-Z0-9]+\.[a-zA-Z]{2,}

# 搜索空行
^$

# 搜索HTML标签
<[^>]+>
```

**实际正则替换示例**：

```
原文件（日志格式）：
2024-01-12 10:30:25 INFO: 用户登录
2024-01-12 10:31:10 ERROR: 密码错误
2024-01-12 10:32:05 INFO: 用户注册

正则替换：
查找：^([0-9-: ]+) ([A-Z]+): (.+)$
替换：[$2] $1 - $3

结果：
[INFO] 2024-01-12 10:30:25 - 用户登录  
[ERROR] 2024-01-12 10:31:10 - 密码错误
[INFO] 2024-01-12 10:32:05 - 用户注册
```

## 7. 📁 文件操作详解


### 7.1 保存文件操作


**Ctrl+O保存功能**：将内存中的内容写入磁盘文件

```
保存操作流程：
┌─ 编辑文件 ─┐    ┌─ Ctrl+O ─┐    ┌─ 确认文件名 ─┐    ┌─ 保存成功 ─┐
│ 修改内容   │ → │ 触发保存  │ → │ 回车确认     │ → │ 写入磁盘   │
└───────────┘    └──────────┘    └─────────────┘    └───────────┘
```

**保存时的选项**：

```bash
保存提示界面：
File Name to Write: filename.txt

可用选项：
Ctrl+T    # 浏览文件系统选择位置
Alt+D     # DOS格式保存（Windows换行符）
Alt+M     # Mac格式保存  
Alt+A     # 追加到文件末尾（不覆盖）
Alt+P     # 添加到文件开头
Alt+B     # 创建备份文件
```

**保存到不同位置**：

```bash
# 保存时修改文件名或路径
原文件：/home/user/test.txt
Ctrl+O → File Name to Write: /home/user/backup/test_backup.txt
# 可以保存到不同目录或使用不同文件名
```

### 7.2 插入文件功能


**Ctrl+R插入文件**：将其他文件的内容插入到当前编辑位置

```
插入文件过程：
1. 将光标移动到要插入内容的位置
2. 按 Ctrl+R
3. 输入要插入文件的路径
4. 回车确认

实际应用场景：
合并配置文件、插入代码模板、添加通用头部等
```

**插入文件示例**：

```
当前编辑文件main.py：
#!/usr/bin/env python3

def main():
|← 光标位置
    pass

插入文件utils.py的内容：
def helper_function():
    return "helper"

执行 Ctrl+R，选择utils.py后：
#!/usr/bin/env python3

def main():
def helper_function():
    return "helper"
    pass
```

### 7.3 退出编辑器


**Ctrl+X退出功能**：安全退出nano编辑器

```
退出流程图：
Ctrl+X → 检查是否有未保存修改
         ├─ 无修改 → 直接退出
         └─ 有修改 → 询问是否保存
                   ├─ Y → 保存后退出
                   ├─ N → 不保存退出  
                   └─ Ctrl+C → 取消退出
```

**退出时的提示**：

```
有未保存修改时：
Save modified buffer? (Answering "No" will DESTROY changes.)
 Y Yes     保存并退出
 N No      丢弃修改直接退出
^C Cancel  取消退出继续编辑
```

> **⚠️ 重要提醒**：选择"N"会丢失所有未保存的修改，请谨慎操作。

## 8. ✂️ 编辑功能进阶


### 8.1 文本剪切与复制


**Ctrl+K剪切行**：剪切当前光标所在的整行

```
剪切操作演示：
原文本：
line 1
line 2  ← 光标在此行
line 3

按 Ctrl+K 后：
line 1
line 3  ← line 2被剪切，存入剪切板
```

**连续剪切多行**：

```bash
# 连续按Ctrl+K可以剪切多行
原文本：
function test() {
    console.log("line 1");  ← 光标在此
    console.log("line 2");
    console.log("line 3");
}

连续按3次Ctrl+K：
function test() {
}
# 三行console.log都被剪切
```

### 8.2 文本粘贴功能


**Ctrl+U粘贴**：将剪切板中的内容粘贴到光标位置

```
粘贴操作流程：
1. 先用Ctrl+K剪切内容
2. 移动光标到目标位置
3. 按Ctrl+U粘贴内容

移动文本示例：
原始：
section A
section B  ← 要移动的内容
section C

操作步骤：
1. 光标放在section B行，按Ctrl+K剪切
2. 光标移动到section C行后面  
3. 按Ctrl+U粘贴

结果：
section A
section C
section B  ← 成功移动
```

### 8.3 复制文本功能


**Alt+6复制**：复制当前行或选中文本（不删除原文本）

```
复制与剪切对比：
Ctrl+K (剪切)：   内容被移除 + 存入剪切板
Alt+6 (复制)：    内容保留 + 存入剪切板

复制操作示例：
原文本：
important line  ← 光标在此行

按Alt+6复制，移动光标后按Ctrl+U粘贴：
important line  ← 原始行保留
important line  ← 复制的行
```

### 8.4 文本选择与操作


**选择文本的方法**：

```bash
方法1：鼠标选择（需要启用mouse选项）
set mouse  # 在配置文件中启用鼠标

方法2：键盘选择
Alt+A     # 开始选择模式
箭头键     # 移动选择范围
Ctrl+K    # 剪切选中内容
Alt+6     # 复制选中内容

方法3：选择整行
Ctrl+6    # 标记开始位置
移动光标   # 选择范围  
Ctrl+K    # 剪切选中行
```

**选择模式演示**：

```
开始选择前：
Hello World  
This is a test
End of text

按Alt+A开始选择，移动光标选择"World This is"：
Hello [World  
This is] a test
End of text

按Ctrl+K剪切选中内容：
Hello  a test
End of text
```

## 9. 🧭 导航操作技巧


### 9.1 翻页操作


**Ctrl+Y/V翻页**：快速浏览长文件的必备技能

```
翻页快捷键：
Ctrl+Y    # 向上翻页（Page Up）
Ctrl+V    # 向下翻页（Page Down）

翻页效果：
每次翻页移动约一屏高度的内容
比逐行移动快得多
```

**翻页应用场景**：

```
长配置文件：
# 配置文件有500行
# 使用方向键逐行移动太慢
# Ctrl+V快速跳到文件中部
# Ctrl+Y快速返回上方

长代码文件：
# 查看函数定义
```

### 9.2 行首行尾移动


**Ctrl+A/E快速定位**：在当前行内快速移动光标

```
行内移动示例：
let message = "Hello World, this is a very long line of code";
|← 光标在行首

按Ctrl+E：
let message = "Hello World, this is a very long line of code";
                                                              |← 光标跳到行尾

按Ctrl+A：
let message = "Hello World, this is a very long line of code";
|← 光标跳回行首
```

**实际应用**：

```bash
# 编辑长命令行时特别有用
原光标位置：

需要在行首添加内容：
Ctrl+A → 光标跳到行首
输入：# sudo apt install package-name --some-long-option

需要在行尾添加内容：  
Ctrl+E → 光标跳到行尾
输入： && echo "安装完成"
```

### 9.3 跳转到指定行


**Alt+G跳转行**：直接跳转到指定行号，处理大文件时非常高效

```
跳转操作步骤：

2. 输入目标行号
3. 回车确认
4. 光标直接跳到指定行

跳转示例：
文件有1000行，当前在第50行
Alt+G → Enter line number:
输入 800 → 回车
光标直接跳到第800行
```
