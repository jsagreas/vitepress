## 1. 🎯 Jenkins简介与核心概念


### 1.1 什么是Jenkins


> **💡 核心理解**
> Jenkins就像工厂里的流水线管理员，帮你自动化完成从代码编写到最终部署的整个过程

**简单理解**：
```
传统开发流程：
开发者写代码 → 手动测试 → 手动打包 → 手动部署 → 祈祷不出错 😰

Jenkins自动化流程：
开发者提交代码 → Jenkins自动测试 → 自动打包 → 自动部署 → 自动通知结果 😎
```

**Jenkins的作用**：
- **自动化构建**：代码一提交，自动编译打包
- **自动化测试**：运行测试用例，发现问题
- **自动化部署**：测试通过后自动发布到服务器
- **持续监控**：实时监控项目状态，及时报告问题

### 1.2 为什么需要Jenkins


**解决的痛点**：

| 传统手动方式 | Jenkins自动化 |
|-------------|---------------|
| 🔴 **容易出错** | 🟢 **标准化流程** |
| 🔴 **重复劳动** | 🟢 **一次配置，终身受益** |
| 🔴 **效率低下** | 🟢 **24小时不间断工作** |
| 🔴 **难以追踪** | 🟢 **完整的构建历史** |

> **🔍 深入思考**
> 想象一下：如果每次做菜都要重复准备食材、调料、清洗餐具，是不是很累？Jenkins就是帮你把这些重复工作自动化的"厨房助手"

## 2. 🔧 系统要求检查


### 2.1 硬件要求


**最低配置**：
- **内存**：至少 `512MB`，推荐 `1GB+`
- **磁盘空间**：至少 `1GB`，推荐 `10GB+`
- **CPU**：无特殊要求，双核足够

**推荐配置**：
```
🏢 小团队（5-10人）：
├── 内存：2-4GB
├── 磁盘：20GB SSD
└── CPU：4核

🏭 中型团队（10-50人）：
├── 内存：8-16GB
├── 磁盘：100GB SSD
└── CPU：8核

🏗️ 大型团队（50+人）：
├── 内存：16GB+
├── 磁盘：500GB+ SSD
└── CPU：16核+
```

### 2.2 操作系统支持


**支持的操作系统**：
- ✅ **Linux**（Ubuntu、CentOS、RHEL等）
- ✅ **Windows**（Windows 10/11、Server 2016+）
- ✅ **macOS**（10.13+）
- ✅ **Docker容器**

> **⚠️ 常见误区**
> 很多新手以为Jenkins只能在Linux上运行，其实Windows也完全没问题！选择你最熟悉的系统即可

### 2.3 网络要求


**端口规划**：
```
默认端口配置：
┌─────────────────┐
│ Jenkins Web界面  │ ← 默认8080端口
├─────────────────┤
│ JNLP端口        │ ← 默认50000端口（Agent连接用）
├─────────────────┤
│ SSH端口         │ ← 默认22端口（远程部署用）
└─────────────────┘
```

**防火墙设置**：
- 开放Jenkins Web端口（默认8080）
- 如果使用Agent节点，开放50000端口
- 确保能访问代码仓库（Git、SVN等）

## 3. ☕ JDK环境配置


### 3.1 为什么需要JDK


> **💡 核心理解**
> Jenkins是用Java开发的，就像开车需要汽油一样，Jenkins运行需要Java环境

**JDK版本要求**：
- **Java 8**：最低要求，但不推荐新项目使用
- **Java 11**：长期支持版本，**强烈推荐** ⭐⭐⭐
- **Java 17**：最新长期支持版本，未来趋势

### 3.2 JDK安装步骤


**Windows系统**：

```bash
# 1. 检查是否已安装Java
java -version

# 2. 如果没有，从Oracle官网下载JDK 11
# 下载地址：https://www.oracle.com/java/technologies/downloads/

# 3. 安装后设置环境变量
# JAVA_HOME = C:\Program Files\Java\jdk-11.x.x
# PATH 添加 %JAVA_HOME%\bin
```

**Linux系统**：

```bash
# Ubuntu/Debian系统
sudo apt update
sudo apt install openjdk-11-jdk

# CentOS/RHEL系统
sudo yum install java-11-openjdk-devel

# 验证安装
java -version
javac -version
```

### 3.3 验证JDK安装


**检查命令**：
```bash
# 查看Java版本
java -version

# 期望输出类似：
openjdk version "11.0.x" 2023-xx-xx
OpenJDK Runtime Environment (build 11.0.x+xx)
OpenJDK 64-Bit Server VM (build 11.0.x+xx, mixed mode)
```

> **🔧 实践检查清单**
> - [ ] Java版本 >= 11
> - [ ] JAVA_HOME环境变量正确设置
> - [ ] PATH包含Java bin目录
> - [ ] 命令行能正常执行`java -version`

## 4. 🚀 Jenkins安装方式详解


### 4.1 WAR包部署方式（推荐新手）


**为什么选择WAR包**：
- ✅ **简单直接**：下载即用，无需复杂安装
- ✅ **跨平台**：任何支持Java的系统都能运行
- ✅ **易于管理**：升级只需替换WAR文件

**安装步骤**：

**步骤1：下载Jenkins WAR包**
```bash
# 访问官网下载最新稳定版
# https://www.jenkins.io/download/

# 或使用wget下载（Linux）
wget https://get.jenkins.io/war-stable/latest/jenkins.war
```

**步骤2：启动Jenkins**
```bash
# 基本启动命令
java -jar jenkins.war

# 自定义端口启动
java -jar jenkins.war --httpPort=9090

# 后台运行（Linux）
nohup java -jar jenkins.war > jenkins.log 2>&1 &
```

**步骤3：访问Jenkins**
```
浏览器访问：http://localhost:8080
```

> **💭 理解辅助**
> WAR包就像一个"便携版"软件，双击就能运行，不需要安装到系统里

### 4.2 Docker部署方式（推荐进阶）


**为什么选择Docker**：
- ✅ **环境隔离**：不会影响宿主机环境
- ✅ **快速部署**：一条命令启动完整环境
- ✅ **版本管理**：轻松切换不同Jenkins版本

**安装步骤**：

**步骤1：安装Docker**
```bash
# Windows/Mac：下载Docker Desktop
# Linux：使用包管理器安装
sudo apt install docker.io  # Ubuntu
sudo yum install docker      # CentOS
```

**步骤2：运行Jenkins容器**
```bash
# 拉取并运行Jenkins镜像
docker run -d \
  --name jenkins \
  -p 8080:8080 \
  -p 50000:50000 \
  -v jenkins_home:/var/jenkins_home \
  jenkins/jenkins:lts
```

**参数解释**：
```
命令详解：
├── -d：后台运行
├── --name jenkins：容器名称
├── -p 8080:8080：端口映射（宿主机:容器）
├── -p 50000:50000：Agent连接端口
└── -v jenkins_home:/var/jenkins_home：数据持久化
```

### 4.3 系统包安装方式


**Ubuntu/Debian系统**：
```bash
# 添加Jenkins仓库
curl -fsSL https://pkg.jenkins.io/debian/jenkins.io.key | sudo tee \
  /usr/share/keyrings/jenkins-keyring.asc > /dev/null

echo deb [signed-by=/usr/share/keyrings/jenkins-keyring.asc] \
  https://pkg.jenkins.io/debian binary/ | sudo tee \
  /etc/apt/sources.list.d/jenkins.list > /dev/null

# 安装Jenkins
sudo apt update
sudo apt install jenkins
```

**CentOS/RHEL系统**：
```bash
# 添加Jenkins仓库
sudo wget -O /etc/yum.repos.d/jenkins.repo \
  https://pkg.jenkins.io/redhat/jenkins.repo
sudo rpm --import https://pkg.jenkins.io/redhat/jenkins.io.key

# 安装Jenkins
sudo yum install jenkins
```

### 4.4 安装方式对比


| 安装方式 | 适用场景 | 优势 ⭐ | 劣势 ❌ |
|---------|---------|---------|---------|
| **WAR包** | 新手学习、快速测试 | 简单直接、跨平台 | 手动管理服务 |
| **Docker** | 开发环境、容器化部署 | 环境隔离、版本管理 | 需要Docker知识 |
| **系统包** | 生产环境、长期运行 | 系统服务管理、稳定 | 系统依赖、升级复杂 |

> **🎯 选择建议**
> - **新手学习**：选择WAR包，简单直接
> - **团队开发**：选择Docker，环境统一
> - **生产部署**：选择系统包，稳定可靠

## 5. 🔑 首次启动与配置


### 5.1 获取初始管理员密码


**WAR包方式**：
```bash
# Jenkins启动后，控制台会显示密码位置
# 通常在：/root/.jenkins/secrets/initialAdminPassword

# 查看密码
cat /root/.jenkins/secrets/initialAdminPassword
```

**Docker方式**：
```bash
# 查看容器日志获取密码
docker logs jenkins

# 或进入容器查看文件
docker exec jenkins cat /var/jenkins_home/secrets/initialAdminPassword
```

**系统包方式**：
```bash
# 密码文件位置
sudo cat /var/lib/jenkins/secrets/initialAdminPassword
```

### 5.2 首次访问配置


**访问流程**：
```
浏览器访问Jenkins → 输入管理员密码 → 安装推荐插件 → 创建管理员用户 → 配置Jenkins URL → 开始使用
```

**界面引导**：
```
首次访问界面：
┌─────────────────────────┐
│   解锁Jenkins           │
├─────────────────────────┤
│   [输入管理员密码框]     │
│                         │
│   密码位置提示：        │
│   /var/jenkins_home/... │
└─────────────────────────┘
```

### 5.3 插件安装选择


**推荐插件vs自定义安装**：

> **💡 核心理解**
> 推荐插件就像"装修套餐"，包含了最常用的功能；自定义安装像"单点菜品"，你想要什么就装什么

**建议选择**：
- **新手**：选择"安装推荐的插件" ✅
- **高级用户**：选择"选择插件来安装"

**推荐插件包含**：
- Git插件（代码管理）
- Pipeline插件（流水线）
- Build Timeout插件（构建超时）
- Credentials插件（凭据管理）
- Workspace Cleanup插件（工作空间清理）

### 5.4 创建管理员用户


**用户信息配置**：
```
账户设置：
├── 用户名：admin（推荐）
├── 密码：强密码（8位以上，包含字母数字特殊字符）
├── 确认密码：重复输入
├── 全名：管理员
└── 邮箱地址：your-email@example.com
```

> **⚠️ 安全提醒**
> 管理员密码一定要设置复杂密码，这是整个CI/CD系统的钥匙！

### 5.5 Jenkins URL配置


**URL设置说明**：
```
Jenkins URL配置：
┌─────────────────────────┐
│ Jenkins URL:            │
│ http://your-server:8080 │
├─────────────────────────┤
│ ✅ 保存并完成           │
└─────────────────────────┘
```

**URL配置原则**：
- **本地开发**：`http://localhost:8080`
- **团队使用**：`http://服务器IP:8080`
- **生产环境**：`https://jenkins.company.com`
