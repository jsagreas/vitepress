## 1. 📝 引用类标签


### 1.1 `<blockquote>` 块级引用


**🔸 基本语法**
```html
<blockquote cite="https://example.com/source">
    <p>这是一段来自外部的引用内容。</p>
</blockquote>
```

**📋 核心特性**
```
元素类型：块级元素
语义含义：大段落引用内容
默认样式：左右缩进，上下边距
cite属性：引用来源URL（可选）
```

**✅ 推荐用法**
```html
<!-- 文章引用 -->
<blockquote cite="https://www.w3.org/standards/">
    <p>Web标准确保网页在不同浏览器中的一致性表现。</p>
    <footer>—— <cite>W3C官方文档</cite></footer>
</blockquote>

<!-- 用户评论 -->
<blockquote>
    <p>这个产品非常好用，推荐购买！</p>
    <footer>—— 用户张三</footer>
</blockquote>
```

### 1.2 `<q>` 行内引用


**🔸 基本语法**
```html
<p>正如专家所说：<q>细节决定成败</q>。</p>
```

**📋 核心特性**
```
元素类型：行内元素
语义含义：短句引用
默认样式：自动添加引号
浏览器差异：引号样式可能不同
```

**🔸 自动引号效果**
```html
<p>他说：<q>明天见</q>，然后就离开了。</p>
<!-- 浏览器显示：他说："明天见"，然后就离开了。 -->

<p lang="fr">Il a dit: <q>Bonjour</q></p>
<!-- 法语环境：Il a dit: « Bonjour » -->
```

### 1.3 `<cite>` 引用来源


**🔸 基本语法**
```html
<p>根据<cite>《HTML权威指南》</cite>一书所述...</p>
<p>引用自<cite>https://developer.mozilla.org</cite></p>
```

**📋 核心特性**
```
语义含义：引用作品标题、来源
默认样式：斜体显示
适用范围：书籍、文章、网站、电影等
不适用：人名（用<span>或其他标签）
```

**✅ 正确使用**
```html
<!-- 书籍引用 -->
<p>参考资料：<cite>JavaScript高级程序设计</cite></p>

<!-- 网站引用 -->
<p>数据来源：<cite>MDN Web Docs</cite></p>

<!-- 结合blockquote -->
<blockquote>
    <p>代码是写给人看的，只是恰好能够运行。</p>
    <footer>—— <cite>《代码整洁之道》</cite></footer>
</blockquote>
```

## 2. 💻 代码类标签


### 2.1 `<code>` 行内代码


**🔸 基本语法**
```html
<p>使用<code>console.log()</code>可以输出调试信息。</p>
```

**📋 核心特性**
```
元素类型：行内元素
语义含义：代码片段
默认样式：等宽字体
使用场景：变量名、函数名、短代码
```

**✅ 实际应用**
```html
<!-- 变量名 -->
<p>将结果存储在<code>result</code>变量中。</p>

<!-- 函数调用 -->
<p>调用<code>getElementById('demo')</code>获取元素。</p>

<!-- 命令行 -->
<p>运行<code>npm install</code>安装依赖。</p>
```

### 2.2 `<pre>` 预格式化文本


**🔸 基本语法**
```html
<pre>
function hello() {
    console.log("Hello World");
    return true;
}
</pre>
```

**📋 核心特性**
```
元素类型：块级元素
保留格式：空格、换行、缩进完全保留
默认样式：等宽字体，无自动换行
常见用途：代码块、ASCII艺术、诗歌
```

**✅ 代码块展示**
```html
<pre><code>
// JavaScript示例
const users = [
    { name: "张三", age: 25 },
    { name: "李四", age: 30 }
];

users.forEach(user => {
    console.log(`${user.name}: ${user.age}岁`);
});
</code></pre>
```

### 2.3 交互相关代码标签


**🔸 `<kbd>` 键盘输入**
```html
<p>按<kbd>Ctrl</kbd> + <kbd>C</kbd>复制内容。</p>
<p>使用<kbd>Alt</kbd> + <kbd>Tab</kbd>切换窗口。</p>
```

**🔸 `<samp>` 程序输出**
```html
<p>程序运行结果：</p>
<samp>Hello World<br>Process finished with exit code 0</samp>
```

**🔸 `<var>` 变量名称**
```html
<p>在方程 <var>y</var> = <var>mx</var> + <var>b</var> 中...</p>
<p>函数接收参数<var>username</var>和<var>password</var>。</p>
```

### 2.4 代码标签对比


| **标签** | **用途** | **元素类型** | **典型场景** |
|---------|---------|-------------|-------------|
| `<code>` | `代码片段` | `行内` | `变量名、函数名` |
| `<pre>` | `预格式化` | `块级` | `代码块、ASCII图` |
| `<kbd>` | `键盘输入` | `行内` | `快捷键、命令` |
| `<samp>` | `程序输出` | `行内` | `运行结果、终端输出` |
| `<var>` | `变量名` | `行内` | `数学变量、参数名` |

## 3. 🔤 缩写与定义标签


### 3.1 `<abbr>` 缩写说明


**🔸 基本语法**
```html
<p><abbr title="HyperText Markup Language">HTML</abbr>是网页标记语言。</p>
```

**📋 核心特性**
```
语义含义：缩写或首字母缩略词
title属性：必需，提供完整形式
默认样式：可能有下划线或虚线
交互效果：鼠标悬停显示完整形式
```

**✅ 实际应用**
```html
<!-- 技术术语 -->
<p><abbr title="Cascading Style Sheets">CSS</abbr>用于样式设计。</p>
<p><abbr title="Application Programming Interface">API</abbr>接口文档。</p>

<!-- 组织机构 -->
<p><abbr title="World Wide Web Consortium">W3C</abbr>制定Web标准。</p>

<!-- 常见缩写 -->
<p>请在<abbr title="as soon as possible">ASAP</abbr>回复邮件。</p>
```

### 3.2 可访问性增强


**🔸 屏幕阅读器支持**
```html
<!-- 为屏幕阅读器提供更好支持 -->
<abbr title="United States of America" aria-label="美国">USA</abbr>

<!-- 首次出现时完整说明 -->
<p>
    <abbr title="Human Computer Interaction">HCI</abbr>
    (Human Computer Interaction) 是研究人与计算机交互的学科。
    后续提到<abbr title="Human Computer Interaction">HCI</abbr>时...
</p>
```

## 5. 📋 核心要点总结


### 5.1 必知要点


```
🔸 块级引用：<blockquote> 用于大段引用，配合cite属性
🔸 行内引用：<q> 自动添加引号，短句引用
🔸 引用来源：<cite> 标记作品标题和来源
🔸 行内代码：<code> 用于变量名、函数名等短代码
🔸 代码块：<pre> 保留格式，常与<code>结合使用
🔸 缩写说明：<abbr> 配合title属性提供完整形式
```

### 5.2 使用指南


**🎯 快速选择**
```
大段引用   → <blockquote>
短句引用   → <q>
作品标题   → <cite>
行内代码   → <code>
代码块     → <pre><code>
键盘操作   → <kbd>
程序输出   → <samp>
变量名     → <var>
缩写词     → <abbr>
```

### 5.3 最佳实践


**✅ 语义化组合**
```html
<!-- 完整的引用结构 -->
<blockquote cite="source-url">
    <p>引用内容</p>
    <footer>—— <cite>来源</cite></footer>
</blockquote>

<!-- 代码文档结构 -->
<p>使用<code>function</code>关键字定义函数：</p>
<pre><code>
function myFunction() {
    return "Hello";
}
</code></pre>
```

**⚠️ 注意事项**
```
- blockquote不要用于视觉缩进
- cite标签用于作品，不用于人名
- pre标签内容会原样显示，注意格式
- abbr的title属性是必需的
- 代码标签要选择合适的语义
```

**关键记忆**：
- 引用要有来源，代码要有语义
- blockquote大段，q短句
- pre保格式，code显等宽
- abbr要title，cite是来源
