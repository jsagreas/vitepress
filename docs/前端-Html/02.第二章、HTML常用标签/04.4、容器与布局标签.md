## 1. 📦 通用容器标签


### 1.1 `<div>` 块级容器


**🔸 基本语法**
```html
<div class="container">
    <p>块级内容</p>
</div>
```

**📋 核心特性**
```
元素类型：块级元素
语义含义：无特定语义，通用容器
默认样式：独占一行，宽度100%
主要用途：布局分组、样式控制
```

**✅ 典型用法**
```html
<!-- 页面布局容器 -->
<div class="header">头部内容</div>
<div class="main-content">主体内容</div>
<div class="sidebar">侧边栏</div>

<!-- 样式分组 -->
<div class="card">
    <h3>卡片标题</h3>
    <p>卡片内容</p>
</div>

<!-- 功能模块包装 -->
<div class="user-profile">
    <img src="avatar.jpg" alt="头像">
    <span>用户名</span>
</div>
```

### 1.2 `<span>` 行内容器


**🔸 基本语法**
```html
<p>这是<span class="highlight">重点内容</span>的示例。</p>
```

**📋 核心特性**
```
元素类型：行内元素
语义含义：无特定语义，通用容器
默认样式：不换行，宽度由内容决定
主要用途：行内样式控制、文本标记
```

**✅ 实际应用**
```html
<!-- 文本样式控制 -->
<p>价格：<span class="price">￥299</span></p>

<!-- 数据标记 -->
<p>用户ID：<span data-user-id="12345">张三</span></p>

<!-- 图标与文字组合 -->
<button>
    <span class="icon">📁</span>
    <span class="text">保存文件</span>
</button>
```

### 1.3 div vs span 对比


| **特性** | **div** | **span** |
|---------|---------|----------|
| **元素类型** | `块级元素` | `行内元素` |
| **默认样式** | `独占一行` | `不换行` |
| **宽度** | `100%` | `内容宽度` |
| **嵌套规则** | `可嵌套块级和行内` | `只能嵌套行内` |
| **主要用途** | `布局容器` | `文本标记` |

## 2. 🏗️ HTML5语义化布局标签


### 2.1 页面主体结构标签


**🔸 `<header>` 页面头部**
```html
<header>
    <h1>网站标题</h1>
    <nav>
        <ul>
            <li><a href="#home">首页</a></li>
            <li><a href="#about">关于</a></li>
        </ul>
    </nav>
</header>
```

**🔸 `<footer>` 页面底部**
```html
<footer>
    <p>&copy; 2024 公司名称. 保留所有权利。</p>
    <nav>
        <a href="#privacy">隐私政策</a>
        <a href="#terms">服务条款</a>
    </nav>
</footer>
```

**🔸 `<main>` 主要内容**
```html
<main>
    <article>
        <h2>文章标题</h2>
        <p>文章内容...</p>
    </article>
</main>
```

### 2.2 导航与侧边栏标签


**🔸 `<nav>` 导航区域**
```html
<!-- 主导航 -->
<nav class="main-nav">
    <ul>
        <li><a href="#products">产品</a></li>
        <li><a href="#services">服务</a></li>
        <li><a href="#contact">联系</a></li>
    </ul>
</nav>

<!-- 面包屑导航 -->
<nav aria-label="面包屑">
    <ol>
        <li><a href="/">首页</a></li>
        <li><a href="/category">分类</a></li>
        <li>当前页面</li>
    </ol>
</nav>
```

**🔸 `<aside>` 侧边栏内容**
```html
<aside>
    <section>
        <h3>相关文章</h3>
        <ul>
            <li><a href="#">推荐文章1</a></li>
            <li><a href="#">推荐文章2</a></li>
        </ul>
    </section>
    
    <section>
        <h3>标签云</h3>
        <div class="tags">
            <span>HTML</span>
            <span>CSS</span>
            <span>JavaScript</span>
        </div>
    </section>
</aside>
```

### 2.3 内容分区标签


**🔸 `<section>` 内容区块**
```html
<section>
    <h2>产品特性</h2>
    <p>这个区块介绍产品的主要特性...</p>
</section>

<section>
    <h2>用户评价</h2>
    <p>这个区块展示用户的评价信息...</p>
</section>
```

**🔸 `<article>` 独立文章**
```html
<article>
    <header>
        <h2>文章标题</h2>
        <p>发布时间：2024-01-15</p>
    </header>
    
    <section>
        <p>文章正文内容...</p>
    </section>
    
    <footer>
        <p>作者：张三</p>
        <p>标签：HTML, 前端开发</p>
    </footer>
</article>
```

### 2.4 语义标签特性对比


| **标签** | **语义含义** | **使用场景** | **可嵌套** |
|---------|-------------|-------------|-----------|
| `<header>` | `页面/区块头部` | `网站头部、文章头部` | `导航、标题、介绍` |
| `<footer>` | `页面/区块底部` | `版权、链接、联系信息` | `导航、版权、作者信息` |
| `<main>` | `页面主要内容` | `页面核心内容区域` | `article、section` |
| `<nav>` | `导航链接` | `主导航、面包屑、分页` | `ul、ol、a` |
| `<aside>` | `侧边相关内容` | `侧边栏、相关链接` | `section、nav` |
| `<section>` | `内容区块` | `主题相关的内容组` | `header、footer、p` |
| `<article>` | `独立完整内容` | `文章、博客、新闻` | `header、section、footer` |

## 3. 🏛️ 页面结构设计


### 3.1 典型页面布局结构


**📐 经典布局示例**
```
┌─────────────────────────────────────┐
│              <header>               │ ← 网站头部
├─────────────────────────────────────┤
│  <nav>                              │ ← 主导航
├─────────────────────────────────────┤
│  <main>              │   <aside>    │ ← 主内容 + 侧边栏
│                      │              │
│  <article>           │  相关链接    │
│  <section>           │  广告位      │
│  <section>           │  标签云      │
│                      │              │
├─────────────────────────────────────┤
│              <footer>               │ ← 网站底部
└─────────────────────────────────────┘
```

**🔸 完整代码结构**
```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>语义化页面布局</title>
</head>
<body>
    <header>
        <h1>网站名称</h1>
        <nav>
            <ul>
                <li><a href="#home">首页</a></li>
                <li><a href="#blog">博客</a></li>
                <li><a href="#about">关于</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <article>
            <header>
                <h2>文章标题</h2>
                <p>发布时间：2024-01-15</p>
            </header>
            
            <section>
                <h3>章节一</h3>
                <p>章节内容...</p>
            </section>
            
            <section>
                <h3>章节二</h3>
                <p>章节内容...</p>
            </section>
            
            <footer>
                <p>作者：张三</p>
            </footer>
        </article>
    </main>
    
    <aside>
        <section>
            <h3>相关文章</h3>
            <nav>
                <ul>
                    <li><a href="#">相关文章1</a></li>
                    <li><a href="#">相关文章2</a></li>
                </ul>
            </nav>
        </section>
    </aside>
    
    <footer>
        <p>&copy; 2024 网站版权信息</p>
        <nav>
            <a href="#privacy">隐私政策</a>
            <a href="#terms">使用条款</a>
        </nav>
    </footer>
</body>
</html>
```

## 4. ⭐ 语义化布局最佳实践


### 4.1 语义化选择原则


**🎯 标签选择流程**
```
需要布局容器？
├─ 有明确语义 → 选择语义标签
│   ├─ 页面头部 → <header>
│   ├─ 导航区域 → <nav>
│   ├─ 主要内容 → <main>
│   ├─ 独立文章 → <article>
│   ├─ 内容区块 → <section>
│   ├─ 侧边内容 → <aside>
│   └─ 页面底部 → <footer>
└─ 无特定语义 → <div> 或 <span>
```

### 4.2 SEO优化建议


**🔍 搜索引擎友好**
```html
<!-- 良好的语义化结构有助于SEO -->
<article itemscope itemtype="http://schema.org/Article">
    <header>
        <h1 itemprop="headline">文章标题</h1>
        <time itemprop="datePublished" datetime="2024-01-15">
            2024年1月15日
        </time>
    </header>
    
    <section itemprop="articleBody">
        <p>文章正文内容...</p>
    </section>
    
    <footer>
        <address itemprop="author">
            <span itemprop="name">张三</span>
        </address>
    </footer>
</article>
```

### 4.3 可访问性增强


**♿ 无障碍访问优化**
```html
<!-- 为屏幕阅读器提供更好的导航 -->
<nav aria-label="主导航">
    <ul>
        <li><a href="#home" aria-current="page">首页</a></li>
        <li><a href="#about">关于我们</a></li>
    </ul>
</nav>

<main>
    <h1>页面主标题</h1>
    <section aria-labelledby="section1-title">
        <h2 id="section1-title">章节标题</h2>
        <p>章节内容...</p>
    </section>
</main>

<aside aria-label="相关链接">
    <h3>推荐阅读</h3>
    <nav>
        <ul>
            <li><a href="#">相关文章</a></li>
        </ul>
    </nav>
</aside>
```

### 4.4 常见错误与避免


**❌ 常见误用**
```html
<!-- 错误：过度使用div -->
<div class="header">
    <div class="title">标题</div>
    <div class="nav">导航</div>
</div>

<!-- 错误：语义不当 -->
<section>
    <div>这应该是p标签</div>
</section>

<!-- 错误：嵌套错误 -->
<main>
    <main>嵌套main标签</main> <!-- main不能嵌套 -->
</main>
```

**✅ 正确写法**
```html
<!-- 正确：语义化标签 -->
<header>
    <h1>标题</h1>
    <nav>导航</nav>
</header>

<!-- 正确：合理语义 -->
<section>
    <p>这是段落内容</p>
</section>

<!-- 正确：单一main -->
<main>
    <article>文章内容</article>
</main>
```

## 5. 📋 核心要点总结


### 5.1 必知要点


```
🔸 通用容器：<div> 块级，<span> 行内，无语义
🔸 页面结构：<header> <main> <footer> 构成基本框架
🔸 导航区域：<nav> 用于所有导航链接
🔸 内容分区：<section> 主题区块，<article> 独立文章
🔸 侧边内容：<aside> 相关但非主要内容
🔸 语义优先：有语义用语义标签，无语义用div/span
```

### 5.2 布局选择指南


**🎯 快速决策**
```
页面头部    → <header>
页面导航    → <nav>
主要内容    → <main>
独立文章    → <article>
内容区块    → <section>
相关内容    → <aside>
页面底部    → <footer>
通用容器    → <div>
行内容器    → <span>
```

### 5.3 现代化建议


**⭐ 最佳实践组合**
```html
<!-- 现代语义化页面结构 -->
<body>
    <header>
        <nav>主导航</nav>
    </header>
    
    <main>
        <article>
            <header>文章头部</header>
            <section>内容区块</section>
            <footer>文章底部</footer>
        </article>
    </main>
    
    <aside>
        <section>相关内容</section>
    </aside>
    
    <footer>
        <nav>底部导航</nav>
    </footer>
</body>
```

**关键记忆**：
- 语义化优于div堆砌
- main唯一，header/footer可多个
- section需要标题，article要独立
- nav包含导航，aside放侧边内容
