## 1. 📝 段落标签


### 1.1 `<p>` 段落标签


**🔸 基本语法**
```html
<p>这是一个段落内容</p>
<p>这是另一个段落内容</p>
```

**📋 核心特性**
```
元素类型：块级元素
默认样式：上下外边距，独占一行
语义含义：表示一个文本段落
嵌套规则：不能嵌套其他块级元素
```

**✅ 正确用法**
```html
<p>段落文本内容，可以包含 <strong>强调文本</strong> 和 <a href="#">链接</a>。</p>
<p>另一个独立的段落。</p>
```

**❌ 错误用法**
```html
<!-- 错误：p标签内不能嵌套块级元素 -->
<p>
    <div>这样是错误的</div>
</p>

<!-- 错误：不要用p标签做布局 -->
<p></p> <!-- 空p标签用来占位 -->
```

## 2. 🔄 换行与分隔线


### 2.1 `<br>` 换行标签


**🔸 基本语法**
```html
第一行文本<br>
第二行文本<br>
第三行文本
```

**📋 核心特性**
```
元素类型：行内元素，单标签
作用：强制换行，不产生段落间距
使用场景：诗歌、地址、歌词等需要保持行结构的内容
注意：避免用br进行布局
```

**✅ 合理使用**
```html
<address>
张三<br>
北京市朝阳区<br>
邮编：100000
</address>

<p>
床前明月光，<br>
疑是地上霜。<br>
举头望明月，<br>
低头思故乡。
</p>
```

### 2.2 `<hr>` 水平分隔线


**🔸 基本语法**
```html
<p>第一部分内容</p>
<hr>
<p>第二部分内容</p>
```

**📋 核心特性**
```
元素类型：块级元素，单标签
语义含义：主题转换或内容分隔
默认样式：水平线，上下外边距
现代用法：更多用CSS border替代样式
```

**🎨 样式控制**
```html
<!-- HTML结构 -->
<hr>

<!-- CSS样式 -->
<style>
hr {
    border: none;
    height: 2px;
    background-color: #ccc;
    margin: 20px 0;
}
</style>
```

## 3. 📑 标题标签


### 3.1 六级标题 `<h1>` ~ `<h6>`


**🔸 基本语法**
```html
<h1>一级标题</h1>
<h2>二级标题</h2>
<h3>三级标题</h3>
<h4>四级标题</h4>
<h5>五级标题</h5>
<h6>六级标题</h6>
```

### 3.2 标题层级规范


**📊 标题层级结构**
```
页面标题结构示例：
<h1>网站主标题</h1>          ← 页面主题，通常唯一
  <h2>章节标题</h2>          ← 主要章节
    <h3>小节标题</h3>        ← 子章节
      <h4>具体内容标题</h4>  ← 详细内容
        <h5>细分标题</h5>    ← 更细分类
          <h6>最小标题</h6>  ← 最小层级
```

**✅ 正确的层级使用**
```html
<h1>HTML学习指南</h1>
  <h2>基础标签</h2>
    <h3>文本标签</h3>
      <h4>段落标签</h4>
      <h4>标题标签</h4>
    <h3>列表标签</h3>
  <h2>表单标签</h2>
```

**❌ 错误的层级使用**
```html
<!-- 错误：跳级使用 -->
<h1>主标题</h1>
<h3>直接跳到三级</h3>  <!-- 缺少h2 -->

<!-- 错误：为了样式选择标题 -->
<h3>看起来大小合适</h3>  <!-- 应该用CSS控制样式 -->
```

### 3.3 默认样式表现


| **标签** | **默认字体大小** | **字体粗细** | **上下边距** |
|---------|-----------------|-------------|-------------|
| `<h1>`  | `2em (32px)`    | `bold`      | `0.67em`    |
| `<h2>`  | `1.5em (24px)`  | `bold`      | `0.83em`    |
| `<h3>`  | `1.17em (18px)` | `bold`      | `1em`       |
| `<h4>`  | `1em (16px)`    | `bold`      | `1.33em`    |
| `<h5>`  | `0.83em (13px)` | `bold`      | `1.67em`    |
| `<h6>`  | `0.67em (11px)` | `bold`      | `2.33em`    |

## 4. 🔍 语义化与SEO


### 4.1 SEO重要性排序


**📈 搜索引擎权重**
```
SEO权重排序：
h1 > h2 > h3 > h4 > h5 > h6

重要性递减：
┌─────────────┬─────────────┬─────────────┐
│     h1      │     h2      │     h3      │
│  最高权重   │   高权重    │   中等权重  │
│ 页面主题    │  主要章节   │   子章节    │
└─────────────┴─────────────┴─────────────┘
       ↓           ↓           ↓
┌─────────────┬─────────────┬─────────────┐
│     h4      │     h5      │     h6      │
│   较低权重  │   低权重    │   最低权重  │
│  详细内容   │   细分内容  │   补充内容  │
└─────────────┴─────────────┴─────────────┘
```

### 4.2 最佳实践


**✅ SEO友好的结构**
```html
<!-- 每页一个h1，描述页面主题 -->
<h1>产品详情页 - iPhone 15</h1>

<!-- h2用于主要内容区块 -->
<h2>产品特性</h2>
<h2>技术参数</h2>
<h2>用户评价</h2>

<!-- h3用于细分内容 -->
<h2>技术参数</h2>
  <h3>屏幕规格</h3>
  <h3>处理器性能</h3>
  <h3>摄像头配置</h3>
```

### 4.3 可访问性考虑


**🔸 屏幕阅读器支持**
```
标题的可访问性作用：
- 提供文档结构导航
- 快速跳转到特定内容
- 理解页面层次关系
- 语义化朗读体验
```

## 5. 📋 核心要点总结


### 5.1 必知要点


```
🔸 段落标签：<p> 用于文本段落，块级元素，不能嵌套块级元素
🔸 换行标签：<br> 单标签，用于段内换行，避免用于布局
🔸 分隔线：<hr> 表示主题转换，语义化分隔
🔸 标题层级：h1-h6 语义递减，SEO权重递减，不跳级使用
🔸 SEO原则：每页一个h1，标题要反映内容层次
```

### 5.2 实用建议


**🎯 最佳实践**
```
结构规划：
1. 先设计内容层级，再选择标题级别
2. 用CSS控制样式，不要为了样式选择标题
3. 保持标题的语义化和层次性
4. 每个页面只用一个h1标签
```

**⚠️ 常见错误**
```
避免的做法：
- 用多个<br>代替段落
- 用<p></p>空标签做间距
- 为了字体大小跳级使用标题
- 一个页面多个h1标签
```

**关键记忆**：
- p段落、br换行、hr分隔
- h1到h6权重递减
- 语义优先，样式用CSS
- 结构清晰，SEO友好
