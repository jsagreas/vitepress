## 1. 🌐 GET请求是什么


### 1.1 生活中的GET请求

想象你去图书馆**借书**：
- 你告诉管理员："我要借《JavaScript入门》这本书"
- 管理员根据书名找到书，然后**给你**这本书
- 你**获取**到了想要的书

GET请求就是这样的**获取数据**过程！

### 1.2 GET请求的本质

**🔸 核心概念**
```
GET = 获取数据
- 就像"取快递"一样，你提供地址，快递员给你包裹
- 网页向服务器说："给我某某数据"
- 服务器把数据"返回"给网页
```

**💡 特点总结**
- **获取数据**：专门用来从服务器拿数据
- **安全操作**：不会改变服务器上的数据（只是看看）
- **可重复**：发送100次同样的请求，结果都一样
- **URL可见**：参数直接显示在网址里

## 2. 📝 axios.get()基本语法


### 2.1 最简单的用法

```javascript
// 最基础的GET请求 - 就像说"给我首页数据"
axios.get('https://api.example.com/users')
  .then(response => {
    console.log(response.data); // 这里就是服务器返回的数据
  });
```

### 2.2 完整语法格式

```javascript
axios.get(url, config)
     ↑    ↑      ↑
   方法  网址   配置
```

**🔸 参数说明**
```
url：要访问的网址（必填）
config：额外配置（可选）
- params：URL参数
- headers：请求头
- timeout：超时时间
等等...
```

### 2.3 两种写法对比

```javascript
// 写法1：分开写（推荐新手）
const url = 'https://api.example.com/users';
const config = { params: { page: 1 } };
axios.get(url, config);

// 写法2：合并写（熟练后使用）
axios.get('https://api.example.com/users', {
  params: { page: 1 }
});
```

## 3. 🔗 URL参数传递详解


### 3.3 什么是URL参数

想象你在餐厅点餐：
- 基础：我要一份面条
- 详细：我要一份**牛肉面**，**不要香菜**，**微辣**

URL参数就是这些**具体要求**！

### 3.2 params配置的作用

**🔸 自动处理参数**
```javascript
// 你只需要写对象
axios.get('/api/products', {
  params: {
    category: '手机',
    price: '1000-2000',
    page: 1
  }
});

// axios自动转换成网址
// 实际请求：/api/products?category=手机&price=1000-2000&page=1
```

**💡 为什么用params而不是直接写网址？**
```javascript
// ❌ 不推荐：自己拼接网址
axios.get('/api/products?category=手机&price=1000-2000&page=1');

// ✅ 推荐：使用params
axios.get('/api/products', {
  params: { category: '手机', price: '1000-2000', page: 1 }
});

原因：
1. axios会自动编码中文和特殊字符
2. 代码更清晰，不容易出错
3. 参数多了也不会乱
```

### 3.3 参数类型处理

```javascript
axios.get('/api/search', {
  params: {
    keyword: '苹果手机',    // 字符串：自动编码中文
    price: 5999,           // 数字：自动转成字符串
    inStock: true,         // 布尔值：转成'true'
    tags: ['热销', '新品']  // 数组：转成tags[]=热销&tags[]=新品
  }
});
```

## 4. 📊 响应数据结构解析


### 4.1 响应对象包含什么

服务器返回的不只是数据，还有很多**额外信息**：

```javascript
axios.get('/api/users/1').then(response => {
  console.log(response);
  // response对象包含：
  // {
  //   data: {...},      // 真正的数据内容
  //   status: 200,      // 状态码（200表示成功）
  //   headers: {...},   // 响应头信息
  //   config: {...}     // 请求配置信息
  // }
});
```

### 4.2 最重要的data属性

**🔸 data就是你要的数据**
```javascript
axios.get('/api/users/1').then(response => {
  // ✅ 获取用户数据
  const userData = response.data;
  console.log(userData.name);    // 用户姓名
  console.log(userData.email);   // 用户邮箱
});
```

### 4.3 状态码的含义

```
📋 常见状态码
200：成功了，数据获取正常
404：找不到，可能网址写错了
500：服务器出错了
401：没有权限访问
```

**💡 实际使用**
```javascript
axios.get('/api/users/1').then(response => {
  if (response.status === 200) {
    console.log('数据获取成功');
    console.log(response.data);
  }
}).catch(error => {
  console.log('请求失败：', error.message);
});
```

## 5. 🎯 GET请求实战场景


### 5.1 获取用户列表

**场景**：显示所有注册用户

```javascript
// 获取用户列表
function getUserList() {
  axios.get('/api/users')
    .then(response => {
      const users = response.data;
      // 显示用户列表
      users.forEach(user => {
        console.log(`用户：${user.name}, 邮箱：${user.email}`);
      });
    });
}
```

### 5.2 获取商品详情

**场景**：用户点击商品，查看详细信息

```javascript
// 获取指定商品详情
function getProductDetail(productId) {
  axios.get(`/api/products/${productId}`)
    .then(response => {
      const product = response.data;
      // 显示商品信息
      document.getElementById('product-name').textContent = product.name;
      document.getElementById('product-price').textContent = product.price;
    });
}

// 使用：获取ID为123的商品详情
getProductDetail(123);
```

### 5.3 分页查询数据

**场景**：数据太多，需要分页显示

```javascript
// 分页获取商品列表
function getProductList(page = 1, size = 10) {
  axios.get('/api/products', {
    params: {
      page: page,     // 第几页
      size: size      // 每页多少条
    }
  }).then(response => {
    const data = response.data;
    console.log(`总共${data.total}条商品`);
    console.log(`当前第${data.page}页`);
    console.log('商品列表：', data.products);
  });
}

// 使用：获取第2页数据，每页20条
getProductList(2, 20);
```

### 5.4 搜索和筛选

**场景**：用户搜索"苹果手机"，价格3000-8000元

```javascript
// 搜索商品
function searchProducts(keyword, minPrice, maxPrice) {
  axios.get('/api/products/search', {
    params: {
      q: keyword,           // 搜索关键词
      price_min: minPrice,  // 最低价格
      price_max: maxPrice   // 最高价格
    }
  }).then(response => {
    const products = response.data;
    console.log(`找到${products.length}个商品`);
    products.forEach(product => {
      console.log(`${product.name} - ¥${product.price}`);
    });
  });
}

// 使用：搜索苹果手机，价格3000-8000
searchProducts('苹果手机', 3000, 8000);
```

## 6. ❓ 常见问题与解决方案


### 6.1 中文参数问题

**问题**：参数包含中文时显示乱码

```javascript
// ✅ 正确做法：使用params自动编码
axios.get('/api/search', {
  params: { keyword: '苹果手机' }  // axios自动处理中文
});

// ❌ 错误做法：手动拼接
axios.get('/api/search?keyword=苹果手机');  // 可能出现乱码
```

### 6.2 参数为空的处理

```javascript
// 处理可能为空的参数
function searchProducts(keyword, category) {
  const params = {};
  
  if (keyword) params.keyword = keyword;
  if (category) params.category = category;
  
  axios.get('/api/products', { params });
}
```

### 6.3 数组参数的传递

```javascript
// 传递多个标签
axios.get('/api/products', {
  params: {
    tags: ['热销', '新品', '推荐']
  }
});
// 实际请求：/api/products?tags[]=热销&tags[]=新品&tags[]=推荐
```

## 7. 📋 核心要点总结


### 7.1 必须掌握的核心概念

```
🔸 GET请求本质：从服务器获取数据，就像"取快递"
🔸 基本语法：axios.get(url, { params: {...} })
🔸 params作用：自动处理URL参数，避免手动拼接
🔸 response.data：这里是真正要用的数据
🔸 状态码含义：200成功，404找不到，500服务器错误
```

### 7.2 实用记忆要点

**🔹 GET请求三步走**
```
1. 确定要访问的网址
2. 准备参数（如果有的话）
3. 处理返回的数据
```

**🔹 参数传递最佳实践**
```
- 用params配置，不要手动拼接URL
- 中文参数交给axios自动处理
- 数组参数直接传数组
- 空参数要过滤掉
```

**🔹 常见使用场景**
- **获取列表**：用户列表、商品列表、文章列表
- **获取详情**：商品详情、用户信息、文章内容  
- **搜索筛选**：关键词搜索、条件筛选、分页查询

### 7.3 新手避坑指南

```
🚫 常见错误：
1. 忘记写.then()处理结果
2. 直接使用response而不是response.data
3. 手动拼接URL导致中文乱码
4. 不处理请求失败的情况

✅ 正确做法：
1. 永远记得处理返回结果
2. 使用response.data获取真正数据
3. 用params传递所有参数
4. 用.catch()处理错误情况
```

**核心记忆**：
- GET = 获取数据，就像取快递
- params = 自动处理参数，比手拼URL安全
- response.data = 真正要用的数据
- 三个场景：列表、详情、搜索
