## 1. 🎯 JSP动作标签概述


### 1.1 什么是JSP动作标签


**简单理解**：JSP动作标签就像是JSP页面中的"功能按钮"，每个标签都有特定的作用

```
想象你在写一个网页：
- 需要包含其他页面内容 → 用<jsp:include>
- 需要跳转到其他页面 → 用<jsp:forward>  
- 需要传递参数 → 用<jsp:param>
- 需要操作Java对象 → 用<jsp:useBean>等

就像工具箱里的不同工具，每个都有专门用途
```

### 1.2 动作标签的特点


**🔸 XML语法格式**
```jsp
<!-- 动作标签都是这种格式 -->
<jsp:标签名 属性="值">
    <!-- 可能有内容 -->
</jsp:标签名>

<!-- 或者自闭合格式 -->
<jsp:标签名 属性="值" />
```

**🔸 运行时执行**
- **编译时**：JSP脚本元素（如`<% %>`）在页面编译时就确定了内容
- **运行时**：动作标签在每次请求时都重新执行，内容可能不同

**🔸 功能分类**
```
页面控制类：
├── <jsp:include> → 包含其他页面
├── <jsp:forward> → 转发到其他页面
└── <jsp:param>   → 传递参数

对象操作类：
├── <jsp:useBean>     → 创建或获取Java对象
├── <jsp:setProperty> → 设置对象属性
└── <jsp:getProperty> → 获取对象属性值
```

### 1.3 为什么需要动作标签


**解决的问题**：
- ❌ **代码重复**：多个页面需要相同内容
- ❌ **页面跳转复杂**：需要简单的页面转发机制  
- ❌ **参数传递麻烦**：页面间传参需要简化
- ❌ **Java代码混乱**：在JSP中直接写Java代码不够优雅

**提供的解决方案**：
- ✅ **页面模块化**：把公共部分抽取出来重复使用
- ✅ **简化跳转**：一个标签就能实现页面转发
- ✅ **优雅传参**：标准化的参数传递方式
- ✅ **封装Java操作**：用标签代替Java代码

## 2. 📄 动态包含：jsp:include


### 2.1 什么是动态包含


**通俗解释**：就像搭积木一样，把不同的页面片段组合成一个完整页面

```
主页面：index.jsp
┌─────────────────┐
│   网站头部       │ ← header.jsp (用jsp:include包含)
├─────────────────┤
│   导航菜单       │ ← nav.jsp (用jsp:include包含)  
├─────────────────┤
│   主要内容       │ ← 主页面自己的内容
├─────────────────┤
│   网站底部       │ ← footer.jsp (用jsp:include包含)
└─────────────────┘

这样每个部分都可以单独维护，修改头部只需要改header.jsp
```

### 2.2 基本语法和使用


**🔸 基础语法**
```jsp
<jsp:include page="要包含的页面路径" />

<!-- 或者带参数的形式 -->
<jsp:include page="要包含的页面路径">
    <jsp:param name="参数名" value="参数值" />
</jsp:include>
```

**🔸 实际应用示例**

**主页面：main.jsp**
```jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" %>
<!DOCTYPE html>
<html>
<head>
    <title>我的网站</title>
</head>
<body>
    <!-- 包含页面头部 -->
    <jsp:include page="header.jsp" />
    
    <!-- 主要内容 -->
    <div class="content">
        <h1>欢迎来到我的网站</h1>
        <p>这里是主要内容区域</p>
    </div>
    
    <!-- 包含页面底部 -->
    <jsp:include page="footer.jsp" />
</body>
</html>
```

**头部页面：header.jsp**
```jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" %>
<header style="background-color: #333; color: white; padding: 10px;">
    <h2>我的个人网站</h2>
    <nav>
        <a href="home.jsp" style="color: white;">首页</a> |
        <a href="about.jsp" style="color: white;">关于我</a> |
        <a href="contact.jsp" style="color: white;">联系方式</a>
    </nav>
</header>
```

### 2.3 动态包含 vs 静态包含


| 特性对比 | **动态包含**`<jsp:include>` | **静态包含**`<%@ include %>` |
|---------|----------------------------|---------------------------|
| **执行时机** | `运行时包含` | `编译时包含` |
| **文件处理** | `被包含文件独立编译` | `被包含文件内容直接插入` |
| **参数传递** | `支持传递参数` | `不支持参数传递` |
| **性能** | `每次请求都重新处理` | `编译后固定内容，性能更好` |
| **灵活性** | `内容可以动态变化` | `内容固定不变` |

**🔸 选择建议**
```
使用动态包含的情况：
✅ 被包含页面内容会动态变化
✅ 需要向被包含页面传递参数
✅ 被包含页面需要独立维护和测试

使用静态包含的情况：
✅ 被包含内容完全固定（如版权声明）
✅ 追求最佳性能
✅ 简单的HTML片段包含
```

### 2.4 带参数的动态包含


**🔸 实际应用场景**
```jsp
<!-- 主页面：向被包含页面传递用户信息 -->
<jsp:include page="user_info.jsp">
    <jsp:param name="username" value="张三" />
    <jsp:param name="role" value="管理员" />
</jsp:include>
```

**被包含页面：user_info.jsp**
```jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" %>
<%
    // 接收传递过来的参数
    String username = request.getParameter("username");
    String role = request.getParameter("role");
%>
<div class="user-info">
    <p>当前用户：<%= username %></p>
    <p>用户角色：<%= role %></p>
    <p>登录时间：<%= new java.util.Date() %></p>
</div>
```

**🔸 参数传递原理**
```
浏览器请求 → 主页面处理
             ↓
         执行到jsp:include
             ↓
         创建新的request对象（包含原有参数+新参数）
             ↓  
         调用被包含页面处理
             ↓
         被包含页面可以通过request.getParameter()获取参数
             ↓
         返回处理结果，插入到主页面中
```

## 3. 🔄 页面转发：jsp:forward


### 3.1 什么是页面转发


**通俗理解**：就像接力赛跑，当前页面把"接力棒"（请求）传给下一个页面

```
用户访问流程：

用户浏览器 → A.jsp页面 → 判断需要转发 → B.jsp页面 → 返回结果
    ↑                                      ↓
    └──────────── 用户看到的是B页面的内容 ←────────┘

重点：用户地址栏显示的还是A.jsp，但看到的内容是B.jsp的
```

### 3.2 基本语法和使用


**🔸 基础语法**
```jsp
<jsp:forward page="目标页面路径" />

<!-- 带参数的转发 -->
<jsp:forward page="目标页面路径">
    <jsp:param name="参数名" value="参数值" />
</jsp:forward>
```

**🔸 实际应用示例**

**登录验证页面：login_check.jsp**
```jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" %>
<%
    // 获取用户提交的登录信息
    String username = request.getParameter("username");
    String password = request.getParameter("password");
    
    // 简单的登录验证逻辑
    if ("admin".equals(username) && "123456".equals(password)) {
        // 登录成功，转发到管理员页面
%>
        <jsp:forward page="admin_home.jsp">
            <jsp:param name="loginTime" value="<%= new java.util.Date() %>" />
        </jsp:forward>
<%
    } else {
        // 登录失败，转发到错误页面
%>
        <jsp:forward page="login_error.jsp">
            <jsp:param name="errorMsg" value="用户名或密码错误" />
        </jsp:forward>
<%
    }
%>
```

**管理员主页：admin_home.jsp**
```jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" %>
<!DOCTYPE html>
<html>
<head>
    <title>管理员后台</title>
</head>
<body>
    <h1>欢迎进入管理员后台</h1>
    <p>登录时间：<%= request.getParameter("loginTime") %></p>
    
    <div>
        <h3>管理功能</h3>
        <ul>
            <li><a href="user_manage.jsp">用户管理</a></li>
            <li><a href="system_config.jsp">系统配置</a></li>
        </ul>
    </div>
</body>
</html>
```

### 3.3 转发 vs 重定向


| 特性对比 | **转发**`<jsp:forward>` | **重定向**`response.sendRedirect()` |
|---------|------------------------|-----------------------------------|
| **地址栏变化** | `不变（用户看不出来）` | `变成新页面地址` |
| **请求次数** | `1次请求` | `2次请求` |
| **数据共享** | `request数据可以共享` | `request数据丢失` |
| **性能** | `更快（服务器内部处理）` | `较慢（需要重新请求）` |
| **使用场景** | `内部页面跳转` | `跳转到外部网站或需要刷新地址栏` |

**🔸 图解对比**
```
转发过程：
用户 → [请求] → 服务器A页面 → [内部转发] → 服务器B页面 → [响应] → 用户
                    ↑_________________↓
                    在服务器内部完成

重定向过程：  
用户 → [请求] → 服务器A页面 → [302响应] → 用户 → [新请求] → 服务器B页面 → [响应] → 用户
      ↑________________________________________________↓
                      需要用户浏览器重新发起请求
```

### 3.4 转发的注意事项


**🔸 转发的限制条件**
- ✅ **只能转发到同一Web应用内**的页面
- ✅ **必须在响应提交之前**执行转发
- ✅ **转发后的代码不会再执行**

**⚠️ 常见错误示例**
```jsp
<%
    out.println("这行会输出");  // ← 这行会执行
%>
<jsp:forward page="other.jsp" />
<%
    out.println("这行不会输出"); // ← 转发后这行不会执行
%>
```

**✅ 正确的使用方式**
```jsp
<%
    String result = someBusinessLogic();
    if ("success".equals(result)) {
        // 成功时转发
%>
        <jsp:forward page="success.jsp" />
<%
        return; // 确保后续代码不执行
    }
    
    // 失败时的处理逻辑
    request.setAttribute("error", "操作失败");
%>
<jsp:forward page="error.jsp" />
```

## 4. 📤 参数传递：jsp:param


### 4.1 参数传递的作用


**通俗解释**：`<jsp:param>`就像传纸条一样，让一个页面可以给另一个页面传递信息

```
页面A想告诉页面B：
- 当前用户是谁
- 要显示什么内容  
- 执行什么操作

通过jsp:param就能把这些信息传递过去
```

### 4.2 基本语法


**🔸 语法格式**
```jsp
<jsp:param name="参数名" value="参数值" />
```

**🔸 使用位置**
- 只能在`<jsp:include>`或`<jsp:forward>`内部使用
- 不能单独使用

### 4.3 参数传递的实际应用


**🔸 动态导航菜单示例**

**主页面：index.jsp**
```jsp
<!-- 传递当前页面标识，让导航知道高亮哪个菜单 -->
<jsp:include page="navigation.jsp">
    <jsp:param name="currentPage" value="home" />
    <jsp:param name="userLevel" value="vip" />
</jsp:include>

<div class="content">
    <h1>网站首页内容</h1>
</div>
```

**导航页面：navigation.jsp**
```jsp
<%
    String currentPage = request.getParameter("currentPage");
    String userLevel = request.getParameter("userLevel");
%>
<nav class="main-nav">
    <a href="index.jsp" 
       class="<%= "home".equals(currentPage) ? "active" : "" %>">
       首页
    </a>
    
    <a href="products.jsp"
       class="<%= "products".equals(currentPage) ? "active" : "" %>">
       产品
    </a>
    
    <% if ("vip".equals(userLevel)) { %>
        <a href="vip.jsp">VIP专区</a>
    <% } %>
</nav>

<style>
.main-nav a.active {
    background-color: #007cba;
    color: white;
}
</style>
```

**🔸 表单数据处理示例**

**数据处理页面：process_data.jsp**
```jsp
<%
    // 处理表单数据
    String action = request.getParameter("action");
    String result = "";
    
    if ("save".equals(action)) {
        // 执行保存操作
        result = "数据保存成功";
    } else if ("delete".equals(action)) {
        // 执行删除操作  
        result = "数据删除成功";
    }
%>

<!-- 转发到结果页面，传递处理结果 -->
<jsp:forward page="result.jsp">
    <jsp:param name="message" value="<%= result %>" />
    <jsp:param name="messageType" value="success" />
</jsp:forward>
```

**结果显示页面：result.jsp**
```jsp
<%
    String message = request.getParameter("message");
    String messageType = request.getParameter("messageType");
    
    // 根据消息类型确定显示样式
    String cssClass = "success".equals(messageType) ? "alert-success" : "alert-error";
%>

<div class="<%= cssClass %>">
    <p><%= message %></p>
    <a href="javascript:history.back()">返回上一页</a>
</div>
```

### 4.4 参数传递的原理


**🔸 底层工作机制**
```
1. jsp:param收集参数信息
   ├── name="username", value="张三"
   └── name="role", value="admin"

2. 创建新的请求对象
   ├── 包含原始request的所有参数
   └── 添加jsp:param指定的新参数

3. 目标页面接收参数
   ├── request.getParameter("username") → "张三"
   └── request.getParameter("role") → "admin"
```

**🔸 参数覆盖规则**
```jsp
<!-- 如果原请求已有同名参数会怎样？ -->
<%-- 原始请求：?username=李四 --%>

<jsp:forward page="target.jsp">
    <jsp:param name="username" value="张三" />
</jsp:forward>

<!-- 
结果：target.jsp中
request.getParameter("username") 返回 "张三"  (新参数覆盖原参数)
request.getParameterValues("username") 返回 ["张三", "李四"] (都能获取到)
-->
```

## 5. ☕ JavaBean操作标签


### 5.1 什么是JavaBean


**通俗解释**：JavaBean就像一个标准化的"数据盒子"，有固定的格式来存储和操作数据

```
想象一个用户信息盒子：
┌─────────────────┐
│   User对象      │
├─────────────────┤
│ - name: 张三    │ ← 私有属性
│ - age: 25       │
│ - email: xx@xx  │
├─────────────────┤
│ + getName()     │ ← 公共方法
│ + setName()     │
│ + getAge()      │
│ + setAge()      │
└─────────────────┘

JSP动作标签可以直接操作这个"盒子"，不需要写Java代码
```

### 5.2 JavaBean的基本要求


**🔸 标准JavaBean必须满足**
```java
public class User {
    // 1. 私有属性
    private String name;
    private int age;
    
    // 2. 无参构造方法
    public User() {}
    
    // 3. 公共的getter和setter方法
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    
    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
}
```

### 5.3 jsp:useBean - 创建和获取对象


**🔸 基本语法**
```jsp
<jsp:useBean id="对象名" class="完整类名" scope="作用域" />
```

**🔸 作用域说明**
- **page**：只在当前页面有效（默认）
- **request**：在同一次请求中有效  
- **session**：在用户会话期间有效
- **application**：在整个Web应用中有效

**🔸 实际应用示例**

**用户信息处理：user_demo.jsp**
```jsp
<%@ page import="com.example.User" %>

<!-- 创建或获取User对象，存储在session中 -->
<jsp:useBean id="user" class="com.example.User" scope="session" />

<!DOCTYPE html>
<html>
<head>
    <title>用户信息管理</title>
</head>
<body>
    <h2>当前用户信息</h2>
    
    <!-- 显示用户信息 -->
    <p>姓名：<jsp:getProperty name="user" property="name" /></p>
    <p>年龄：<jsp:getProperty name="user" property="age" /></p>
    
    <!-- 表单：修改用户信息 -->
    <form method="post">
        <label>姓名：</label>
        <input type="text" name="name" value="<jsp:getProperty name="user" property="name" />" />
        <br><br>
        
        <label>年龄：</label>
        <input type="text" name="age" value="<jsp:getProperty name="user" property="age" />" />
        <br><br>
        
        <input type="submit" value="更新信息" />
    </form>
</body>
</html>
```

### 5.4 jsp:setProperty - 设置对象属性


**🔸 四种设置方式**

**方式1：设置所有匹配的属性**
```jsp
<!-- 自动将表单参数设置到对象属性（参数名要与属性名一致） -->
<jsp:setProperty name="user" property="*" />
```

**方式2：设置指定属性为参数值**
```jsp
<!-- 将request参数"username"的值设置给name属性 -->
<jsp:setProperty name="user" property="name" param="username" />
```

**方式3：设置指定属性为固定值**
```jsp
<!-- 直接设置固定值 -->
<jsp:setProperty name="user" property="name" value="管理员" />
<jsp:setProperty name="user" property="age" value="30" />
```

**方式4：设置指定属性为表达式值**
```jsp
<!-- 使用表达式计算值 -->
<jsp:setProperty name="user" property="age" value="<%= 2024 - 1990 %>" />
```

**🔸 完整的用户信息更新示例**
```jsp
<%@ page import="com.example.User" %>

<!-- 获取或创建用户对象 -->
<jsp:useBean id="user" class="com.example.User" scope="session" />

<%
    // 判断是否是表单提交
    if ("POST".equals(request.getMethod())) {
%>
        <!-- 批量设置属性 -->
        <jsp:setProperty name="user" property="*" />
        
        <div class="success-message">
            ✅ 用户信息更新成功！
        </div>
<%
    }
%>
```

### 5.5 jsp:getProperty - 获取对象属性


**🔸 基本语法**
```jsp
<jsp:getProperty name="对象名" property="属性名" />
```

**🔸 实际应用场景**

**用户资料卡片：user_card.jsp**
```jsp
<jsp:useBean id="user" class="com.example.User" scope="session" />

<div class="user-card">
    <div class="avatar">👤</div>
    <div class="user-info">
        <h3><jsp:getProperty name="user" property="name" /></h3>
        <p>年龄：<jsp:getProperty name="user" property="age" />岁</p>
        <p>邮箱：<jsp:getProperty name="user" property="email" /></p>
        
        <%-- 条件显示 --%>
        <%
            // 获取属性值进行判断
            String name = ((User)session.getAttribute("user")).getName();
            if (name != null && !name.isEmpty()) {
        %>
            <p class="welcome">欢迎回来，<jsp:getProperty name="user" property="name" />!</p>
        <% } else { %>
            <p class="prompt">请先设置您的姓名</p>
        <% } %>
    </div>
</div>
```

### 5.6 JavaBean标签的综合应用


**🔸 用户注册完整示例**

**用户Bean：User.java**
```java
package com.example;

public class User {
    private String username;
    private String email;
    private int age;
    private String registerTime;
    
    public User() {
        // 设置默认注册时间
        this.registerTime = new java.util.Date().toString();
    }
    
    // 省略getter和setter方法...
}
```

**注册页面：register.jsp**
```jsp
<%@ page import="com.example.User" %>
<jsp:useBean id="newUser" class="com.example.User" scope="request" />
```

## 6. 💡 动作标签最佳实践


### 6.1 选择合适的动作标签


**🔸 标签选择指南**
```
页面组织需求：
├── 需要动态包含内容 → jsp:include
├── 需要页面跳转 → jsp:forward  
├── 需要传递参数 → jsp:param
└── 需要操作Java对象 → jsp:useBean系列

性能考虑：
├── 固定内容包含 → 静态包含 <%@ include %>
├── 动态内容包含 → jsp:include
├── 内部页面跳转 → jsp:forward
└── 需要刷新地址栏 → response.sendRedirect()
```

### 6.2 常见错误和避免方法


**❌ 错误1：转发后继续输出内容**
```jsp
<jsp:forward page="success.jsp" />
<p>这段文字会导致错误</p>  <!-- 转发后不应该有内容输出 -->
```

**✅ 正确做法**
```jsp
<jsp:forward page="success.jsp" />
<%
    return; // 确保不再执行后续代码
%>
```

**❌ 错误2：重复创建Bean对象**
```jsp
<!-- 每次访问都会创建新对象，浪费内存 -->
<jsp:useBean id="user" class="User" scope="page" />
```

**✅ 正确做法**
```jsp
<!-- 使用合适的作用域，避免重复创建 -->
<jsp:useBean id="user" class="User" scope="session" />
```

### 6.3 性能优化建议


**🔸 包含优化**
```
静态内容：使用 <%@ include %> 编译时包含
动态内容：使用 <jsp:include> 运行时包含
条件包含：先判断条件再决定是否包含
```

**🔸 对象管理优化**
```jsp
<!-- 合理使用作用域 -->
<jsp:useBean id="config" class="Config" scope="application" />  <!-- 全局配置 -->
<jsp:useBean id="user" class="User" scope="session" />          <!-- 用户信息 -->
<jsp:useBean id="form" class="FormData" scope="request" />       <!-- 表单数据 -->
<jsp:useBean id="temp" class="TempData" scope="page" />          <!-- 临时数据 -->
```

### 6.4 代码组织建议


**🔸 页面结构规范**
```jsp
<%-- 1. 页面指令和导入 --%>
<%@ page language="java" contentType="text/html; charset=UTF-8" %>
<%@ page import="com.example.*" %>

<%-- 2. Java逻辑代码（尽量简化） --%>

        // 登录处理逻辑
    }

<%-- 4. 页面内容 --%>

<html>

    <title>页面标题</title>
</head>
<body>
    <%-- 5. 页面包含 --%>
    <jsp:include page="header.jsp" />
    
    <%-- 6. 主要内容 --%>
    <div class="content">
        <!-- 页面具体内容 -->
    </div>
    
    <%-- 7. 页面转发（如果需要） --%>
    <% if (needForward) { %>

</html>
```

## 7. 📋 核心要点总结


### 7.1 必须掌握的核心概念


```
🔸 动作标签本质：运行时执行的XML格式标签，用于页面控制和对象操作
🔸 jsp:include：动态包含，运行时包含其他页面内容
🔸 jsp:forward：页面转发，服务器内部跳转，地址栏不变
🔸 jsp:param：参数传递，配合include和forward使用
🔸 jsp:useBean：创建或获取JavaBean对象
🔸 jsp:setProperty：设置JavaBean属性值
🔸 jsp:getProperty：获取JavaBean属性值
```

### 7.2 关键理解要点


**🔹 动作标签的执行时机**
```
编译时：JSP页面转换为Servlet类
运行时：每次请求都会执行动作标签
        ↓
动作标签的内容可能每次都不同（这就是"动态"的含义）
```

**🔹 参数传递的本质**
```
jsp:param的作用：
创建新的请求对象 → 包含原参数+新参数 → 传递给目标页面

目标页面获取参数：
request.getParameter("参数名") → 获取参数值
```

**🔹 作用域的选择原则**
```
page：   临时数据，页面处理完就不需要了
request：一次请求过程中需要的数据
session：用户会话期间需要的数据（如用户信息）
application：全局数据，所有用户共享（如系统配置）
```

### 7.3 实际应用价值


**🎯 解决的实际问题**
- **页面模块化**：通过include实现页面组件重用
- **用户体验**：通过forward实现无感知页面跳转
- **数据传递**：通过param实现页面间数据交换
- **对象管理**：通过JavaBean标签简化对象操作

**🔧 开发效率提升**
- **减少重复代码**：公共页面片段复用
- **简化页面跳转**：不需要编写复杂的Java代码
- **标准化对象操作**：统一的JavaBean操作方式
- **提高可维护性**：结构清晰，职责分离

**核心记忆口诀**：
- 动作标签运行时，功能明确各不同
- include包含用动态，forward转发地址不变
- param传参要配合，useBean对象好操作
- 作用域选择要合理，性能优化记心中
