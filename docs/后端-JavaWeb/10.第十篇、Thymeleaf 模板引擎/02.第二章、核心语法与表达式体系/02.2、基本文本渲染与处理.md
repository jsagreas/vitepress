## 1. 📝 文本渲染基础概念


### 1.1 什么是文本渲染


**🔸 大白话理解**
```
想象你在做一个网页，后端传来用户名"张三"
你需要把这个名字显示在页面上，这个过程就是"文本渲染"

简单说：把后端的数据显示到页面上的过程
```

**💡 为什么需要模板引擎渲染文本**

传统方式的问题：
```
❌ JSP需要嵌入大量Java代码，HTML结构混乱
❌ 纯HTML无法动态显示数据
❌ 前后端耦合严重，维护困难
```

Thymeleaf的优势：
```
✅ HTML语法友好，设计师也能看懂
✅ 自然模板，浏览器可以直接打开预览
✅ 语法简洁，学习成本低
```

### 1.2 Thymeleaf渲染流程图


```
后端Controller                  Thymeleaf引擎                  浏览器
    |                                |                           |
    |--[1]准备数据---------------->  |                           |
    |   username = "张三"            |                           |
    |                                |                           |
    |--[2]传递给模板--------------->  |                           |
    |   return "user"                |                           |
    |                                |                           |
    |                                |--[3]解析模板-------------  |
    |                                |   th:text="${username}"   |
    |                                |                           |
    |                                |--[4]替换内容-------------  |
    |                                |   <span>张三</span>       |
    |                                |                           |
    |                                |--[5]返回HTML------------->|
    |                                |                           |
    |                                |                      [6]显示页面
```

### 1.3 核心渲染属性对比


| 属性 | **用途** | **是否转义** | **使用场景** |
|------|---------|------------|------------|
| `th:text` | 渲染纯文本 | ✅ 自动转义HTML | 显示普通文本、用户输入 |
| `th:utext` | 渲染HTML内容 | ❌ 不转义 | 显示富文本、HTML片段 |
| `th:inline` | 行内文本 | ✅ 自动转义 | 简化文本插入 |

## 2. 🎯 th:text属性详解


### 2.1 基本语法和作用


**🔸 核心概念**
```
th:text = "Thymeleaf Text"（文本属性）
作用：把后端数据安全地显示为纯文本
特点：自动转义HTML标签，防止XSS攻击
```

**📋 基础使用示例**

后端Controller代码：
```java
@GetMapping("/user")
public String userPage(Model model) {
    model.addAttribute("username", "张三");
    model.addAttribute("age", 25);
    return "user";
}
```

前端Thymeleaf模板：
```html
<!-- 显示用户名 -->
<p th:text="${username}">默认用户名</p>

<!-- 显示年龄 -->
<span th:text="${age}">0</span>

<!-- 渲染后的HTML -->
<p>张三</p>
<span>25</span>
```

**💡 工作原理**

```
解析过程：
1. Thymeleaf引擎发现 th:text="${username}"
2. 从Model中获取 username 的值 "张三"
3. 替换整个标签的文本内容
4. 原来的"默认用户名"被完全替换掉
```

### 2.2 变量表达式详解


**🔸 ${}表达式（变量表达式）**

```html
<!-- 基本用法：获取Model中的属性 -->
<p th:text="${message}">默认消息</p>

<!-- 对象属性访问 -->
<p th:text="${user.name}">用户名</p>
<p th:text="${user.age}">年龄</p>

<!-- 调用对象方法 -->
<p th:text="${user.getName()}">获取用户名</p>
```

后端准备数据：
```java
User user = new User("李四", 30);
model.addAttribute("user", user);
```

**🔸 多层级对象访问**

```html
<!-- 深层属性访问 -->
<p th:text="${order.user.name}">订单用户名</p>
<p th:text="${order.product.price}">商品价格</p>

<!-- 对应的数据结构 -->
Order对象 {
    User user {
        name: "王五"
    }
    Product product {
        price: 99.99
    }
}
```

### 2.3 th:text的安全特性


**🔸 自动HTML转义**

```html
<!-- 后端传递的数据 -->
model.addAttribute("content", "<script>alert('危险')</script>");

<!-- 使用th:text渲染 -->
<p th:text="${content}">内容</p>

<!-- 实际渲染结果 -->
<p>&lt;script&gt;alert('危险')&lt;/script&gt;</p>

<!-- 浏览器显示为纯文本 -->
<script>alert('危险')</script>
```

**安全机制说明**：
```
🔒 th:text会自动转义特殊字符：
  < 转换为 &lt;
  > 转换为 &gt;
  & 转换为 &amp;
  " 转换为 &quot;

🛡️ 防止XSS攻击（跨站脚本攻击）
✅ 用户输入的内容永远显示为纯文本
```

### 2.4 常见使用场景


**📌 场景1：显示用户信息**

```html
<div class="user-info">
    <h3 th:text="${user.username}">用户名</h3>
    <p>年龄：<span th:text="${user.age}">0</span></p>
    <p>邮箱：<span th:text="${user.email}">email@example.com</span></p>
</div>
```

**📌 场景2：列表数据展示**

```html
<!-- 结合th:each使用 -->
<ul>
    <li th:each="item : ${items}" th:text="${item.name}">商品名称</li>
</ul>
```

**📌 场景3：条件文本显示**

```html
<!-- 结合三元运算符 -->
<p th:text="${user.vip} ? 'VIP用户' : '普通用户'">用户类型</p>

<!-- 使用Elvis运算符（默认值） -->
<p th:text="${user.nickname} ?: '匿名用户'">昵称</p>
```

## 3. 🔓 th:utext属性详解


### 3.1 基本概念和用途


**🔸 核心定义**
```
th:utext = "Thymeleaf Unescaped Text"（非转义文本）
作用：渲染HTML内容，不进行转义
用途：显示富文本、HTML片段

⚠️ 注意：只用于可信内容，不要用于用户输入！
```

**💡 与th:text的本质区别**

```
th:text（安全）：
输入：<b>加粗文本</b>
输出：&lt;b&gt;加粗文本&lt;/b&gt;
显示：<b>加粗文本</b>（纯文本）

th:utext（不转义）：
输入：<b>加粗文本</b>
输出：<b>加粗文本</b>
显示：加粗文本（真正加粗）
```

### 3.2 基础使用示例


**📋 渲染富文本内容**

后端代码：
```java
@GetMapping("/article")
public String article(Model model) {
    String content = "<h2>文章标题</h2><p>这是<strong>重要</strong>内容</p>";
    model.addAttribute("articleContent", content);
    return "article";
}
```

前端模板：
```html
<!-- 使用th:utext渲染HTML -->
<div th:utext="${articleContent}">
    默认内容
</div>

<!-- 渲染后的效果 -->
<div>
    <h2>文章标题</h2>
    <p>这是<strong>重要</strong>内容</p>
</div>
```

### 3.3 适用场景


**✅ 适合使用th:utext的场景**

```
🔸 场景1：富文本编辑器内容
  - 文章内容、博客正文
  - 商品详情描述
  - 用户发布的格式化内容（已过滤）

🔸 场景2：后端生成的HTML片段
  - 动态生成的表格、列表
  - 系统通知消息（HTML格式）

🔸 场景3：可信的静态HTML
  - 配置文件中的HTML模板
  - 系统预设的HTML内容
```

**❌ 不适合使用th:utext的场景**

```
⚠️ 危险场景：
  - 直接显示用户输入的内容
  - 来自外部API的未验证数据
  - URL参数中的内容
  - 表单提交的数据

🚨 安全风险：XSS攻击
```

### 3.4 安全使用建议


**🛡️ 安全实践**

```html
<!-- ❌ 危险做法：直接使用用户输入 -->
<div th:utext="${userInput}"></div>

<!-- ✅ 正确做法1：使用th:text -->
<div th:text="${userInput}"></div>

<!-- ✅ 正确做法2：后端过滤后使用 -->
<div th:utext="${sanitizedContent}"></div>
```

后端安全处理：
```java
// 使用HTML过滤库（如OWASP Java HTML Sanitizer）
String userInput = request.getParameter("content");
String safeContent = htmlSanitizer.sanitize(userInput);
model.addAttribute("sanitizedContent", safeContent);
```

**🔍 判断使用哪个属性**

```
决策流程：
                是否需要HTML效果？
                    /        \
                  否          是
                  ↓           ↓
            使用th:text   内容是否可信？
                              /        \
                            是          否
                            ↓           ↓
                      使用th:utext   先过滤再使用
                                      或使用th:text
```

## 4. 🔐 转义与非转义机制


### 4.1 什么是HTML转义


**🔸 通俗理解**

```
转义就像"翻译"特殊符号：
把浏览器认为是"代码"的符号，变成"文字"

比如：
< 是HTML标签的开始符号
如果你想显示"<"这个字符本身
就需要写成 &lt; （less than的缩写）
```

**常见转义字符对照**

| 原始字符 | **转义后** | **说明** | **示例** |
|---------|-----------|---------|---------|
| `<` | `&lt;` | 小于号 | `&lt;div&gt;` → `<div>` |
| `>` | `&gt;` | 大于号 | `&lt;/div&gt;` → `</div>` |
| `&` | `&amp;` | 与符号 | `Tom &amp; Jerry` → `Tom & Jerry` |
| `"` | `&quot;` | 双引号 | `&quot;hello&quot;` → `"hello"` |
| `'` | `&#39;` | 单引号 | `&#39;yes&#39;` → `'yes'` |

### 4.2 转义的工作原理


**📊 转义过程示意**

```
用户输入：  <script>alert('XSS')</script>
             ↓
Thymeleaf处理（th:text）
             ↓
转义后内容： &lt;script&gt;alert('XSS')&lt;/script&gt;
             ↓
浏览器渲染
             ↓
显示文本：  <script>alert('XSS')</script>（纯文本）
```

**🔸 为什么需要转义**

```
安全原因：防止XSS攻击

攻击示例：
1. 黑客在评论框输入：<script>发送Cookie到黑客服务器</script>
2. 如果不转义直接显示，脚本会执行
3. 用户的Cookie被盗取

转义后：
<script>标签变成纯文本，不会执行
```

### 4.3 转义与非转义对比


**实际效果对比**

后端数据：
```java
String html = "<h1>标题</h1><p>段落内容</p>";
model.addAttribute("content", html);
```

前端渲染对比：
```html
<!-- 方式1：使用th:text（转义） -->
<div th:text="${content}">内容</div>
<!-- 渲染结果 -->
<div>&lt;h1&gt;标题&lt;/h1&gt;&lt;p&gt;段落内容&lt;/p&gt;</div>
<!-- 浏览器显示 -->
<h1>标题</h1><p>段落内容</p>


<!-- 方式2：使用th:utext（不转义） -->
<div th:utext="${content}">内容</div>
<!-- 渲染结果 -->
<div><h1>标题</h1><p>段落内容</p></div>
<!-- 浏览器显示 -->
标题（大号加粗）
段落内容
```

### 4.4 特殊字符处理技巧


**🔧 混合内容处理**

```html
<!-- 需求：显示"价格 < 100元"这样的文本 -->

<!-- ❌ 错误写法 -->
<p th:text="'价格 < 100元'">价格</p>
<!-- 会报错，因为<被认为是标签开始 -->

<!-- ✅ 正确写法1：使用转义 -->
<p th:text="'价格 &lt; 100元'">价格</p>

<!-- ✅ 正确写法2：使用单引号包裹 -->
<p th:text="'价格 < 100元'">价格</p>

<!-- ✅ 正确写法3：使用字符串拼接 -->
<p th:text="'价格 ' + '<' + ' 100元'">价格</p>
```

## 5. ➕ 字符串连接操作


### 5.1 基本连接方式


**🔸 方式1：使用 + 运算符**

```html
<!-- 简单拼接 -->
<p th:text="'Hello, ' + ${username}">问候语</p>

<!-- 多个变量拼接 -->
<p th:text="${firstName} + ' ' + ${lastName}">全名</p>

<!-- 拼接固定文本和变量 -->
<p th:text="'用户年龄：' + ${age} + '岁'">年龄信息</p>
```

后端数据：
```java
model.addAttribute("username", "张三");
model.addAttribute("firstName", "张");
model.addAttribute("lastName", "三");
model.addAttribute("age", 25);
```

渲染结果：
```html
<p>Hello, 张三</p>
<p>张 三</p>
<p>用户年龄：25岁</p>
```

**🔸 方式2：使用字面量替换**

```html
<!-- 使用 | | 包裹，内部可以直接使用${} -->
<p th:text="|Hello, ${username}!|">问候语</p>

<!-- 多变量替换 -->
<p th:text="|${firstName} ${lastName}，今年 ${age} 岁|">用户信息</p>

<!-- 优势：语法更简洁直观 -->
```

### 5.2 字符串连接场景


**📌 场景1：构建完整消息**

```html
<!-- 欢迎消息 -->
<h2 th:text="|欢迎您，${user.name}！|">欢迎消息</h2>

<!-- 订单信息 -->
<p th:text="|订单号：${order.id}，总价：${order.total}元|">订单</p>

<!-- 动态标题 -->
<title th:text="|${pageTitle} - 我的网站|">页面标题</title>
```

**📌 场景2：URL路径拼接**

```html
<!-- 动态链接 -->
<a th:href="'/user/' + ${userId}">用户详情</a>

<!-- 带参数的URL -->
<a th:href="|/search?keyword=${keyword}&page=${page}|">搜索结果</a>

<!-- 图片路径 -->
<img th:src="|/images/avatar/${user.id}.jpg|" alt="头像">
```

**📌 场景3：条件拼接**

```html
<!-- 根据条件拼接不同文本 -->
<p th:text="|用户状态：${user.active ? '活跃' : '已禁用'}|">状态</p>

<!-- 复杂拼接 -->
<p th:text="|${user.name}（${user.vip ? 'VIP会员' : '普通会员'}）|">用户</p>
```

### 5.3 字符串操作方法


**🔧 Thymeleaf内置字符串工具**

```html
<!-- 转大写 -->
<p th:text="${#strings.toUpperCase(name)}">NAME</p>

<!-- 转小写 -->
<p th:text="${#strings.toLowerCase(name)}">name</p>

<!-- 截取字符串 -->
<p th:text="${#strings.substring(text, 0, 10)}">前10个字符</p>

<!-- 替换内容 -->
<p th:text="${#strings.replace(text, 'old', 'new')}">替换后</p>

<!-- 判断是否为空 -->
<p th:if="${!#strings.isEmpty(message)}" th:text="${message}">消息</p>
```

**实际应用示例**

```html
<!-- 用户昵称首字母大写 -->
<p th:text="${#strings.capitalize(nickname)}">昵称</p>

<!-- 限制文本长度（省略号） -->
<p th:text="${#strings.abbreviate(description, 50)}">描述</p>

<!-- 组合使用 -->
<p th:text="|编号：${#strings.toUpperCase(code)}|">编号</p>
```

## 6. 🔄 EL表达式兼容


### 6.1 什么是EL表达式


**🔸 基本概念**

```
EL = Expression Language（表达式语言）
来源：JSP技术，用于在页面中访问数据
Thymeleaf兼容类似的语法，降低学习成本
```

**传统JSP的EL表达式**：
```jsp
<!-- JSP写法 -->
${user.name}
${sessionScope.username}
```

**Thymeleaf的兼容写法**：
```html
<!-- Thymeleaf写法（兼容EL） -->
<p th:text="${user.name}">用户名</p>
<p th:text="${session.username}">会话用户</p>
```

### 6.2 常用EL风格语法


**🔸 对象属性访问**

```html
<!-- 点号访问 -->
<p th:text="${user.name}">用户名</p>
<p th:text="${order.product.price}">价格</p>

<!-- 等价的括号访问 -->
<p th:text="${user['name']}">用户名</p>
<p th:text="${order['product']['price']}">价格</p>
```

**🔸 集合元素访问**

```html
<!-- 数组/List访问 -->
<p th:text="${users[0].name}">第一个用户</p>
<p th:text="${scores[2]}">第三个分数</p>

<!-- Map访问 -->
<p th:text="${userMap['admin'].name}">管理员名称</p>
<p th:text="${config['app.name']}">应用名称</p>
```

### 6.3 作用域对象访问


**🔸 Web作用域对象**

```html
<!-- 请求作用域（默认） -->
<p th:text="${username}">用户名</p>
<p th:text="${param.id}">请求参数ID</p>

<!-- Session作用域 -->
<p th:text="${session.user.name}">会话用户</p>

<!-- Application作用域 -->
<p th:text="${application.appName}">应用名称</p>
```

**对应的后端设置**：
```java
// Request作用域
model.addAttribute("username", "张三");

// Session作用域
session.setAttribute("user", user);

// Application作用域
servletContext.setAttribute("appName", "我的应用");
```

### 6.4 运算符支持


**🔸 算术运算**

```html
<!-- 加减乘除 -->
<p th:text="${price + tax}">含税价格</p>
<p th:text="${total - discount}">折后价</p>
<p th:text="${quantity * price}">总价</p>
<p th:text="${total / count}">平均值</p>

<!-- 取模 -->
<p th:text="${number % 2 == 0 ? '偶数' : '奇数'}">奇偶</p>
```

**🔸 比较运算**

```html
<!-- 数值比较 -->
<p th:if="${age >= 18}">成年人</p>
<p th:if="${score > 60}">及格</p>

<!-- 文本比较 -->
<p th:if="${status == 'active'}">活跃状态</p>
<p th:if="${role != 'admin'}">非管理员</p>
```

**🔸 逻辑运算**

```html
<!-- AND运算 -->
<p th:if="${age >= 18 && hasLicense}">可以开车</p>

<!-- OR运算 -->
<p th:if="${isVip || isMember}">会员特权</p>

<!-- NOT运算 -->
<p th:if="${!isDeleted}">有效记录</p>
```

## 7. 📝 占位符语法使用


### 7.1 基本占位符语法


**🔸 什么是占位符**

```
占位符 = 预留的位置 + 动态填充的值
就像填空题：我的名字是____（占位符）
运行时填入：我的名字是张三（实际值）
```

**Thymeleaf的占位符语法**：
```html
<!-- 基础语法：#{...} -->
<p th:text="#{welcome.message}">欢迎消息</p>

<!-- 带参数的占位符 -->
<p th:text="#{user.greeting(${username})}">用户问候</p>
```

### 7.2 国际化消息占位符


**🔸 配置消息文件**

messages.properties（中文）：
```properties
welcome.message=欢迎访问我们的网站
user.greeting=你好，{0}！
order.info=订单号：{0}，金额：{1}元
```

messages_en.properties（英文）：
```properties
welcome.message=Welcome to our website
user.greeting=Hello, {0}!
order.info=Order: {0}, Amount: ${1}
```

**🔸 模板使用**

```html
<!-- 简单消息 -->
<h1 th:text="#{welcome.message}">欢迎</h1>

<!-- 带一个参数 -->
<p th:text="#{user.greeting(${username})}">问候</p>

<!-- 带多个参数 -->
<p th:text="#{order.info(${orderId}, ${amount})}">订单信息</p>
```

后端代码：
```java
model.addAttribute("username", "张三");
model.addAttribute("orderId", "20231001");
model.addAttribute("amount", 199.99);
```

渲染结果（中文环境）：
```html
<h1>欢迎访问我们的网站</h1>
<p>你好，张三！</p>
<p>订单号：20231001，金额：199.99元</p>
```

### 7.3 URL占位符


**🔸 路径变量占位符**

```html
<!-- URL路径占位符 -->
<a th:href="@{/user/{id}(id=${userId})}">用户详情</a>

<!-- 多个占位符 -->
<a th:href="@{/order/{orderId}/item/{itemId}(orderId=${oid}, itemId=${iid})}">
    订单项详情
</a>

<!-- 渲染结果 -->
<a href="/user/123">用户详情</a>
<a href="/order/456/item/789">订单项详情</a>
```

**🔸 查询参数占位符**

```html
<!-- 单个查询参数 -->
<a th:href="@{/search(keyword=${keyword})}">搜索</a>

<!-- 多个查询参数 -->
<a th:href="@{/list(page=${page}, size=${size})}">列表</a>

<!-- 混合路径变量和查询参数 -->
<a th:href="@{/user/{id}/posts(id=${userId}, status=${status})}">
    用户帖子
</a>

<!-- 渲染结果 -->
<a href="/search?keyword=Java">搜索</a>
<a href="/list?page=1&size=10">列表</a>
<a href="/user/123/posts?status=active">用户帖子</a>
```

### 7.4 内联文本占位符


**🔸 [[...]] 内联语法**

```html
<!-- 传统写法 -->
<p th:text="${message}">消息</p>

<!-- 内联写法 -->
<p>消息：[[${message}]]</p>

<!-- 多个变量内联 -->
<p>用户 [[${username}]] 的积分是 [[${points}]]</p>
```

**🔸 [(...)]] 非转义内联**

```html
<!-- 转义内联（安全） -->
<p>内容：[[${htmlContent}]]</p>

<!-- 非转义内联（用于HTML） -->
<div>[(${htmlContent})]</div>

<!-- 等价于 -->
<p th:text="${htmlContent}">内容</p>
<div th:utext="${htmlContent}"></div>
```

## 8. 📋 核心要点总结


### 8.1 六大核心知识点


```
🔸 th:text  → 安全渲染文本，自动转义HTML
🔸 th:utext → 渲染HTML内容，不转义（仅用于可信数据）
🔸 转义机制  → 防止XSS攻击的安全保障
🔸 字符串连接 → + 运算符 和 |...| 语法
🔸 EL兼容   → 降低学习成本，支持常见表达式
🔸 占位符   → 国际化消息、URL构建、内联文本
```

### 8.2 安全使用决策树


```
                   需要显示后端数据？
                       /        \
                     是          否
                     ↓           ↓
              数据来源可信？    直接写HTML
                 /        \
               是          否
               ↓           ↓
          需要HTML效果？   使用th:text
             /        \    （永远安全）
           是          否
           ↓           ↓
      使用th:utext   使用th:text
      （富文本等）   （普通文本）
```

### 8.3 语法选择建议


**文本渲染选择**

| 需求 | **推荐语法** | **原因** |
|------|------------|---------|
| 普通文本 | `th:text` | 安全，防XSS |
| 富文本内容 | `th:utext` | 保留HTML格式 |
| 简短文本 | `[[${...}]]` | 语法简洁 |
| HTML片段 | `[(${...})]` | 不转义内联 |

**字符串拼接选择**

| 场景 | **推荐语法** | **示例** |
|------|------------|---------|
| 简单拼接 | `\|...\|` | `\|Hello, ${name}!\|` |
| 复杂运算 | `+` 运算符 | `${a} + '-' + ${b}` |
| URL构建 | `@{...}` | `@{/user/{id}(id=${uid})}` |

### 8.4 常见问题速查


**❓ Q1：什么时候用th:text，什么时候用th:utext？**

```
✅ 用th:text：
  - 显示用户输入（评论、昵称等）
  - 普通文本数据
  - 不需要HTML格式的内容

✅ 用th:utext：
  - 富文本编辑器内容（已过滤）
  - 后端生成的HTML片段
  - 确认安全的HTML内容
```

**❓ Q2：如何显示特殊字符（如 < > &）？**

```
方式1：使用转义字符
<p th:text="'价格 &lt; 100'">价格</p>

方式2：使用字符串拼接
<p th:text="'价格 ' + '<' + ' 100'">价格</p>

方式3：使用内联语法
<p>价格 [[${price}]] < 100</p>
```

**❓ Q3：字符串拼接哪种方式最好？**

```
推荐 |...| 语法：
✅ 语法简洁直观
✅ 可读性强
✅ 不需要繁琐的引号和加号

示例：
|用户${name}，积分${points}| ← 清晰
${name} + '，积分' + ${points} ← 繁琐
```
