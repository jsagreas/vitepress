## 1. 🗺️ JDBC学习路线图


### 1.1 什么是JDBC？为什么要学？


> **通俗理解**：JDBC就像是Java程序与数据库之间的"翻译官"，帮助Java程序和数据库进行对话。

**🔍 生活中的例子**
```
就像你去国外旅行：
你（Java程序）想和当地人（数据库）交流
但你们说不同的语言
这时需要一个翻译（JDBC）来帮忙沟通

Java程序 ←→ JDBC翻译官 ←→ 数据库
   |           |            |
 发出请求    翻译转换      执行操作
```

**🎯 学习JDBC的核心价值**

| **价值** | **具体体现** | **实际应用** |
|---------|-------------|-------------|
| **数据持久化** | 程序数据需要永久保存 | 用户注册信息、订单记录 |
| **数据管理** | 增删改查数据操作 | 商品管理、用户管理 |
| **企业必备** | Web项目核心技术 | 电商网站、管理系统 |
| **框架基础** | MyBatis、Hibernate底层 | 理解框架工作原理 |

### 2.3 JDBC驱动依赖


**📚 Maven依赖配置**

> **什么是JDBC驱动？** 
> 简单说就是让Java能够"听懂"MySQL语言的插件，不同数据库需要不同的驱动。

```xml
<!-- MySQL 8.x 驱动（推荐） -->
<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-java</artifactId>
    <version>8.0.33</version>
</dependency>

<!-- MySQL 5.x 驱动（兼容版本） -->
<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-java</artifactId>
    <version>5.1.49</version>
</dependency>
```

   ↓

   ↓

| **配置项** | **推荐值** | **说明** |

## 5. 📊 示例数据库设计


### 5.1 学习用数据库表设计


**🏗️ 实战数据库设计**

> **设计思路**：我们设计一个简单的"学生管理系统"，包含学生信息、课程信息和选课记录，这样能覆盖JDBC的各种操作场景。

```sql
-- 创建数据库
CREATE DATABASE jdbc_demo 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;

USE jdbc_demo;
```

### 5.2 核心表结构


**👨‍🎓 学生表(students)**

```sql
CREATE TABLE students (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '学生ID',
    name VARCHAR(50) NOT NULL COMMENT '学生姓名',
    age INT NOT NULL COMMENT '年龄',
    gender ENUM('男', '女') NOT NULL COMMENT '性别',
    email VARCHAR(100) UNIQUE COMMENT '邮箱',
    phone VARCHAR(20) COMMENT '电话号码',
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
);
```

**📚 课程表(courses)**

```sql
CREATE TABLE courses (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '课程ID',
    course_name VARCHAR(100) NOT NULL COMMENT '课程名称',
    course_code VARCHAR(20) UNIQUE NOT NULL COMMENT '课程编码',
    credits INT NOT NULL DEFAULT 1 COMMENT '学分',
    teacher VARCHAR(50) COMMENT '授课老师',
    description TEXT COMMENT '课程描述'
);
```

**📋 选课记录表(enrollments)**

```sql
CREATE TABLE enrollments (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '选课记录ID',
    student_id INT NOT NULL COMMENT '学生ID',
    course_id INT NOT NULL COMMENT '课程ID',
    score DECIMAL(5,2) DEFAULT NULL COMMENT '分数',
    enroll_date DATE NOT NULL COMMENT '选课日期',
    status ENUM('已选', '已完成', '已退课') DEFAULT '已选' COMMENT '状态',
    FOREIGN KEY (student_id) REFERENCES students(id),
    FOREIGN KEY (course_id) REFERENCES courses(id),
    UNIQUE KEY unique_enrollment (student_id, course_id)
);
```

### 6.4 常见学习难点预警


**⚠️ 新手容易踩的坑**

```
学习难点分析：

环境配置问题 → 驱动版本冲突 → SQL语法错误 → 异常处理不当
      |              |              |              |
   端口/密码错误    新旧版本API    大小写敏感    资源没关闭
      |              |              |              |
  仔细检查配置    选择合适版本    使用规范SQL   用try-with-resources
```

**🔧 解决策略**
- **环境问题**：按步骤逐一验证，不要跳步骤
- **版本兼容**：选择稳定版本，避免使用最新测试版
- **SQL问题**：使用数据库工具先测试SQL语句
- **异常处理**：学会看日志，理解异常信息含义
