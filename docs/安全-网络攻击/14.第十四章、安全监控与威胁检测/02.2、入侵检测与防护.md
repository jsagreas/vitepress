## 1. 🔍 入侵检测系统概述


### 1.1 什么是入侵检测


**入侵检测**就像是网络世界中的**"安全监控摄像头"**，时刻监视着网络中的活动，发现异常行为就立即报警。

```
生活中的类比：
银行的监控系统 → 入侵检测系统(IDS)
• 监控摄像头 → 网络流量监控
• 安保人员 → 安全分析师
• 报警系统 → 告警机制
• 可疑行为识别 → 攻击模式识别
```

### 1.2 IDS与IPS的区别


**🔸 IDS（入侵检测系统）**
```
作用：发现 + 报警
工作方式：被动监控，发现威胁后报告

就像家里的烟雾报警器：
• 检测到烟雾 → 发出警报
• 但不会自动灭火
• 需要人工干预处理
```

**🔸 IPS（入侵防护系统）**
```
作用：发现 + 阻止
工作方式：主动防护，发现威胁立即阻断

就像自动灭火系统：
• 检测到火灾 → 自动喷水灭火
• 既能发现也能处理
• 实时响应威胁
```

### 1.3 检测系统的价值


**💡 为什么需要入侵检测？**

```
传统防护的局限性：
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   防火墙     │    │   杀毒软件   │    │   访问控制   │
│  (边界防护)  │    │  (已知威胁)  │    │  (权限管理)  │
└─────────────┘    └─────────────┘    └─────────────┘
       ↓                  ↓                  ↓
   只能防已知      只能检测已知病毒    无法检测内部威胁

入侵检测的补充价值：
✅ 检测未知攻击
✅ 发现内部威胁  
✅ 攻击溯源分析
✅ 安全态势感知
```

## 3. 🔬 检测技术原理深入


### 3.1 签名检测技术


**🔸 什么是签名检测？**

签名检测就像是**"通缉令比对"**，系统里存储了已知攻击的"指纹"，一旦发现匹配的模式就立即报警。

```
工作原理：
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  网络流量    │ → │  模式匹配    │ → │  告警输出    │
│ (数据包)     │    │ (规则库)     │    │ (安全事件)   │
└─────────────┘    └─────────────┘    └─────────────┘

例如检测SQL注入：
流量内容："' OR 1=1--"
匹配规则：content:"OR 1=1"
检测结果：SQL注入攻击告警
```

**💡 签名检测的特点**
```
优点：
✅ 准确率高（误报少）
✅ 响应速度快
✅ 易于理解和维护

缺点：
❌ 只能检测已知攻击
❌ 需要定期更新规则库
❌ 容易被变形攻击绕过
```

### 3.2 异常行为检测


**🔸 行为基线建立**

异常检测就像是**"了解一个人的正常行为"**，先观察正常情况，然后识别异常。

```
基线学习过程：
第1周：观察正常流量模式
• 用户登录时间：9:00-18:00
• 平均数据传输：10MB/小时
• 常用端口：80, 443, 22

第2周：建立行为模型
• 登录时间模型
• 流量大小模型  
• 端口访问模型

检测阶段：发现异常
• 凌晨3点大量登录 → 异常
• 突然传输100GB数据 → 异常
• 访问异常端口 → 异常
```

**🔧 异常检测算法示例**
```python
# 简化的流量异常检测
class TrafficAnomalyDetector:
    def __init__(self):
        self.baseline_traffic = 0  # 基线流量
        self.threshold = 2.0       # 异常阈值
    
    def learn_baseline(self, traffic_samples):
        """学习正常流量基线"""
        self.baseline_traffic = sum(traffic_samples) / len(traffic_samples)
        print(f"基线流量: {self.baseline_traffic} MB/h")
    
    def detect_anomaly(self, current_traffic):
        """检测流量异常"""
        ratio = current_traffic / self.baseline_traffic
        
        if ratio > self.threshold:
            return {
                'alert': True,
                'message': f'流量异常: {current_traffic}MB/h (正常: {self.baseline_traffic}MB/h)',
                'severity': 'HIGH' if ratio > 5 else 'MEDIUM'
            }
        return {'alert': False}

# 使用示例
detector = TrafficAnomalyDetector()
detector.learn_baseline([10, 12, 8, 11, 9])  # 学习基线
result = detector.detect_anomaly(50)  # 检测当前流量
```

### 3.3 检测技术对比


| 检测方式 | **工作原理** | **适用场景** | **优缺点** |
|---------|------------|-------------|-----------|
| 🔍 **签名检测** | `已知攻击模式匹配` | `已知威胁检测` | `准确但有盲区` |
| 📊 **异常检测** | `偏离正常行为基线` | `0day攻击检测` | `全面但误报多` |
| 🧠 **启发式检测** | `攻击行为特征分析` | `变形攻击检测` | `平衡准确性` |
| 🤖 **机器学习** | `AI模型自动识别` | `复杂攻击检测` | `智能但需训练` |

### 4.2 深度包检测（DPI）


**🔸 DPI技术原理**

深度包检测就像是**"安检X光机"**，不仅看包裹外表，还要透视内部内容。

```
检测层次对比：
传统检测：只看信封
┌─────────────────┐
│ 源IP: 1.1.1.1   │ ← 只检查这些
│ 目标IP: 2.2.2.2 │
│ 端口: 80        │
└─────────────────┘

DPI检测：拆开信件看内容
┌─────────────────┐
│ HTTP请求头      │
│ GET /admin.php  │ ← 检查应用层内容
│ Cookie信息      │
│ POST数据        │
└─────────────────┘
```

**🔧 DPI检测示例**
```python
# 简化的HTTP内容检测
def deep_packet_inspection(packet_data):
    """深度包检测示例"""
    
    # 1. 协议识别
    if 'HTTP' in packet_data:
        protocol = 'HTTP'
    elif 'SMTP' in packet_data:
        protocol = 'SMTP'
    else:
        protocol = 'UNKNOWN'
    
    # 2. 内容检测
    threats = []
    
    # 检测SQL注入
    if "' OR 1=1" in packet_data:
        threats.append('SQL_INJECTION')
    
    # 检测XSS攻击
    if '<script>' in packet_data.lower():
        threats.append('XSS_ATTACK')
    
    # 检测文件上传
    if 'Content-Type: multipart/form-data' in packet_data:
        if '.php' in packet_data or '.jsp' in packet_data:
            threats.append('MALICIOUS_UPLOAD')
    
    return {
        'protocol': protocol,
        'threats': threats,
        'risk_level': 'HIGH' if threats else 'LOW'
    }
```

### 5.2 文件完整性监控


**🔸 文件完整性检查原理**

```
工作流程：
1. 基线建立阶段：
   计算重要文件的哈希值
   /etc/passwd     → MD5: a1b2c3d4...
   /etc/shadow     → MD5: e5f6g7h8...
   /bin/ls         → MD5: i9j0k1l2...

2. 定期检查阶段：
   重新计算哈希值并对比
   /etc/passwd     → MD5: a1b2c3d4... ✓ 未变化
   /etc/shadow     → MD5: e5f6g7h8... ✓ 未变化  
   /bin/ls         → MD5: x9y8z7w6... ✗ 文件被修改！

3. 告警处理：
   发现文件被非授权修改，立即告警
```

**🔧 简单的文件监控脚本**
```bash
#!/bin/bash
# 文件完整性监控脚本

BASELINE_FILE="/var/log/file_baseline.txt"
CURRENT_CHECK="/tmp/current_check.txt"

# 重要文件列表
CRITICAL_FILES=(
    "/etc/passwd"
    "/etc/shadow"
    "/etc/sudoers"
    "/bin/bash"
    "/usr/bin/sudo"
)

# 生成当前哈希值
for file in "${CRITICAL_FILES[@]}"; do
    if [[ -f "$file" ]]; then
        md5sum "$file" >> "$CURRENT_CHECK"
    fi
done

# 与基线对比
if [[ -f "$BASELINE_FILE" ]]; then
    if ! diff "$BASELINE_FILE" "$CURRENT_CHECK" > /dev/null; then
        echo "⚠️  文件完整性检查失败！"
        echo "变化的文件："
        diff "$BASELINE_FILE" "$CURRENT_CHECK"
        
        # 发送告警邮件
        mail -s "文件完整性告警" admin@company.com < /tmp/alert.txt
    else
        echo "✅ 文件完整性检查通过"
    fi
else
    # 首次运行，建立基线
    cp "$CURRENT_CHECK" "$BASELINE_FILE"
    echo "📝 基线文件已建立"
fi
```

### 5.3 系统行为监控


**🔸 进程监控**

```
异常进程检测：
正常进程特征：
• 已知的系统进程
• 合理的CPU/内存使用
• 正常的网络连接

异常进程特征：
• 未知的可执行文件
• 异常高的资源使用
• 可疑的网络连接
• 隐藏进程行为
```

**🔧 进程异常检测**
```python
import psutil
import time

class ProcessMonitor:
    def __init__(self):
        self.baseline_processes = set()
        self.suspicious_names = ['nc', 'ncat', 'telnet', 'wget', 'curl']
    
    def learn_baseline(self):
        """学习正常进程基线"""
        for proc in psutil.process_iter(['name', 'exe']):
            try:
                self.baseline_processes.add(proc.info['name'])
            except:
                pass
        print(f"基线进程数量: {len(self.baseline_processes)}")
    
    def detect_anomalies(self):
        """检测异常进程"""
        alerts = []
        
        for proc in psutil.process_iter(['pid', 'name', 'exe', 'cpu_percent']):
            try:
                name = proc.info['name']
                
                # 检测新进程
                if name not in self.baseline_processes:
                    alerts.append(f"新进程发现: {name} (PID: {proc.info['pid']})")
                
                # 检测可疑进程名
                if any(sus in name.lower() for sus in self.suspicious_names):
                    alerts.append(f"可疑进程: {name} (PID: {proc.info['pid']})")
                
                # 检测高CPU使用率
                if proc.info['cpu_percent'] > 80:
                    alerts.append(f"高CPU使用: {name} ({proc.info['cpu_percent']}%)")
                    
            except:
                continue
        
        return alerts
```

## 6. 🍯 蜜罐技术与攻击诱捕


### 6.1 蜜罐技术概述


**蜜罐（Honeypot）**就像是**"诱饵陷阱"**，故意放置看似有价值但实际是陷阱的目标，吸引攻击者上钩。

```
蜜罐工作原理：
真实网络环境：
┌─────────────────────────────────────┐
│  ┌─────────┐  ┌─────────┐  ┌──────┐ │
│  │ Web服务器│  │ 数据库   │  │ 蜜罐  │ │
│  │(真实)   │  │(真实)   │  │(陷阱) │ │
│  └─────────┘  └─────────┘  └──────┘ │
└─────────────────────────────────────┘
                    ↑
                攻击者被蜜罐吸引
                进入监控环境
```

### 6.2 蜜罐分类与部署


**🔸 按交互程度分类**

| 蜜罐类型 | **交互程度** | **部署难度** | **检测能力** | **风险程度** |
|---------|------------|-------------|-------------|-------------|
| 🍯 **低交互蜜罐** | `模拟基本服务` | `简单` | `基础攻击` | `低风险` |
| 🍯 **中交互蜜罐** | `模拟常用应用` | `中等` | `复杂攻击` | `中等风险` |
| 🍯 **高交互蜜罐** | `完整操作系统` | `复杂` | `高级攻击` | `高风险` |

**🔧 简单蜜罐搭建示例**

## 7. 🔒 端口隐藏与重定向技术


### 7.1 端口隐藏技术


**端口隐藏**就像是给房子装**"隐形门"**，真实的服务端口被隐藏，只有知道"暗号"的人才能访问。

### 7.2 端口敲门（Port Knocking）


**🔸 端口敲门原理**

```
工作流程：
1. 正常状态：所有端口关闭
   端口22(SSH)：关闭 ❌
   端口80(HTTP)：关闭 ❌
   端口443(HTTPS)：关闭 ❌

2. 敲门序列：按顺序访问特定端口
   客户端 → 1234端口 (第1次敲门)
   客户端 → 5678端口 (第2次敲门)  
   客户端 → 9012端口 (第3次敲门)

3. 验证成功：目标端口打开
   端口22(SSH)：开放 ✅ (仅对该IP开放)
```

**🔧 端口敲门配置示例**
```bash
# knockd配置文件 /etc/knockd.conf
[options]
logfile = /var/log/knockd.log

[openSSH]
sequence    = 1234,5678,9012
seq_timeout = 5
command     = /sbin/iptables -I INPUT -s %IP% -p tcp --dport 22 -j ACCEPT
tcpflags    = syn

[closeSSH]  
sequence    = 9012,5678,1234
seq_timeout = 5
command     = /sbin/iptables -D INPUT -s %IP% -p tcp --dport 22 -j ACCEPT
tcpflags    = syn

# 客户端敲门脚本
#!/bin/bash
TARGET_HOST="192.168.1.100"
knock $TARGET_HOST 1234 5678 9012
ssh user@$TARGET_HOST
knock $TARGET_HOST 9012 5678 1234  # 关闭端口
```

### 7.3 端口重定向技术


**🔸 iptables端口重定向**

```bash
# 场景：将真实SSH端口22隐藏，通过8022端口访问

# 1. 修改SSH服务端口
echo "Port 2222" >> /etc/ssh/sshd_config
systemctl restart sshd

# 2. 设置端口重定向
# 外部8022端口 → 内部2222端口
iptables -t nat -A PREROUTING -p tcp --dport 8022 -j REDIRECT --to-port 2222

# 3. 阻止直接访问22端口
iptables -A INPUT -p tcp --dport 22 -j DROP

# 4. 客户端访问方式
ssh -p 8022 user@server_ip  # 通过8022端口访问SSH
```

**🔸 反向代理隐藏**

```bash
# 使用nginx隐藏真实服务端口
# nginx配置示例

upstream hidden_service {
    server 127.0.0.1:8080;  # 真实服务端口
}

server {
    listen 80;
    server_name example.com;
    
    location /secret_path/ {
        proxy_pass http://hidden_service/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        
        # 访问控制
        allow 192.168.1.0/24;  # 只允许内网访问
        deny all;
    }
    
    location / {
        return 404;  # 其他路径返回404
    }
}
```
