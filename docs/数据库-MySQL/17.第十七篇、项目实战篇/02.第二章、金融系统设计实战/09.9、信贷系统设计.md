## 1. 💼 信贷系统概述


### 1.1 什么是信贷系统


**简单理解**：信贷系统就是银行放贷的"智能管家"，从客户申请贷款到最后还清款项的全过程管理系统。

**核心作用**：
```
申请阶段：审核客户资质，评估还款能力
放款阶段：控制放款流程，确保资金安全
还款阶段：管理还款计划，监控还款情况
风控阶段：识别风险，防范坏账损失
```

### 1.2 信贷生命周期管理


**完整流程图**：
```
客户申请 → 资质审核 → 风险评估 → 额度审批 → 放款执行
    ↓
还款管理 ← 贷后监控 ← 合同签署 ← 额度确定 ← 定价决策
    ↓
逾期处理 → 催收管理 → 不良资产处置 → 账务核销
```

**三个关键阶段**：
- **贷前管理**：客户筛选、风险评估、额度审批
- **贷中管理**：合同管理、放款控制、资金监控
- **贷后管理**：还款跟踪、逾期处理、风险预警

**放款前置检查**：
```sql
-- 放款风控检查函数
DELIMITER //
CREATE FUNCTION check_disbursement_risk(p_loan_id VARCHAR(32))
RETURNS VARCHAR(20)
READS SQL DATA
DETERMINISTIC
BEGIN
    DECLARE v_overdue_count INT DEFAULT 0;
    DECLARE v_debt_ratio DECIMAL(8,4) DEFAULT 0;
    DECLARE v_result VARCHAR(20) DEFAULT 'pass';
    
    -- 检查客户逾期记录
    SELECT COUNT(*) INTO v_overdue_count
    FROM loan_repayment 
    WHERE customer_id = (SELECT customer_id FROM loans WHERE loan_id = p_loan_id)
    AND overdue_days > 30
    AND created_at > DATE_SUB(NOW(), INTERVAL 12 MONTH);
    
    -- 检查负债比率
    SELECT total_debt_ratio INTO v_debt_ratio
    FROM customer_financial_profile 
    WHERE customer_id = (SELECT customer_id FROM loans WHERE loan_id = p_loan_id);
    
    -- 风险判断逻辑
    IF v_overdue_count > 2 OR v_debt_ratio > 0.7 THEN
        SET v_result = 'reject';
    ELSEIF v_overdue_count > 0 OR v_debt_ratio > 0.5 THEN
        SET v_result = 'warning';
    END IF;
    
    RETURN v_result;
END //
DELIMITER ;
```

## 5. 📅 还款计划生成


### 5.1 还款方式理解


**常见还款方式对比**：

| 还款方式 | **特点** | **适用场景** | **优缺点** |
|---------|---------|-------------|-----------|
| **等额本息** | `每月还款金额相等` | `工薪族稳定收入` | `前期利息多，总利息较高` |
| **等额本金** | `每月本金相等，利息递减` | `收入递减人群` | `前期压力大，总利息较少` |
| **先息后本** | `只还利息，到期还本金` | `短期周转，资金紧张` | `压力集中在最后` |
| **灵活还款** | `根据现金流灵活安排` | `收入不稳定人群` | `管理复杂，风险较高` |

### 5.2 还款计划表设计


**还款计划主表**：
```sql
-- 还款计划表

    principal_amount DECIMAL(12,2) NOT NULL COMMENT '应还本金',
    interest_amount DECIMAL(12,2) NOT NULL COMMENT '应还利息',
    total_amount DECIMAL(12,2) AS (principal_amount + interest_amount) COMMENT '应还总额',
    outstanding_principal DECIMAL(15,2) COMMENT '剩余本金',
```

### 5.3 还款计划生成算法


**等额本息计算**：
```sql
-- 等额本息还款计划生成存储过程
DELIMITER //
CREATE PROCEDURE generate_equal_payment_plan(
    IN p_loan_id VARCHAR(32),
    IN p_principal DECIMAL(15,2),
    IN p_rate DECIMAL(8,6),
    IN p_periods INT
)
BEGIN
    DECLARE v_monthly_payment DECIMAL(12,2);
    DECLARE v_period INT DEFAULT 1;
    DECLARE v_remaining_principal DECIMAL(15,2) DEFAULT p_principal;
    DECLARE v_interest_payment DECIMAL(12,2);
    DECLARE v_principal_payment DECIMAL(12,2);
    DECLARE v_due_date DATE;
    
    -- 计算月供：PMT = P * [r(1+r)^n] / [(1+r)^n - 1]
    SET v_monthly_payment = p_principal * 
        (p_rate * POWER(1 + p_rate, p_periods)) / 
        (POWER(1 + p_rate, p_periods) - 1);
    
    -- 生成每期还款计划
    WHILE v_period <= p_periods DO
        SET v_due_date = DATE_ADD((SELECT disbursed_at FROM loans WHERE loan_id = p_loan_id), 
                                  INTERVAL v_period MONTH);
        SET v_interest_payment = v_remaining_principal * p_rate;
        SET v_principal_payment = v_monthly_payment - v_interest_payment;
        SET v_remaining_principal = v_remaining_principal - v_principal_payment;
        
        INSERT INTO repayment_plan (
            plan_id, loan_id, period_num, due_date,
            principal_amount, interest_amount, outstanding_principal
        ) VALUES (
            CONCAT(p_loan_id, '_', LPAD(v_period, 3, '0')),
            p_loan_id, v_period, v_due_date,
            v_principal_payment, v_interest_payment, v_remaining_principal
        );
        
        SET v_period = v_period + 1;
    END WHILE;
END //
DELIMITER ;
```

## 6. ⚠️ 逾期管理机制


### 6.1 逾期管理核心概念


**什么是逾期**：
客户超过约定还款日期仍未还款的情况。逾期管理就是银行对这种情况的处理机制。

**逾期严重程度分级**：
```
逾期1-30天：   轻微逾期，电话提醒
逾期31-90天：  一般逾期，上门催收
逾期91-180天： 严重逾期，法律程序
逾期180天以上：不良资产，资产处置
```

### 7.1 信用评分模型(FICO评分机制)

```
还款历史 (35%): 过往还款记录，逾期情况

信用历史长度(15%): 信用账户平均年龄
信用类型组合(10%): 信贷产品多样性
```

```sql
-- 综合风险评估函数
DELIMITER //
CREATE FUNCTION calculate_risk_score(p_customer_id VARCHAR(32))
RETURNS DECIMAL(6,2)
READS SQL DATA
DETERMINISTIC
BEGIN
    DECLARE v_credit_score INT DEFAULT 0;
    DECLARE v_debt_ratio DECIMAL(6,4) DEFAULT 0;
    DECLARE v_income DECIMAL(12,2) DEFAULT 0;
    DECLARE v_risk_score DECIMAL(6,2) DEFAULT 0;
    
    -- 获取客户数据
    SELECT total_score INTO v_credit_score 
    FROM customer_credit_score WHERE customer_id = p_customer_id;
    
    SELECT debt_to_income_ratio, monthly_income INTO v_debt_ratio, v_income
    FROM customer_financial_profile WHERE customer_id = p_customer_id;
    
    -- 风险评分计算 (分数越低风险越高)
    SET v_risk_score = (v_credit_score / 850 * 40) +  -- 信用分占40%
                       (GREATEST(0, (0.5 - v_debt_ratio)) * 100 * 30) +  -- 负债比占30%
                       (LEAST(100, v_income / 1000 * 10) * 30);  -- 收入占30%
    
    RETURN ROUND(v_risk_score, 2);
END //
DELIMITER ;
```

## 8. 📊 信贷数据分析


### 8.1 信贷组合风险管理


**组合风险分析视图**：
```sql
-- 信贷组合分析视图
CREATE VIEW portfolio_risk_analysis AS
SELECT 
    p.product_name,
    COUNT(l.loan_id) as loan_count,
    SUM(l.loan_amount) as total_exposure,
    AVG(cs.total_score) as avg_credit_score,
    COUNT(CASE WHEN o.overdue_days > 30 THEN 1 END) as overdue_count,
    (COUNT(CASE WHEN o.overdue_days > 30 THEN 1 END) / COUNT(l.loan_id) * 100) as overdue_rate,
    SUM(CASE WHEN o.overdue_days > 90 THEN l.loan_amount ELSE 0 END) as npl_amount
FROM credit_products p
LEFT JOIN loans l ON p.product_id = l.product_id
LEFT JOIN customer_credit_score cs ON l.customer_id = cs.customer_id
LEFT JOIN overdue_records o ON l.loan_id = o.loan_id
WHERE l.status = 'active'
GROUP BY p.product_id;
```

```sql
-- 业务流程自动化统计
CREATE VIEW process_automation_stats AS
SELECT 
    DATE(created_at) as business_date,
    COUNT(*) as total_applications,
    COUNT(CASE WHEN approval_method = 'auto' THEN 1 END) as auto_approved,
    COUNT(CASE WHEN approval_method = 'manual' THEN 1 END) as manual_reviewed,
    (COUNT(CASE WHEN approval_method = 'auto' THEN 1 END) / COUNT(*) * 100) as automation_rate,
    AVG(TIMESTAMPDIFF(MINUTE, created_at, approved_at)) as avg_processing_minutes
FROM loan_applications
WHERE created_at >= DATE_SUB(CURRENT_DATE, INTERVAL 30 DAY)
GROUP BY DATE(created_at)
ORDER BY business_date DESC;
```

### 8.4 信贷合规报告


**合规指标监控**：
```sql
-- 合规报告视图
CREATE VIEW compliance_report AS
SELECT 
    '资本充足率' as indicator_name,
    CONCAT(ROUND(total_capital / risk_weighted_assets * 100, 2), '%') as current_value,
    '≥8%' as regulatory_requirement,
    CASE WHEN total_capital / risk_weighted_assets >= 0.08 THEN '达标' ELSE '不达标' END as status
FROM (
    SELECT 
        SUM(loan_amount * risk_weight) as risk_weighted_assets,
        (SELECT SUM(amount) FROM capital_reserves WHERE reserve_type = 'tier1') as total_capital
    FROM loans l 
    JOIN risk_weights rw ON l.risk_grade = rw.grade
    WHERE l.status = 'active'
) t

UNION ALL

SELECT 
    '不良贷款率',
    CONCAT(ROUND(npl_amount / total_loans * 100, 2), '%'),
    '≤5%',
    CASE WHEN npl_amount / total_loans <= 0.05 THEN '达标' ELSE '不达标' END
FROM (
    SELECT 
        SUM(CASE WHEN overdue_days > 90 THEN loan_amount ELSE 0 END) as npl_amount,
        SUM(loan_amount) as total_loans
    FROM loans l
    LEFT JOIN overdue_records o ON l.loan_id = o.loan_id
    WHERE l.status = 'active'
) t;
```

## 9. 📋 核心要点总结


### 9.1 必须掌握的核心概念


```
🔸 信贷生命周期：贷前→贷中→贷后三阶段管理
🔸 授信额度管理：客户可贷资金的总量控制
🔸 风险评估体系：多维度评估客户还款能力和意愿  
🔸 还款计划管理：科学制定还款方案，降低违约风险
🔸 逾期处置机制：分级管理，从催收到资产处置
🔸 数据驱动决策：通过数据分析优化信贷流程
```

### 9.2 关键理解要点


**🔹 信贷风险的本质**
```
信贷风险 = 违约概率 × 违约损失率 × 风险敞口

控制手段：
• 降低违约概率：严格准入标准，动态风险监控
• 减少损失率：完善担保措施，及时催收处置  
• 控制风险敞口：合理授信额度，分散投放
```

**🔹 信贷定价的平衡艺术**
```
定价考虑因素：
• 资金成本：银行获取资金的成本
• 风险溢价：补偿预期损失的溢价
• 运营成本：信贷业务运营费用
• 合理利润：股东回报要求

定价策略：差异化定价，风险与收益匹配
```

**🔹 数字化转型的核心**
```
传统信贷 vs 数字化信贷：
传统：人工审批，周期长，成本高
数字：自动化决策，秒级审批，规模效应

关键技术：
• 大数据风控：多维数据交叉验证
• 机器学习：智能评分和预测模型
• 流程自动化：减少人工干预，提升效率
```

### 9.3 实际应用价值


**🎯 业务价值**
- **风险控制**：建立全面的风险识别和管控体系
- **效率提升**：自动化流程，快速响应客户需求  
- **成本优化**：规模化运营，降低单笔业务成本
- **客户体验**：便捷申请，快速放款，个性化服务

**🔧 技术架构要点**
- **高并发处理**：应对大量贷款申请的并发访问
- **数据一致性**：确保资金流转和账务处理的准确性
- **系统集成**：与核心银行系统、征信系统等对接
- **安全防护**：客户信息保护和资金安全保障

**核心记忆口诀**：
- 信贷管理三阶段，贷前贷中贷后全
- 风险评估是核心，数据模型来支撑  
- 额度管理控总量，还款计划降风险
- 逾期处置要及时，合规经营是底线
