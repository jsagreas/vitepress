## 1. 🎯 MHA监控体系概述


### 1.1 什么是MHA监控体系


**🔸 简单理解**
MHA监控体系就像给数据库高可用环境安装了一套"健康检查系统"，随时监视MySQL主从复制集群的运行状况，一旦发现问题立即报警并采取行动。

```
传统运维方式：                MHA监控体系：
数据库出问题 → 人工发现     自动监控 → 实时发现 → 自动处理
     ↓                        ↓
手动切换主库                秒级自动故障转移
     ↓                        ↓  
业务中断时间长              业务中断时间极短
```

### 1.2 MHA监控的核心作用


**💡 核心功能**
- **🔍 实时状态检测**：持续监控主库、从库的运行状态
- **⚡ 故障快速发现**：几秒内发现主库宕机或网络故障
- **🚨 智能告警通知**：多渠道告警，确保运维人员及时知晓
- **🔄 自动故障处理**：无人值守的自动故障转移
- **📊 性能指标收集**：收集关键性能数据用于分析优化

## 2. 🔍 MHA状态监控实践


### 2.1 masterha_check_status检查工具


**🔸 工具作用**
`masterha_check_status` 是MHA提供的状态检查命令，用来查看MHA管理器当前的运行状态。

**💻 基本用法**
```bash
# 检查MHA管理器状态
masterha_check_status --conf=/etc/mha/app1.conf

# 输出示例：
app1 (pid:12345) is running(0:PING_OK), master:192.168.1.100
```

**📋 状态说明**
| 状态码 | 含义 | 说明 |
|--------|------|------|
| `PING_OK` | 正常运行 | MHA正在监控，主库状态正常 |
| `NOT_RUNNING` | 未运行 | MHA管理器进程未启动 |
| `INITIALIZING` | 初始化中 | MHA正在启动初始化 |
| `FAILOVER_RUNNING` | 故障转移中 | 正在执行自动故障转移 |

### 2.2 详细状态检查脚本


**🔧 自定义监控脚本**
```bash
#!/bin/bash
# MHA状态全面检查脚本

MHA_CONF="/etc/mha/app1.conf"
LOG_FILE="/var/log/mha/status_check.log"

# 检查MHA管理器状态
check_mha_status() {
    echo "$(date): 检查MHA状态..." >> $LOG_FILE
    
    STATUS=$(masterha_check_status --conf=$MHA_CONF 2>&1)
    
    if echo "$STATUS" | grep -q "is running"; then
        echo "✅ MHA运行正常" >> $LOG_FILE
        return 0
    else
        echo "❌ MHA状态异常: $STATUS" >> $LOG_FILE
        return 1
    fi
}

# 检查MySQL复制状态
check_replication() {
    # 这里可以添加检查从库复制延迟的逻辑
    mysql -h slave1 -e "SHOW SLAVE STATUS\G" | grep "Seconds_Behind_Master"
}

# 执行检查
check_mha_status
check_replication
```

### 2.3 监控指标详解


**📊 核心监控指标**

`🔸 连接性监控`
- **主库连接状态**：确保能正常连接到主库
- **从库连接状态**：检查所有从库的连接
- **SSH连接状态**：验证MHA节点间的SSH连接

`🔸 复制监控`
- **复制延迟**：从库落后主库的时间
- **复制状态**：IO线程和SQL线程运行状态
- **GTID一致性**：确保事务ID的一致性

`🔸 性能监控`
- **QPS(每秒查询数)**：数据库查询压力
- **连接数**：当前活跃连接数量
- **缓冲池使用率**：InnoDB缓冲池的使用情况

### 3.3 日志监控脚本


**🔧 日志实时监控**
```bash
#!/bin/bash
# MHA日志监控脚本

MHA_LOG="/var/log/mha/app1.log"
ALERT_KEYWORDS="error|warning|timeout|failed|died"

# 实时监控日志中的关键词
tail -f $MHA_LOG | while read line; do
    if echo "$line" | grep -iE "$ALERT_KEYWORDS"; then
        # 发现异常信息，发送告警
        echo "🚨 MHA告警: $line"
        # 这里可以调用告警脚本
        send_alert "MHA异常" "$line"
    fi
done
```

**📊 日志分析报表**

## 4. 🚨 告警机制配置


### 4.1 告警触发条件


**🎯 告警触发场景**
- **主库连接失败**：超过设定的ping超时时间
- **复制延迟过大**：从库延迟超过阈值(如30秒)
- **MHA进程异常**：MHA管理器进程意外停止
- **故障转移事件**：发生了自动或手动故障转移
- **磁盘空间不足**：MySQL数据目录空间使用率过高

## 5. 🔧 自动化部署与管理


### 5.1 MHA自动化部署脚本


**🚀 一键部署脚本**
```bash
#!/bin/bash
# MHA自动化部署脚本

# 配置变量
MHA_VERSION="0.58"
INSTALL_DIR="/opt/mha"
CONFIG_DIR="/etc/mha"

# 创建目录
create_directories() {
    echo "📁 创建MHA目录..."
    mkdir -p $INSTALL_DIR
    mkdir -p $CONFIG_DIR
    mkdir -p /var/log/mha
}

# 安装MHA依赖
install_dependencies() {
    echo "📦 安装依赖包..."
    yum install -y perl-DBD-MySQL perl-Config-Tiny perl-Log-Dispatch
}

# 下载并安装MHA
install_mha() {
    echo "⬇️ 下载MHA安装包..."
    cd /tmp
    wget https://github.com/yoshinorim/mha4mysql-manager/releases/download/v$MHA_VERSION/mha4mysql-manager-$MHA_VERSION.tar.gz
    
    echo "🔧 编译安装MHA..."
    tar -zxf mha4mysql-manager-$MHA_VERSION.tar.gz
    cd mha4mysql-manager-$MHA_VERSION
    perl Makefile.PL
    make && make install
}

# 生成配置文件
generate_config() {
    echo "📝 生成MHA配置文件..."
    cat > $CONFIG_DIR/app1.conf << 'EOF'
[server default]
manager_workdir=/var/log/mha
manager_log=/var/log/mha/app1.log
remote_workdir=/tmp
ssh_user=root
repl_user=repl
repl_password=replpassword
ping_interval=3
ping_type=SELECT

[server1]
hostname=192.168.1.100
port=3306
candidate_master=1

[server2]  
hostname=192.168.1.101
port=3306
candidate_master=1

[server3]
hostname=192.168.1.102
port=3306
no_master=1
EOF
}

# 执行部署
main() {
    echo "🚀 开始MHA自动化部署..."
    create_directories
    install_dependencies
    install_mha
    generate_config
    echo "✅ MHA部署完成！"
}

main
```

### 5.2 运维管理工具


**🛠️ MHA管理脚本集**
```bash
#!/bin/bash
# MHA运维管理工具 mha_admin.sh

CONF_FILE="/etc/mha/app1.conf"

# 显示帮助信息
show_help() {
    cat << EOF
MHA管理工具使用说明:

🔍 状态查看:
  ./mha_admin.sh status    - 查看MHA运行状态
  ./mha_admin.sh check     - 检查环境配置

🔧 服务控制:  
  ./mha_admin.sh start     - 启动MHA监控
  ./mha_admin.sh stop      - 停止MHA监控
  ./mha_admin.sh restart   - 重启MHA监控

🚨 故障处理:
  ./mha_admin.sh failover  - 手动故障转移
  ./mha_admin.sh recover   - 恢复故障节点

📊 报告生成:
  ./mha_admin.sh report    - 生成运行报告
  ./mha_admin.sh logs      - 查看实时日志
EOF
}

# 检查MHA状态
check_status() {
    echo "🔍 检查MHA状态..."
    masterha_check_status --conf=$CONF_FILE
}

# 启动MHA监控
start_mha() {
    echo "🚀 启动MHA监控..."
    nohup masterha_manager --conf=$CONF_FILE > /dev/null 2>&1 &
    sleep 2
    check_status
}

# 停止MHA监控
stop_mha() {
    echo "⏹️ 停止MHA监控..."
    masterha_stop --conf=$CONF_FILE
}

# 根据参数执行对应操作
case "$1" in
    "status")   check_status ;;
    "start")    start_mha ;;
    "stop")     stop_mha ;;
    "restart")  stop_mha; sleep 3; start_mha ;;
    "help"|*)   show_help ;;
esac
```

**📊 MHA指标定义**
```bash
# MHA Prometheus Exporter脚本示例
#!/bin/bash
# 生成MHA监控指标供Prometheus采集

METRICS_FILE="/tmp/mha_metrics.prom"

# 生成MHA状态指标
echo "# HELP mha_status MHA manager running status" > $METRICS_FILE
echo "# TYPE mha_status gauge" >> $METRICS_FILE

if masterha_check_status --conf=/etc/mha/app1.conf | grep -q "running"; then
    echo "mha_status 1" >> $METRICS_FILE
else
    echo "mha_status 0" >> $METRICS_FILE
fi

# 生成复制延迟指标
echo "# HELP mysql_slave_lag_seconds MySQL slave lag in seconds" >> $METRICS_FILE
echo "# TYPE mysql_slave_lag_seconds gauge" >> $METRICS_FILE

# 从各个从库获取延迟信息
for slave in slave1 slave2; do
    lag=$(mysql -h $slave -e "SHOW SLAVE STATUS\G" | grep "Seconds_Behind_Master" | awk '{print $2}')
    echo "mysql_slave_lag_seconds{instance=\"$slave\"} $lag" >> $METRICS_FILE
done
```
