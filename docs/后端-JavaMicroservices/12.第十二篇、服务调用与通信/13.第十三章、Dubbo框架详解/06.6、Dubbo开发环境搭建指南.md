## 2. ☕ JDK环境配置


### 2.1 什么是JDK

**通俗理解**：JDK就是Java开发工具包，是运行和开发Java程序的基础。

```
JDK = 发动机
Java程序 = 汽车
没有发动机，汽车跑不了
```

**🔸 JDK版本选择建议**
- **JDK 8**：稳定成熟，企业主流（推荐新手）
- **JDK 11**：长期支持版本，性能提升
- **JDK 17**：最新LTS版本，新特性多

### 2.2 JDK安装步骤


#### 📥 下载JDK


> 💡 **推荐下载源**：
> - Oracle官网：https://www.oracle.com/java/technologies/downloads/
> - OpenJDK：https://adoptium.net/

**Windows系统**：
1. 下载 `.exe` 安装包
2. 双击运行，选择安装目录
3. 建议路径：`C:\Program Files\Java\jdk-8`

**macOS系统**：
1. 下载 `.dmg` 安装包
2. 双击安装即可

**Linux系统**：
```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install openjdk-8-jdk

# CentOS/RedHat
sudo yum install java-1.8.0-openjdk-devel
```

#### ⚙️ 配置环境变量


**Windows配置**：

```
步骤1: 右键"此电脑" → "属性" → "高级系统设置"
步骤2: 点击"环境变量"
步骤3: 新建系统变量

变量名：JAVA_HOME
变量值：C:\Program Files\Java\jdk-8

步骤4: 编辑Path变量，添加：
%JAVA_HOME%\bin
```

**macOS/Linux配置**：

```bash
# 编辑配置文件
vim ~/.bash_profile  # 或 ~/.zshrc

# 添加以下内容
export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk1.8.0_xxx.jdk/Contents/Home
export PATH=$JAVA_HOME/bin:$PATH

# 使配置生效
source ~/.bash_profile
```

#### ✅ 验证安装


```bash
# 检查Java版本
java -version

# 应该看到类似输出：
# java version "1.8.0_xxx"
# Java(TM) SE Runtime Environment...

# 检查编译器
javac -version

# 应该看到：
# javac 1.8.0_xxx
```

> ⚠️ **常见问题**：
> - 提示"java不是内部或外部命令"：环境变量未配置或未生效
> - 解决：重启命令行窗口或重启电脑

## 3. 📦 构建工具安装


### 3.1 什么是构建工具

**通俗理解**：构建工具就像"采购员+装配工"，帮你自动下载需要的库，并把项目打包好。

```
手工方式：
❌ 自己下载jar包
❌ 手动配置classpath
❌ 手动编译打包

使用Maven/Gradle：
✅ 自动下载依赖
✅ 统一项目结构
✅ 一键编译打包
```

### 3.2 Maven安装配置


#### 📥 下载安装Maven


> 💡 **下载地址**：https://maven.apache.org/download.cgi

**步骤说明**：

```
步骤1: 下载二进制压缩包
       apache-maven-3.8.x-bin.zip

步骤2: 解压到指定目录
       建议路径：C:\Program Files\Apache\maven

步骤3: 配置环境变量
       MAVEN_HOME = C:\Program Files\Apache\maven
       Path添加：%MAVEN_HOME%\bin

步骤4: 验证安装
       mvn -version
```

#### ⚙️ Maven核心配置


**配置本地仓库路径**：

```xml
<!-- 编辑 maven/conf/settings.xml -->
<settings>
  <!-- 本地仓库位置 -->
  <localRepository>D:/maven/repository</localRepository>
</settings>
```

**配置阿里云镜像**（加速下载）：

```xml
<mirrors>
  <mirror>
    <id>aliyun</id>
    <name>阿里云公共仓库</name>
    <url>https://maven.aliyun.com/repository/public</url>
    <mirrorOf>central</mirrorOf>
  </mirror>
</mirrors>
```

> 💡 **为什么配置镜像**：Maven默认从国外服务器下载，速度很慢，配置国内镜像可以大幅加速。

### 3.3 Gradle安装（可选）


**Gradle vs Maven对比**：

| 特性 | **Maven** | **Gradle** |
|------|----------|----------|
| 配置文件 | `pom.xml`（XML格式） | `build.gradle`（Groovy/Kotlin） |
| 学习曲线 | 📗 简单，新手友好 | 📘 稍陡，功能强大 |
| 构建速度 | 🐢 较慢 | 🚀 更快 |
| 社区资源 | 📚 丰富 | 📖 成长中 |

**新手建议**：先学Maven，熟悉后再尝试Gradle

#### 📥 Gradle快速安装


```bash
# Windows (使用Chocolatey)
choco install gradle

# macOS (使用Homebrew)
brew install gradle

# 验证安装
gradle -version
```

## 4. 📋 注册中心部署


### 4.1 什么是注册中心

**通俗理解**：注册中心就像"通讯录"或"电话簿"，记录了所有服务的地址，让服务之间能互相找到。

```
没有注册中心：
消费者：我要调用订单服务
         但订单服务在哪？不知道！❌

有注册中心：
消费者 → 注册中心：订单服务在哪？
注册中心 → 消费者：在192.168.1.100:20880
消费者 → 订单服务：调用成功！✅
```

**🔸 注册中心的核心作用**
- **服务注册**：服务启动时向注册中心报到
- **服务发现**：消费者从注册中心查询服务地址
- **健康检查**：定期检查服务是否存活
- **动态更新**：服务下线时自动从注册中心移除

### 4.2 Zookeeper注册中心


#### 🐘 Zookeeper简介


> **Zookeeper**：一个分布式协调服务，Dubbo最常用的注册中心。

**特点**：
- ✅ 成熟稳定，企业广泛使用
- ✅ 支持分布式集群部署
- ❌ 配置相对复杂

#### 📥 Zookeeper安装


**Windows安装**：

```
步骤1: 下载Zookeeper
       地址：https://zookeeper.apache.org/releases.html
       选择：apache-zookeeper-3.8.x-bin.tar.gz

步骤2: 解压到指定目录
       例如：D:\zookeeper

步骤3: 配置文件
       复制 conf/zoo_sample.cfg → conf/zoo.cfg
       
步骤4: 编辑 zoo.cfg
       dataDir=D:/zookeeper/data
       clientPort=2181

步骤5: 启动Zookeeper
       运行：bin\zkServer.cmd
```

**Linux安装**：

```bash
# 下载解压
wget https://dlcdn.apache.org/zookeeper/zookeeper-3.8.3/apache-zookeeper-3.8.3-bin.tar.gz
tar -zxvf apache-zookeeper-3.8.3-bin.tar.gz
cd apache-zookeeper-3.8.3-bin

# 配置
cp conf/zoo_sample.cfg conf/zoo.cfg
vim conf/zoo.cfg

# 修改数据目录
dataDir=/opt/zookeeper/data

# 启动
bin/zkServer.sh start

# 查看状态
bin/zkServer.sh status
```

#### ✅ 验证Zookeeper


```bash
# 连接Zookeeper客户端
zkCli.cmd  # Windows
# 或
bin/zkCli.sh  # Linux

# 成功连接后会看到：
# [zk: localhost:2181(CONNECTED) 0]

# 测试命令
ls /
# 输出：[zookeeper]
```

### 4.3 Nacos注册中心


#### 🚀 Nacos简介


> **Nacos**：阿里开源的注册中心，比Zookeeper更现代化。

**优势对比**：

| 特性 | **Zookeeper** | **Nacos** |
|------|--------------|----------|
| 安装难度 | 🟡 中等 | 🟢 简单 |
| 管理界面 | ❌ 无 | ✅ 有Web控制台 |
| 配置中心 | ❌ 不支持 | ✅ 内置支持 |
| 学习成本 | 📘 需要学习ZK概念 | 📗 开箱即用 |

**新手推荐**：优先选择Nacos，更容易上手

#### 📥 Nacos安装


**单机模式安装**（适合开发测试）：

```bash
# 下载Nacos
wget https://github.com/alibaba/nacos/releases/download/2.2.3/nacos-server-2.2.3.tar.gz

# 解压
tar -zxvf nacos-server-2.2.3.tar.gz
cd nacos/bin

# Windows启动
startup.cmd -m standalone

# Linux/macOS启动
sh startup.sh -m standalone
```

**访问控制台**：
- URL：http://localhost:8848/nacos
- 默认账号：`nacos`
- 默认密码：`nacos`

#### 🎯 Nacos核心配置


**修改数据库存储**（可选）：

```yaml
# 编辑 conf/application.properties

# 配置MySQL数据库
spring.datasource.platform=mysql
db.num=1
db.url.0=jdbc:mysql://127.0.0.1:3306/nacos?characterEncoding=utf8
db.user.0=root
db.password.0=123456
```

> 💡 **提示**：默认使用内置Derby数据库，生产环境建议使用MySQL

## 5. 🖥️ Dubbo Admin管理控制台


### 5.1 什么是Dubbo Admin

**通俗理解**：Dubbo Admin是一个可视化管理界面，让你能看到所有服务的状态，就像"监控大屏"。

**🔸 主要功能**
- 📊 服务查询：查看所有注册的服务
- 🔍 服务详情：查看服务提供者和消费者
- 📈 服务治理：动态配置、路由规则
- 🐛 问题排查：查看调用关系、性能指标

### 5.2 Dubbo Admin部署


#### 📥 下载部署


**方式一：使用Docker（推荐）**：

```bash
# 拉取镜像
docker pull apache/dubbo-admin

# 启动容器
docker run -d \
  --name dubbo-admin \
  -p 8080:8080 \
  -e admin.registry.address=zookeeper://127.0.0.1:2181 \
  -e admin.config-center=zookeeper://127.0.0.1:2181 \
  apache/dubbo-admin
```

**方式二：源码部署**：

```bash
# 克隆代码
git clone https://github.com/apache/dubbo-admin.git
cd dubbo-admin

# 修改配置
vim dubbo-admin-server/src/main/resources/application.properties

# 配置注册中心地址
admin.registry.address=zookeeper://127.0.0.1:2181
admin.config-center=zookeeper://127.0.0.1:2181

# 编译打包
mvn clean package -Dmaven.test.skip=true

# 启动
java -jar dubbo-admin-server/target/dubbo-admin-server-0.5.0.jar
```

#### 🎯 访问控制台


```
访问地址：http://localhost:8080
默认账号：root
默认密码：root
```

**🔸 控制台界面说明**

| 菜单 | **功能说明** |
|------|------------|
| 服务查询 | 查看所有服务列表 |
| 服务详情 | 查看服务的提供者和消费者 |
| 服务测试 | 在线测试服务调用 |
| 服务治理 | 配置路由规则、负载均衡 |

## 7. ✅ 环境验证


### 7.1 验证清单

```
☑️ JDK
   命令：java -version

   期望：显示Maven版本和Java环境

   期望：能正常登录

☑️ IDEA
   打开IDEA，创建Maven项目测试
```

### 7.2 创建测试项目


**快速验证环境是否正常**：

```xml
<!-- 创建Maven项目，pom.xml添加 -->
<dependencies>
    <dependency>
        <groupId>org.apache.dubbo</groupId>
        <artifactId>dubbo</artifactId>
        <version>3.2.0</version>
    </dependency>
    <dependency>
        <groupId>org.apache.dubbo</groupId>
        <artifactId>dubbo-dependencies-zookeeper</artifactId>
        <version>3.2.0</version>
        <type>pom</type>
    </dependency>
</dependencies>
```

**如果依赖能正常下载，说明环境配置成功！**

### 7.3 常见问题排查


**问题1：Maven下载依赖很慢**
```
✅ 解决：检查settings.xml是否配置了阿里云镜像
```

**问题2：Zookeeper启动失败**
```
✅ 检查：2181端口是否被占用
命令：netstat -ano | findstr 2181
解决：关闭占用进程或修改端口
```

**问题3：IDEA无法识别Maven项目**
```
✅ 解决：右键pom.xml → Add as Maven Project
或者：File → Invalidate Caches → Restart
```
